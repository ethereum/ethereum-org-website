# Gemini Feedback: Latin Script vs Transliteration Rules (Round 3)

Categorization of 50 key terms for non-Latin script languages.

## Category A: Always Keep Latin (Technical Standards & Tickers)
ETH, ERC-20, ERC-721, ERC-1155, Web3, Gwei, IPFS, ENS, Solidity, DeFi

## Category B: Always Translate/Transliterate
- JP prefers katakana transliteration; CN/AR prefer semantic translation
- Terms: Ethereum, Blockchain, Wallet, Node, Validator, Address, Private Key,
  Public Key, Seed Phrase, Mining, Smart Contract, Token, Consensus,
  Decentralized, Protocol, Governance, Liquidity, Bridge, Oracle, Mainnet

## Category C: Context-Dependent (Latin in tags, native in prose)
NFT, DAO, DApp, Stablecoin, Rollup, Sharding, Yield Farming, Hash,
Beacon Chain, The Merge

## Category D: Hybrid (Latin acronym + native full name)
EVM (Ethereum Virtual Machine), PoS (Proof of Stake), PoW (Proof of Work),
Layer 2 / L2, ZK-Rollup, Optimistic Rollup, TVL (Total Value Locked),
EIP (Ethereum Improvement Proposal), MEV (Maximal Extractable Value),
DEX (Decentralized Exchange)
