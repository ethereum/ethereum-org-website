# คำสั่งอัปเดต LLMS.txt (Update LLMS.txt Command)

คำสั่งนี้ช่วยดูแลรักษาไฟล์ `public/llms.txt` โดยการตรวจสอบการเปลี่ยนแปลงจากไฟล์นำทางหลักของเว็บไซต์:

1. **เมนูหลัก**: `src/components/Nav/useNavigation.ts`
2. **เอกสารนักพัฒนา**: `src/data/developer-docs-links.yaml`
3. **ลิงก์ส่วนท้าย (Footer)**: `src/components/Footer.tsx`

## หลักการทำงาน

- เพิ่มลิงก์ที่ขาดหายไปลงในหมวดหมู่ที่เหมาะสม
- คงคำอธิบายและโครงสร้างเดิมที่มีอยู่ไว้
- ปฏิบัติตามโครงสร้างมาตรฐานของ llms.txt
- **เน้นใช้ URL ที่ชี้ไปยังไฟล์ Markdown (.md)** แทน URL หน้าเว็บปกติ เพื่อให้ LLM เข้าใจเนื้อหาได้ดีขึ้น

## ขั้นตอนการดำเนินงาน

### ขั้นตอนที่ 1: วิเคราะห์ไฟล์นำทาง (Navigation)

- **เมนูหลัก**: ดึงโครงสร้างจาก `linkSections` (learn, use, build, participate) เพื่อเอา `href`, `label` และ `description`
- **เอกสารนักพัฒนา**: วิเคราะห์ไฟล์ YAML เพื่อจัดลำดับความสัมพันธ์ของเนื้อหาด้านเทคนิค
- **ส่วนท้าย (Footer)**: ดึงข้อมูลจาก `linkSections` และ `dipperLinks` รวมถึงลิงก์ภายนอกที่สำคัญ (Blog, ESP, Devcon)

### ขั้นตอนที่ 2: วิเคราะห์ไฟล์ llms.txt ปัจจุบัน

- ตรวจสอบหมวดหมู่และลิงก์ที่มีอยู่แล้ว
- ดึงคำอธิบายปัจจุบันและโครงสร้างลำดับชั้นออกมา
