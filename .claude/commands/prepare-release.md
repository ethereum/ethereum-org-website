# คำสั่งเตรียมการ Release (Prepare Release Command)

ช่วยจัดการกระบวนการ Deploy ของ ethereum.org โดยอัตโนมัติ โดยใช้สคริปต์ `src/scripts/prepare-release.sh` เพื่อความแม่นยำ และใช้ความสามารถของ AI ในการวิเคราะห์ประเภทเวอร์ชันรวมถึงการปรับแต่งบันทึกการเปลี่ยนแปลงให้สะอาดเรียบร้อย

## อาร์กิวเมนต์ (Arguments)

รายละเอียดสำหรับ $ARGUMENTS

- `--dry-run` - จำลองขั้นตอนที่จะเกิดขึ้นโดยไม่มีการเปลี่ยนแปลงข้อมูลบน Remote จริง
- `--major` - การ Release เวอร์ชันหลัก (มีการเปลี่ยนแปลงโครงสร้างสำคัญ หรือ Breaking Changes)
- `--minor` - การ Release เวอร์ชันย่อย (เพิ่มฟีเจอร์ใหม่, เนื้อหาใหม่ หรือไฟล์แปลภาษาใหม่)
- `--patch` - การ Release เวอร์ชันแก้ไข (แก้บั๊ก, แก้คำผิด หรืออัปเดตเล็กน้อย)
- _(หากไม่ระบุ)_ - วิเคราะห์การเปลี่ยนแปลงและแนะนำประเภทเวอร์ชันที่เหมาะสม

**โหมด Dry-run**: หากพบ `--dry-run` ใน `$ARGUMENTS` ให้ส่งค่าเป็นอาร์กิวเมนต์แรกไปยังทุกคำสั่งของสคริปต์ เพื่อจำลองขั้นตอนโดยไม่ push หรือสร้าง PR จริง

## ขั้นตอนการทำงาน (Execution Flow)

### ขั้นตอนที่ 1: ตรวจสอบความพร้อม (Pre-flight Checks)

ตรวจสอบว่ามีการใช้ `--dry-run` หรือไม่ หากมีให้ตั้งค่า `DRY_RUN_FLAG="--dry-run"` หากไม่มีให้ปล่อยว่าง

รันสคริปต์เพื่อตรวจสอบสภาพแวดล้อมและซิงค์ Branch:

```bash
./src/scripts/prepare-release.sh $DRY_RUN_FLAG preflight
