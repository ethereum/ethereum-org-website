A tutorial by the **Ethereum Foundation** walking ethereum.org translators through the Crowdin localization platform. The tutorial covers onboarding, the translation editor, translation memory, the glossary, and tips for an efficient workflow for those contributing to the [ethereum.org translation program](/contributing/translation-program/).

*This transcript is an accessible copy of the [original video transcript](https://www.youtube.com/watch?v=Ii7bYhanLs4) published by Ethereum Foundation. It has been lightly edited for readability.*

#### Introduction to Crowdin (0:00) {#introduction-to-crowdin-000}

Hello and welcome to this short video guide on using Crowdin to translate ethereum.org. Crowdin is a localization tool that simplifies content management. It makes the translators' and reviewers' jobs easier with professional translation tools like the translation memory and glossary, and it also contains powerful crowdsourcing and collaboration functionality. This is the tool that we use to translate the website.

Let's take a quick dive and check out how to use it.

#### Getting started (0:39) {#getting-started-039}

Onboarding onto Crowdin is really simple. All you need to do is sign up using an email and password, or log in if you're already signed up.

This is the first screen that you will see in the project. All you have to do is navigate to your language. Once you open up a specific language, you will see folders that are based on priority. We always want the topmost folders to get translated first and added to the website first. As we can see, the site navigation in this case is not 100% translated since it was recently updated, so this is what we will start with.

#### Using the online editor (1:26) {#using-the-online-editor-126}

This is the Crowdin online editor. This is where you will be doing most of your work. Navigate to the first untranslated string, input your translation, and save it by clicking "Save." That's it — we've just suggested a translation, albeit an incorrect one in this case.

You can also use the translation memory, which contains all of the previous suggested translations. It shows the differences between the current string and previously translated strings. This is useful for translating strings that are similar to existing ones, or that are repeated, so no repeated string needs to get translated twice.

#### Comments and collaboration (2:30) {#comments-and-collaboration-230}

In Crowdin, you can also leave a comment for other translators, for the team, or for reviewers. You can create an issue and ask a question, report that the current translation is incorrect, request more context on certain strings, or report mistakes in the source if you find them.

This is one of the features that allows for great collaboration, because it allows people to communicate with each other and raise issues that anyone can see and help resolve.

#### Translation memory (3:16) {#translation-memory-316}

Crowdin also contains a translation memory where you can search for certain terms and see all of their previously approved or suggested translations, regardless of whether the string only appears in the project once or a hundred times. The translation memory is a great tool to make sure that you don't need to translate anything twice.

#### The glossary (3:48) {#the-glossary-348}

Another useful feature of Crowdin is the glossary, which shows descriptions and established translations of key terms. For example, we can see that the word "blockchain" is underlined. Simply by hovering over this underlined word, we can already see what the correct translation of the word is, along with a description of what it means. If there are technical or Ethereum-specific terms and you're not sure what they mean, you can simply check the glossary and see the definition and description for these terms.

#### Filtering and editor settings (4:36) {#filtering-and-editor-settings-436}

These are the key features that Crowdin offers. You can also filter the strings by showing untranslated strings first. You can use more advanced features like only showing strings that haven't been translated yet, so you don't even have to go through anything that's already been translated and approved. This makes for quick work.

You can change the files that you want to open directly in the Crowdin editor by clicking in the top section. You can also change the language that you want to be translating in directly in the editor, so you don't have to go back and forth opening and closing files. You can change your settings and your view, and enable or disable certain things.

#### Closing (5:48) {#closing-548}

That is pretty much it for the basics. These are some of the most basic features that Crowdin offers, but by learning how to use them, your work as a translator will already be very much simplified — much easier, much faster, much more straightforward. Especially using the translation memory and glossary can be very helpful to make sure that you have the appropriate context, that you can see all the previous translations, and that you don't have to translate anything twice if it's already been translated.

Thank you for looking to get involved in translating ethereum.org and help us make the website accessible to everyone regardless of the language that they speak.
