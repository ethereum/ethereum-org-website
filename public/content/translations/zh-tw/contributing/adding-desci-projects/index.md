我們希望確保展示多樣化的專案，並提供 DeSci 領域的良好概覽。

任何人都可以自由建議將專案列入 ethereum.org 的 DeSci 頁面。同樣地，任何人若發現某個專案不再相關或不再符合我們的資格標準，也可以自由建議將其移除。

## 決策框架 {#the-decision-framework}

### 收錄標準：必備條件 {#the-must-haves}

- **開源程式碼/資料** - 程式碼與資料的開放性是 DeSci 的核心原則，因此 DeSci 專案絕不能是閉源的。程式碼庫應該是可存取的，且最好開放接受 PR (拉取請求)。
- **DeSci 專案應具備可證明的去中心化特性** - 這可能包括由 DAO 治理，或是使用包含非託管錢包在內的去中心化技術堆疊來建置。這通常涉及以太坊上可稽核的智能合約。
- **誠實且準確的列表資訊** - 我們期望專案建議的任何列表資訊都是誠實且準確的。若產品偽造列表資訊 (例如在非開源的情況下宣稱產品為「開源」)，將會被移除。
- **對擴大科學參與度有可證明的承諾** - DeSci 專案應能清楚說明他們如何擴大一般大眾對科學的參與，而不僅限於代幣/NFT 持有者。
- **全球皆可存取** - 你的專案沒有地理限制或 KYC 要求，不會將特定人群排除在服務之外。
- **資訊豐富的網站與文件** - 專案網站的訪客必須能了解專案的實際運作內容、它如何貢獻於科學基礎設施的去中心化，以及如何參與。
- **專案應為以太坊生態系統的一部分** - 在 ethereum.org，我們相信以太坊 (及其第二層 (L2)) 是 DeSci 運動合適的基礎層。
- **專案已具備相當的基礎** - 專案擁有真實使用者，且這些使用者已經能夠存取專案服務達數個月之久。

### 加分條件 {#nice-to-haves}

- **支援多種語言** - 你的專案已翻譯成多種語言，讓世界各地的使用者都能存取。
- **教育資源** - 你的產品應具備精心設計的入門引導體驗，以協助並教育使用者。或者提供如文章或影片等教學內容的證明。
- **第三方稽核** - 你的產品已由受信任的第三方進行專業的漏洞稽核。
- **聯絡窗口** - 專案的聯絡窗口 (可能是 DAO 或社群的代表) 將大幅協助我們在發生變更時取得準確資訊。這將使 ethereum.org 在未來收集資訊時的更新工作更易於管理。

## 維護 {#maintenance}

由於以太坊具有流動的本質，團隊與產品來來去去，且創新每天都在發生，因此我們將對內容進行例行檢查，以：

- 確保所有列出的專案仍符合我們的標準
- 驗證是否有被建議的產品比目前列出的產品更符合我們的標準

Ethereum.org 由開源社群維護，我們仰賴社群協助保持內容的最新狀態。如果你發現任何關於已列出專案的資訊需要更新，請在我們的 GitHub 儲存庫中開啟 issue 或 pull request。
