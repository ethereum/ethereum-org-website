設計是任何專案的關鍵組成部分，透過將您的時間和設計技能貢獻給 ethereum.org，您可以幫助改善我們訪客的使用者體驗。為開源專案做出貢獻提供了一個在協作環境中獲得相關經驗和發展技能的機會。您將有機會與其他設計師、開發人員和社群成員合作，他們每個人都有自己獨特的觀點和見解。

最終，這是建立一個多樣化且令人印象深刻的作品集以展示您設計技能的絕佳方式。

## 如何貢獻？ {#how-to-contribute}

### <Emoji text=":one:" size={1} /> &nbsp;提供對早期設計原型的回饋 {#design-critique}

我們有時需要協助測試我們的初步構想。這是一個無需任何技術知識即可做出貢獻的絕佳方式。

1. 設計團隊將在 [Discord](https://discord.com/invite/ethereum-org) 和 [GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) 上分享設計模型。
2. 您將被引導瀏覽設計，並透過留言功能提供回饋。
3. 結果將分享在 GitHub 議題 (issue) 中，然後由團隊關閉。

### <Emoji text=":two:" size={1} /> &nbsp;參與問卷調查研究 {#answer-surveys}

透過以下方式提供對我們網站的回饋：

1. 造訪 ethereum.org 並閱讀頁面。
2. 點擊右下角的回饋小工具，並回答與設計和內容相關的問題。
3. 專注於自由作答形式的問題。

### <Emoji text=":three:" size={1} /> &nbsp;在網站上尋找與設計相關的問題並回報 {#report-design-issues}

ethereum.org 是一個快速成長的網站，擁有許多功能和內容。部分使用者介面 (UI) 很容易過時或有改進的空間。如果您遇到任何此類情況，請回報給我們，以便我們關注處理。

1. 瀏覽網站並留意其設計。
2. 如果您看到任何視覺或使用者體驗 (UX) 問題，請截圖並做筆記。
3. 使用[錯誤回報 (bug report)](https://github.com/ethereum/ethereum-org-website/issues/new/choose)來回報發現的問題。

### <Emoji text=":four:" size={1} /> &nbsp;提出設計變更 {#propose-design-changes}

如果您樂於接受設計挑戰，可以造訪我們的 GitHub 議題看板，並篩選出[與設計相關的議題](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8)。

1. 瀏覽我們的網站並留意其設計，或者前往我們的 GitHub 儲存庫並查看標有[「Design required (需要設計)」標籤](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8)的議題。
2. 構思解決方案並進行設計（最好使用我們的[設計系統](https://www.figma.com/community/file/1134414495420383395)）。
3. 在相應的 GitHub 議題中提出解決方案，或[建立一個新議題。](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. 等待設計團隊審查。

### <Emoji text=":five:" size={1} /> &nbsp;共同建立設計系統 {#contribute-to-design-system}

我們的設計系統讓設計 ethereum.org 變得有趣且輕鬆。如果您是一位經驗豐富的設計師，您可以協助我們為網站準備許多元件。

1. 從 GitHub 上的[設計系統看板](https://github.com/ethereum/ethereum-org-website/labels/design%20system)選擇一個要處理的議題，或建立一個新議題。
2. 請求將所選議題指派給您。
3. 開始在 Figma 中設計所需的元件。
4. 當您需要審查或指導時，在 GitHub 上與設計團隊分享。
5. 設計團隊將進行審查。
6. 設計團隊會將變更整合到主檔案中，並將檔案發布給社群。

### <Emoji text=":six:" size={1} /> &nbsp;在網站上撰寫與設計相關的內容 {#write-design-articles}

以太坊開發者社群非常強大，但設計社群則稍微落後。如果您是一位具備 Web3 知識的設計師，請考慮與更廣泛的社群分享您的學習成果，以便我們能夠共同成長和進步；我們有一個[關於以太坊設計的頁面](/developers/docs/design-and-ux/)，歡迎您來貢獻。您也可以查看我們的[上架政策](/contributing/design/adding-design-resources)。

1. 構思 ethereum.org 應該涵蓋且對該領域設計師有益的設計主題。
2. 前往我們的 GitHub 儲存庫並[提出一個議題](https://github.com/ethereum/ethereum-org-website/issues/new)來提議主題（請先不要撰寫內容）。
3. 等待設計團隊核准。
4. 一旦獲得核准，即可撰寫內容。
5. 在相應的 GitHub 議題中提交內容。

### <Emoji text=":seven:" size={1} /> &nbsp;繪製新插圖 {#prepare-illustrations}

視覺化是解釋抽象主題最強大的工具之一。加入圖表和資訊圖表具有巨大的潛力。畢竟，一圖勝千言。

1. 瀏覽我們的網站，看看哪些頁面可以加入新的資訊圖表。
2. 確保插圖風格與我們的[資產](/assets/)相符。
3. 前往我們的 GitHub 儲存庫並[提出一個議題](https://github.com/ethereum/ethereum-org-website/issues/new)來提議該插圖。
4. 設計團隊將對其進行審查。
5. 我們會建立一個新議題，請開發人員實作這張新圖片。
