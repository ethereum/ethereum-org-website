## 新增報告 {#adding-a-report}

報告會顯示在 [ethereum.org/reports/](/reports/)，並精選來自各種來源與以太坊相關的研究、分析和出版物。任何人都可以建議新增報告。

### 上架政策 {#listing-policy}

Ethereum.org 是一個中立的教育資源。報告頁面的策展目的是：

- **讓讀者了解**以太坊技術、經濟學、安全性、治理，以及更廣泛的政策與監管環境
- 在數據與分析上**保持準確**
- **保持**與以太坊社群和生態系統的**相關性**

本網站不會列出主要用於推廣特定產品、代幣或商業服務的報告。所有建議均由 ethereum.org 團隊審核。

### 收錄標準 {#criteria-for-inclusion}

#### 必備條件 {#must-haves}

- **實質的以太坊內容** - 報告必須實質討論以太坊技術、經濟學、安全性、治理或生態系統，或者涵蓋與以太坊相關的更廣泛政策、監管和金融環境。在更廣泛的加密貨幣報告中僅順帶提及以太坊是不夠的。
- **可靠的來源** - 優先考慮來自具有研究記錄來源的報告。這包括學術機構、研究公司、產業團體、基金會和成熟的開發組織。如果發布者知名度較低，請在您的 PR 描述中提供背景資訊，說明其可信度與專業知識。
- **公開存取** - 報告必須可以免費存取，且沒有付費牆。不接受需要註冊才能查看的 PDF 報告。
- **準確的資訊** - 技術和數據聲明應可驗證。包含明顯事實錯誤或純屬個人意見的報告將不會被列出。

#### 加分條件 {#nice-to-haves}

- **時效性** - 優先考慮較新的報告。除非具有歷史意義，否則不太可能新增超過 2 年的報告。
- **原創研究** - 包含原創數據、分析或模型的報告，其優先級高於現有公開資訊的摘要。

### 建議報告 {#suggest-a-report}

如果您想為 ethereum.org 建議報告且該報告符合標準，請在 GitHub 上建立一個 issue。

<ButtonLink href="https://github.com/ethereum/ethereum-org-website/issues/new?template=content-suggestion.yml">
  建議報告
</ButtonLink>

### 維護 {#maintenance}

報告連結會定期檢查。失效的連結或已撤回的出版物將被移除。如果您發現發布者已將報告移至新的 URL，請[建立一個 issue](https://github.com/ethereum/ethereum-org-website/issues/new?template=content-suggestion.yml) 或發送電子郵件至 [website@ethereum.org](mailto:website@ethereum.org)。
