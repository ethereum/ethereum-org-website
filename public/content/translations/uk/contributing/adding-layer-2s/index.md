# Додавання рішень 2-го рівня {#adding-layer-2}

Ми хочемо переконатися, що ми перерахуємо найкращі можливі ресурси, щоб користувачі могли безпечно та впевнено переміщатися по простору в Шар 2.

Будь-хто може вільно запропонувати додати Шар 2 на ethereum.org. Якщо ми пропустили якесь рішення 2-го рівня, **[запропонуйте його](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A%2Ccontent+%3Afountain_pen%3A&template=suggest_layer2.yaml)!**

Зараз ми підтримуємо L2 на таких сторінках:

- [Оптимістичні зведення](/developers/docs/scaling/optimistic-rollups/)
- [Зведення з нульовим розголошенням](/developers/docs/scaling/zk-rollups/)
- [Рішення 2-го рівня](/layer-2/)

Шар 2 — відносно нова та обнадійлива парадигма для Ethereum. Ми намагалися створити справедливі рамки для розгляду на ethereum.org, але критерії лістингу будуть змінюватися та розвиватися з часом.

## Система прийняття рішень {#decision-framework}

### Критерії для включення: обов'язкові вимоги {#criteria-for-inclusion-the-must-haves}

**Лістинг для L2BEAT**

- Щоб проєкт було розглянуто, він має бути в списку на [L2BEAT](https://l2beat.com). L2BEAT забезпечує надійну оцінку ризиків проєктів для Шар-2, на які ми спираємося для оцінки проектів L2. **Якщо проєкт не представлений на L2BEAT, ми не будемо його розміщувати як L2 на ethereum.org.**
- [Дізнайтеся, як додати свій проєкт L2 до L2BEAT](https://github.com/l2beat/l2beat/blob/master/CONTRIBUTING.md).

**Відкритий код**

- Ваш код має бути доступним, і ви повинні приймати поради від широкої спільноти.

**Категорія рішень 2-го рівня**

Наразі ми вважаємо відповідним це рішення для рівня Шар 2 (L2):

- Обнадійливий роллап
- Незначимий роллап

_Ми не вважаємо інші рішення для масштабування, які не використовують Ethereum для доступності даних або безпеки, рішеннями 2-го рівня._

**Ethereum для доступності даних**

- Доступність даних є важливою відмінністю між іншими рішеннями для масштабування та рішеннями 2-го рівня. Проєкт **повинен** використовувати основну мережу Ethereum для доступності даних, щоб його розглянули для додавання до списку.

**Мости**

- Як користувачі можуть приєднатись до рівня Шар 2?

**Дата запуску проєкту**

- Рішення 2-го рівня, що працює в основній мережі понад 6 місяців

- Нещодавні проєкти, що не пройшли тестування користувачами, рідше розміщуються в лістингу.

**Зовнішній аудит безпеки**

- Зовнішній, чи внутрішній аудит, та безпека вашого продукту повинна бути протестована. Це зменшує ризики для наших користувачів і показує нам, що ви серйозно ставитеся до безпеки.

**База постійних користувачів**

- Ми розглянемо такі показники як TVL history, статистику транзакцій, і чи використовується вони відомими компаніями чи проєктами

**Активна команда розробників**

- Ми не розміщуємо в Шарі 2 (L2), якщо немає дієвої команди проєкту.

**Провідник блоків**

- Для перерахованих проєктів потрібен робочий Провідник блоків, щоб користувачі могли легко переміщатися по ланцюжку.

### Інші критерії: додаткові переваги {#nice-to-haves}

**Обмінний фонд проєкту**

- Чи можуть користувачі депонувати і/та виводити напряму з біржі?

**Посилання на децентралізовані додатки в Шарі 2 (L2) екосистими**

- Ми прагнемо інформувати користувачів, як вони можуть скористатись Шаром 2 (L2). (наприклад, https://portal.arbitrum.io/, https://www.optimism.io/apps)

**Перелік контрактів токену**

- Оскільки активи матимуть нову адресу на Шарі 2, якщо є доступний ресурс переліку токенів, поділіться, будь ласка.

**Служба підтримки рідного гаманця**

- Чи можуть гаманці підтримати L2 природно?
