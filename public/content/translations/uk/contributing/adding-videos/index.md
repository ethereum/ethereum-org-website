# Додавання відео {#adding-videos}

У [галереї відео ethereum.org](/videos/) представлені відео про Етеріум та екосистему Етеріуму від авторів зі спільноти та надійних джерел. Будь-хто може запропонувати відео для додавання.

## Політика розміщення {#listing-policy}

Ethereum.org — це нейтральний освітній ресурс. Галерея відео модерується, щоб:

- **Навчати** користувачів технології Етеріуму, його екосистемі та спільноті
- **Залишатися точною** у своєму технічному контенті
- **Бути актуальною** для спільноти Етеріуму

Сайт не розміщує відео, які переважно рекламують певний продукт, токен або комерційну послугу.


## Критерії для включення {#criteria-for-inclusion}

### Обов'язкові вимоги {#must-haves}

- **Сфокусованість на Етеріумі** — відео має бути переважно про Етеріум, його технологію, екосистему або спільноту. Відео на загальні теми про блокчейн приймаються лише в тому випадку, якщо вони суттєво доповнюють або стосуються освітньої сторінки на сайті, або згадують Етеріум.
- **Освітня цінність** — відео має навчати глядачів чомусь про Етеріум або відзначати глобальну спільноту Етеріуму. Рекламний або маркетинговий контент не приймається.
- **Точна інформація** — технічний контент має бути фактично правильним та актуальним. Застарілі відео про функції, що більше не підтримуються, можуть бути видалені.
- **Якісне виробництво** — відео повинно мати достатньо чітку якість звуку та зображення.
- **Загальнодоступність** — відео має бути розміщене на відкритому ресурсі або доступній платформі, як-от YouTube, і бути у вільному доступі без платного доступу (пейволу) або вимоги реєстрації.

### Бажані вимоги {#nice-to-haves}

- **Наявність транскрипту** — відео з текстовою розшифровкою покращують доступність та SEO. Якщо у вас його немає, команда ethereum.org може допомогти його згенерувати.
- **З надійного джерела** — контент від визнаних викладачів, дослідників та джерел має пріоритет.
- **Своєчасність та довговічність** — контент, який залишається актуальним з часом, має перевагу над матеріалами, що швидко втрачають актуальність.


## Як додати відео {#how-to-add-a-video}

### Варіант 1: Відкрити issue {#open-an-issue}

Якщо ви хочете запропонувати відео, але не бажаєте створювати файли самостійно, відкрийте issue на GitHub з деталями відео, і контриб'ютор допоможе вам його додати.

<ButtonLink href="https://github.com/ethereum/ethereum-org-website/issues/new?template=suggest_video.yaml">
  Запропонувати відео
</ButtonLink>

### Варіант 2: Відкрити pull request {#open-a-pull-request}

Якщо ви хочете додати відео самостійно, виконайте такі кроки:

#### Крок 1: Створіть файл відео {#step-1}

Створіть новий каталог та файл `index.md` за адресою:

```
public/content/videos/{your-video-slug}/index.md
```

Слаг (slug) має бути безпечним для URL-адрес, написаним малими літерами та використовувати дефіси (наприклад, `blockchain-101-visual-demo`).

#### Крок 2: Додайте frontmatter {#step-2}

Додайте наступний YAML frontmatter до вашого `index.md`:

```yaml
---
title: "Your Video Title"
description: "A brief 1–3 sentence summary of the video."
lang: en
youtubeId: "dQw4w9WgXcQ"
uploadDate: 2025-01-15
duration: "12:30"
educationLevel: beginner
topic:
  - "your-topic"
  - "another-topic"
format: explainer
author: Channel Name
---
```

**Довідник полів:**

| Поле | Обов'язково | Опис |
|---|---|---|
| `title` | Так | Назва відео |
| `description` | Так | Короткий опис на 1–3 речення |
| `lang` | Так | Наразі завжди `en` |
| `youtubeId` | Так | ID відео на YouTube (з URL-адреси після `v=`) |
| `uploadDate` | Так | Дата оригінального завантаження у форматі `YYYY-MM-DD` |
| `duration` | Так | Тривалість відео у форматі `H:MM:SS` або `M:SS` |
| `educationLevel` | Так | `beginner`, `intermediate` або `advanced` |
| `topic` | Так | Масив тегів тем для фільтрації в галереї |
| `format` | Так | `explainer`, `presentation`, `interview`, `tutorial` або `panel` |
| `author` | Так | Ім'я автора або назва каналу |
| `breadcrumb` | Ні | Користувацька коротка мітка для навігації «хлібні крихти» |
| `customThumbnailUrl` | Ні | Користувацька URL-адреса мініатюри (за замовчуванням використовується мініатюра YouTube) |

#### Крок 3: Додайте транскрипт (рекомендовано) {#step-3}

Під frontmatter `---` додайте транскрипт відео у форматі markdown:

```markdown
---
title: "..."
# ... решта frontmatter
---

Короткий вступ до змісту відео.

### Назва розділу (0:00)

Текст транскрипту для цього розділу...

### Наступний розділ (5:30)

Більше тексту транскрипту...
```

Використовуйте заголовки `###` з часовими мітками, щоб позначити основні розділи. Це робить транскрипт зручним для швидкого читання та покращує SEO.

Якщо у вас немає транскрипту, ви можете залишити тіло файлу порожнім, і команда згенерує його.

#### Крок 4: Виберіть теги тем {#step-4}

Виберіть теги тем, які відповідають існуючим категоріям, що використовуються в галереї. Поточні категорії та їхні теги включають:

- **Як працює Етеріум**: `how-ethereum-works`, `consensus`, `blockchain`, `cryptography`, `accounts`, `ethereum`, `intro`, `transactions`, `pos`, `smart-contracts`
- **Оновлення мережі**: `network-upgrades`, `upgrades`, `pectra`, `dencun`, `eip-4844`, `blobs`, `fusaka`
- **Дорожня карта та пріоритети**: `roadmap-and-priorities`, `pbs`, `mev`
- **Масштабування та рівень 2 (l2)**: `scaling-and-layer-2`, `scaling`, `layer-2`, `rollups`, `optimistic-rollups`, `zk-rollups`
- **Варіанти використання**: `use-cases`, `defi`, `finance`, `nfts`, `erc-721`, `erc-1155`, `lending`, `dapps`, `restaking`, `eigenlayer`, `dao`, `identity`, `desci`, `refi`
- **Приватність та безпека**: `privacy-and-security`, `privacy`, `authentication`
- **Історії спільноти**: `community-stories`, `contributing`, `translations`, `community`

Щоб ваше відео з'явилося на полиці категорії в галереї, додайте принаймні один ключовий тег категорії (назва жирним шрифтом у форматі kebab-case, наприклад, `use-cases` або `scaling-and-layer-2`). Відео без розпізнаного тегу категорії відображатимуться лише в поданні «Усі» та в результатах пошуку.

Ви також можете використовувати нові теги — вони будуть доступні для майбутніх групувань за категоріями.

#### Крок 5: Надішліть свій PR {#step-5}

Відкрийте pull request зі своїми змінами до гілки `dev`. Команда розгляне вашу заявку та надасть зворотний зв'язок.


## Обслуговування {#maintenance}

Розміщені відео регулярно перевіряються, щоб переконатися, що вони:

- Досі відповідають критеріям розміщення
- Містять точну та актуальну інформацію
- Мають робочі посилання на хостинг/YouTube

Якщо ви помітили проблему з розміщеним відео, [створіть issue](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+✨,content+🖋️&template=suggest_video.yaml) або надішліть електронного листа на [website@ethereum.org](mailto:website@ethereum.org).
