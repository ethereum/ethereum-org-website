Це посібник початкового рівня з використання WebSockets та Alchemy для виконання запитів до блокчейну Етеріум.

## WebSockets проти HTTP {#websockets-vs-http}

На відміну від HTTP, з WebSockets вам не потрібно постійно робити запити, коли ви хочете отримати певну інформацію. WebSockets підтримують мережеве з'єднання для вас (якщо все зроблено правильно) і прослуховують зміни.

Як і з будь-яким мережевим з'єднанням, не варто вважати, що WebSocket залишатиметься відкритим вічно без переривань, але правильна обробка розірваних з'єднань і повторне підключення вручну може бути складним завданням. Ще одним недоліком WebSockets є те, що ви не отримуєте HTTP-коди стану у відповіді, а лише повідомлення про помилку.

​[Alchemy Web3](https://docs.alchemy.com/reference/api-overview) автоматично додає обробку збоїв WebSocket і повторні спроби без необхідності налаштування.

## Спробуйте {#try-it-out}

Найпростіший спосіб протестувати WebSockets — встановити інструмент командного рядка для виконання запитів WebSocket, наприклад [wscat](https://github.com/websockets/wscat). Використовуючи wscat, ви можете надсилати запити наступним чином:

_Примітка: якщо у вас є акаунт Alchemy, ви можете замінити `demo` на власний ключ API. [Зареєструйте безкоштовний акаунт Alchemy тут!](https://auth.alchemy.com/signup)_

```
wscat -c wss://eth-mainnet.ws.alchemyapi.io/ws/demo

>  {"jsonrpc":  "2.0", "id": 0, "method":  "eth_gasPrice"}

<  {"jsonrpc":  "2.0", "result":  "0xb2d05e00", "id": 0}
```

## Як використовувати WebSockets {#how-to-use-websockets}

Для початку відкрийте WebSocket, використовуючи URL-адресу WebSocket для вашого застосунку. Ви можете знайти URL-адресу WebSocket вашого застосунку, відкривши сторінку застосунку на [вашій панелі керування](https://dashboard.alchemy.com/) та натиснувши «View Key». Зверніть увагу, що URL-адреса вашого застосунку для WebSockets відрізняється від його URL-адреси для HTTP-запитів, але обидві можна знайти, натиснувши «View Key».

![Where to find your WebSocket URL in your Alchemy dashboard](./use-websockets.gif)

Будь-який з API, перелічених у [Довіднику з API Alchemy](https://www.alchemy.com/docs/reference/api-overview), можна використовувати через WebSocket. Для цього використовуйте те саме корисне навантаження, яке було б надіслано як тіло HTTP POST-запиту, але замість цього надішліть це корисне навантаження через WebSocket.

## З Web3 {#with-web3}

Перехід на WebSockets під час використання клієнтської бібліотеки, такої як Web3, є простим. Просто передайте URL-адресу WebSocket замість HTTP під час створення екземпляра вашого клієнта Web3. Наприклад:

```js
const web3 = new Web3("wss://eth-mainnet.ws.alchemyapi.io/ws/your-api-key")

web3.eth.getBlockNumber().then(console.log) // -> 7946893
```

## API підписок {#subscription-api}

Під час підключення через WebSocket ви можете використовувати два додаткові методи: `eth_subscribe` та `eth_unsubscribe`. Ці методи дозволять вам прослуховувати певні події та негайно отримувати сповіщення.

### `eth_subscribe` {#eth-subscribe}

Створює нову підписку на вказані події. [Дізнайтеся більше про `eth_subscribe`](https://docs.alchemy.com/reference/eth-subscribe).

#### Параметри {#parameters}

1. Типи підписок
2. Необов'язкові параметри

Перший аргумент визначає тип події для прослуховування. Другий аргумент містить додаткові параметри, які залежать від першого аргументу. Різні типи підписок, їхні параметри та корисні навантаження подій описані нижче.

#### Повертає {#returns}

ID підписки: Цей ідентифікатор буде додано до будь-яких отриманих подій, і його також можна використовувати для скасування підписки за допомогою `eth_unsubscribe`.

#### Події підписки {#subscription-events}

Поки підписка активна, ви отримуватимете події, які є об'єктами з такими полями:

- `jsonrpc`: Завжди "2.0"
- `method`: Завжди "eth_subscription"
- `params`: Об'єкт із такими полями:
  - `subscription`: ID підписки, повернутий викликом `eth_subscribe`, який створив цю підписку.
  - `result`: Об'єкт, вміст якого змінюється залежно від типу підписки.

#### Типи підписок {#subscription-types}

1. `alchemy_newFullPendingTransactions`

3. `logs`

Видає журнали, які є частиною щойно доданих блоків, що відповідають заданим критеріям фільтрації.

Коли відбувається реорганізація ланцюга, журнали, які є частиною блоків у старому ланцюзі, будуть видані знову з властивістю `removed`, встановленою на `true`. Крім того, видаються журнали, які є частиною блоків у новому ланцюзі, що означає, що у випадку реорганізації можна побачити журнали для однієї й тієї ж транзакції кілька разів.

Параметри

1. Об'єкт із такими полями:
   - `address` (необов'язково): рядок, що представляє адресу, або масив таких рядків.
     - Будуть видаватися лише журнали, створені з однієї з цих адрес.
   - `topics`: масив специфікаторів тем.
     - Кожен специфікатор теми — це або `null`, рядок, що представляє тему, або масив рядків.
     - Кожна позиція в масиві, яка не є `null`, обмежує видані журнали лише тими, які мають одну із заданих тем на цій позиції.

Деякі приклади специфікацій тем:

- `[]`: Дозволені будь-які теми.
- `[A]`: A на першій позиції (і будь-що після).
- `[null, B]`: Будь-що на першій позиції та B на другій позиції (і будь-що після).
- `[A, B]`: A на першій позиції та B на другій позиції (і будь-що після).
- `[[A, B], [A, B]]`: (A або B) на першій позиції та (A або B) на другій позиції (і будь-що після).

### `eth_unsubscribe` {#eth-unsubscribe}

Скасовує існуючу підписку, щоб подальші події не надсилалися.

Параметри

1. ID підписки, який раніше був повернутий викликом `eth_subscribe`.

Повертає

`true`, якщо підписку було успішно скасовано, або `false`, якщо підписки із заданим ідентифікатором не існувало.

Приклад:

**Запит**

```
curl https://eth-mainnet.alchemyapi.io/v2/your-api-key
-X POST
-H "Content-Type: application/json"
-d '{"id": 1, "method": "eth_unsubscribe", "params": ["0x9cef478923ff08bf67fde6c64013158d"]}'
```

**Результат**

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": true
}
```
