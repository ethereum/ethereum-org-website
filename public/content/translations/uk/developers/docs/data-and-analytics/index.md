## Вступ {#introduction}

Оскільки використання мережі продовжує зростати, в ончейн-даних з'являтиметься все більше цінної інформації. Зі стрімким збільшенням обсягу даних обчислення та агрегація цієї інформації для створення звітів або керування децентралізованим застосунком (dapp) може стати процесом, що вимагає багато часу та ресурсів.

Використання наявних постачальників даних може прискорити розробку, забезпечити точніші результати та зменшити зусилля на постійне обслуговування. Це дозволить команді зосередитися на основній функціональності, яку намагається надати їхній проєкт.

## Передумови {#prerequisites}

Ви повинні розуміти базову концепцію [оглядачів блоків](/developers/docs/data-and-analytics/block-explorers/), щоб краще зрозуміти їх використання в контексті аналітики даних. Крім того, ознайомтеся з концепцією [індексу](/glossary/#index), щоб зрозуміти переваги, які вони додають до архітектури системи.

Щодо архітектурних основ, варто розуміти, що таке [API](https://www.wikipedia.org/wiki/API) та [REST](https://www.wikipedia.org/wiki/Representational_state_transfer), хоча б у теорії.

## Оглядачі блоків {#block-explorers}

Багато [оглядачів блоків](/developers/docs/data-and-analytics/block-explorers/) пропонують шлюзи [RESTful](https://www.wikipedia.org/wiki/Representational_state_transfer) [API](https://www.wikipedia.org/wiki/API), які забезпечують розробникам доступ до даних у реальному часі про блоки, транзакції, валідаторів, акаунти та іншу ончейн-активність.

Потім розробники можуть обробляти та перетворювати ці дані, щоб надати своїм користувачам унікальну інформацію та можливості взаємодії з [блокчейном](/glossary/#blockchain). Наприклад, [Etherscan](https://etherscan.io) та [Blockscout](https://eth.blockscout.com) надають дані виконання та консенсусу для кожного 12-секундного слота.

## The Graph {#the-graph}

[The Graph](https://thegraph.com/) — це протокол індексування, який забезпечує простий спосіб запиту даних блокчейну через відкриті API, відомі як підграфи.

З The Graph розробники можуть скористатися такими перевагами:

- Децентралізоване індексування: дозволяє індексувати дані блокчейну через кількох індексаторів, усуваючи таким чином будь-яку єдину точку відмови.
- Запити GraphQL: надає потужний інтерфейс GraphQL для запиту індексованих даних, що робить отримання даних надзвичайно простим.
- Налаштування: визначайте власну логіку для перетворення та зберігання даних блокчейну, а також повторно використовуйте підграфи, опубліковані іншими розробниками в мережі The Graph.

Дотримуйтесь цього [короткого посібника](https://thegraph.com/docs/en/quick-start/), щоб створити, розгорнути та зробити запит до підграфа за 5 хвилин.

## Різноманітність клієнтів {#client-diversity}

[Різноманітність клієнтів](/developers/docs/nodes-and-clients/client-diversity/) є важливою для загального стану мережі Етеріум, оскільки вона забезпечує стійкість до помилок та експлойтів. Зараз існує кілька інформаційних панелей різноманітності клієнтів, зокрема [clientdiversity.org](https://clientdiversity.org/), [rated.network](https://www.rated.network), [supermajority.info](https://supermajority.info//) та [Ethernodes](https://ethernodes.org/).

## Dune Analytics {#dune-analytics}

[Dune Analytics](https://dune.com/) попередньо обробляє дані блокчейну в таблиці реляційної бази даних (DuneSQL), дозволяє користувачам робити запити до даних блокчейну за допомогою SQL і створювати інформаційні панелі на основі результатів запитів. Ончейн-дані організовані в 4 необроблені таблиці: `blocks`, `transactions`, (події) `logs` та (виклики) `traces`. Популярні контракти та протоколи були декодовані, і кожен з них має власний набір таблиць подій і викликів. Ці таблиці подій і викликів обробляються далі та організовуються в абстрактні таблиці за типом протоколів, наприклад, dex, кредитування, стейблкоїни тощо.

## SQD {#sqd}

[SQD](https://sqd.dev/) — це децентралізована гіпермасштабована платформа даних, оптимізована для забезпечення ефективного бездозвільного доступу до великих обсягів даних. Наразі вона обслуговує історичні ончейн-дані, включаючи журнали подій, квитанції транзакцій, трасування та різниці станів для кожної транзакції. SQD пропонує потужний набір інструментів для створення користувацьких конвеєрів вилучення та обробки даних, досягаючи швидкості індексування до 150 тисяч блоків на секунду.

Щоб розпочати, відвідайте [документацію](https://docs.sqd.dev/) або перегляньте [приклади EVM](https://github.com/subsquid-labs/squid-evm-examples) того, що ви можете створити за допомогою SQD.

## Мережа SubQuery {#subquery-network}

[SubQuery](https://subquery.network/) — це провідний індексатор даних, який надає розробникам швидкі, надійні, децентралізовані та налаштовувані API для їхніх Web3-проєктів. SubQuery розширює можливості розробників із понад 165 екосистем (включаючи Етеріум) за допомогою багатих індексованих даних для створення інтуїтивно зрозумілого та захопливого досвіду для своїх користувачів. Мережа SubQuery забезпечує роботу ваших безперебійних застосунків за допомогою стійкої та децентралізованої інфраструктурної мережі. Використовуйте набір інструментів розробника блокчейну SubQuery для створення Web3-застосунків майбутнього, не витрачаючи час на створення власного бекенду для обробки даних.

Щоб розпочати, відвідайте [короткий посібник з Етеріуму](https://academy.subquery.network/quickstart/quickstart_chains/ethereum-gravatar.html), щоб почати індексувати дані блокчейну Етеріум за лічені хвилини в локальному середовищі Docker для тестування перед запуском у [керованому сервісі SubQuery](https://managedservice.subquery.network/) або в [децентралізованій мережі SubQuery](https://app.subquery.network/dashboard).

## Codex {#codex}

[Codex](https://www.codex.io/) — це API даних блокчейну в реальному часі, що надає збагачені дані для понад 70 мільйонів токенів у понад 80 мережах. Розробники можуть отримати доступ до структурованих цін на токени, балансів гаманців, історії транзакцій та агрегованої аналітики (обсяг, ліквідність, унікальні гаманці) без підтримки власної інфраструктури індексування. Codex підтримує доставку даних менш ніж за секунду через інтеграції WebSocket та вебхуків.

Щоб розпочати, відвідайте [документацію](https://docs.codex.io), спробуйте [Оглядач](https://docs.codex.io/explore) або зареєструйтеся на [інформаційній панелі](https://dashboard.codex.io/signup).

## EVM Query Language {#evm-query-language}

EVM Query Language (EQL) — це SQL-подібна мова, розроблена для запитів до ланцюгів EVM (Віртуальної машини Етеріуму). Кінцева мета EQL — підтримка складних реляційних запитів до першокласних об'єктів ланцюга EVM (блоків, акаунтів та транзакцій), одночасно надаючи розробникам і дослідникам ергономічний синтаксис для щоденного використання. За допомогою EQL розробники можуть отримувати дані блокчейну, використовуючи знайомий SQL-подібний синтаксис, і усунути потребу у складному шаблонному коді. EQL підтримує стандартні запити даних блокчейну (наприклад, отримання нонсу та балансу акаунта в Етеріумі або отримання поточного розміру блоку та часової мітки) і постійно додає підтримку складніших запитів та наборів функцій.
