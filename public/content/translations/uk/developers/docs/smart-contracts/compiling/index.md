Вам потрібно створити ваш контракт так, щоб ваш веб-додаток і віртуальна машина Ethereum (EVM) могли його зрозуміти.

## Передумови {#prerequisites}

Перед тим як читати про компіляцію, вам може бути корисно ознайомитися з нашим вступом до [смарт-контрактів](/developers/docs/smart-contracts/) і [віртуальної машини Ethereum](/developers/docs/evm/).

## EVM {#the-evm}

```solidity
pragma solidity 0.4.24;

contract Greeter {

    function greet() public view returns (string memory) {
        return "Hello";
    }

}
```

## Вебзастосунки {#web-applications}

Компілятор також створює **двійковий інтерфейс застосунку (Application Binary Interface, ABI)**, який потрібен вашому застосунку для взаємодії з контрактом і виклику його функцій.

ABI- це файл JSON, який описує розгорнутий контракт і його функції смарт-контракту. Це допомагає подолати розрив між web2 та web3

[Клієнтська бібліотека JavaScript](/developers/docs/apis/javascript/) зчитує **ABI**, щоб ви могли викликати свій смарт-контракт в інтерфейсі вашого вебзастосунку.
