Кожен валідатор має **облікові дані для виведення**, які визначають, як і куди можна вивести їхній застейканий ETH та винагороди. Тип облікових даних позначається першим байтом: `0x00`, `0x01` або `0x02`. Розуміння цих типів є важливим для валідаторів, які керують своїм стейком.

## 0x00: Облікові дані до оновлення Шапелла {#0x00-credentials}

Тип `0x00` — це початковий формат облікових даних для виведення, що існував до оновлення Шапелла (квітень 2023 року). Валідатори з цим типом облікових даних не мають встановленої адреси для виведення на рівні виконання, що означає, що їхні кошти залишаються заблокованими на рівні консенсусу. Якщо ви все ще маєте облікові дані `0x00`, ви повинні оновити їх до `0x01` або `0x02`, перш ніж зможете отримувати будь-які виведення.

## 0x01: Застарілі облікові дані для виведення {#0x01-credentials}

Тип `0x01` був представлений з оновленням Шапелла і став стандартом для валідаторів, які хотіли встановити адресу для виведення на рівні виконання. З обліковими даними `0x01`:

- Будь-який баланс понад 32 ETH **автоматично переказується** на вашу адресу для виведення
- Повні виходи проходять через стандартну чергу на вихід
- Винагороди понад 32 ETH не можуть капіталізуватися (накопичуватися) — вони періодично виводяться

**Чому деякі валідатори все ще використовують 0x01:** Це простіше і звичніше. Багато валідаторів внесли депозити після оновлення Шапелла і вже мають цей тип, і він чудово працює для тих, хто хоче автоматичного виведення надлишкового балансу.

**Чому це не рекомендується:** З `0x01` ви втрачаєте можливість капіталізувати винагороди понад 32 ETH. Кожна частка надлишку автоматично виводиться, що обмежує потенціал заробітку вашого валідатора і вимагає окремого управління виведеними коштами.

## 0x02: Облікові дані для виведення з капіталізацією {#0x02-credentials}

Тип `0x02` був представлений з оновленням Пектра і є **рекомендованим вибором** для валідаторів сьогодні. Валідаторів з обліковими даними `0x02` іноді називають «валідаторами з капіталізацією».

З обліковими даними `0x02`:

- Винагороди понад 32 ETH **капіталізуються** з кроком в 1 ETH до максимального ефективного балансу у 2048 ETH
- Часткові виведення потрібно запитувати вручну (автоматичні перекази відбуваються лише після перевищення порогу у 2048 ETH)
- Валідатори можуть консолідувати кілька валідаторів по 32 ETH в одного валідатора з вищим балансом
- Повні виходи все ще підтримуються через стандартну чергу на вихід

Як часткові виведення, так і консолідації можна виконати через [Дії валідатора на Launchpad](https://launchpad.ethereum.org/en/validator-actions).

**Чому валідаторам варто віддати перевагу 0x02:** Це забезпечує кращу ефективність капіталу завдяки капіталізації, більший контроль над тим, коли відбуваються виведення, і підтримує консолідацію валідаторів. Для соло-стейкерів, які з часом накопичують винагороди, це означає, що їхній ефективний баланс — а отже, і їхні винагороди — можуть зростати понад 32 ETH без ручного втручання.

**Важливо:** Після того, як ви конвертуєте `0x01` у `0x02`, ви не зможете скасувати цю дію.

Для отримання детального посібника з конвертації в облікові дані типу 2 та функції MaxEB дивіться [сторінку з поясненням MaxEB](/roadmap/pectra/maxeb/).

## Що мені вибрати? {#what-should-i-pick}

- **Нові валідатори:** Вибирайте `0x02`. Це сучасний стандарт із кращою капіталізацією та гнучкістю.
- **Існуючі валідатори 0x01:** Розгляньте можливість конвертації в `0x02`, якщо ви хочете, щоб винагороди капіталізувалися понад 32 ETH, або плануєте консолідувати валідаторів.
- **Існуючі валідатори 0x00:** Оновіть негайно — ви не зможете вивести кошти без оновлення ваших облікових даних. Спочатку ви повинні конвертувати їх у `0x01`, а потім зможете конвертувати у `0x02`.

## Інструменти для управління обліковими даними для виведення {#withdrawal-credential-tools}

Кілька інструментів підтримують вибір або конвертацію між типами облікових даних:

- **[Ethereum Staking Launchpad](https://launchpad.ethereum.org/en/validator-actions)** — офіційний інструмент для депозитів та управління валідаторами, включаючи конвертацію облікових даних та консолідацію.
- **[Pectra Staking Manager](https://pectrastaking.com)** — веб-інтерфейс із підтримкою підключення гаманця для конвертацій та консолідації.
- **[Pectra Validator Ops CLI Tool](https://github.com/Luganodes/Pectra-Batch-Contract)** — інструмент командного рядка для пакетних конвертацій.
- **[Ethereal](https://github.com/wealdtech/ethereal)** — інструмент командного рядка для операцій в Етеріумі, включаючи управління валідаторами.

Повний список інструментів для консолідації та детальні інструкції з конвертації дивіться в розділі [Інструменти консолідації MaxEB](/roadmap/pectra/maxeb/#consolidation-tooling).
