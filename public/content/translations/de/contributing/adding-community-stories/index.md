## Wie man eine Geschichte beisteuert {#how-to-contribute}

Community-Geschichten erscheinen unter [ethereum.org/stories/](/stories/) und heben persönliche Erfahrungen mit Open-Source- und Sanctuary-Technologien hervor, wie das Ethereum-Ökosystem das Leben der Menschen beeinflusst hat und wie Communitys auf der ganzen Welt Ethereum nutzen. Jeder kann eine Geschichte vorschlagen oder einreichen.

Es gibt drei Möglichkeiten, eine Geschichte beizusteuern:

1. **Hervorgehobene Geschichten** - Ausführliche, geschriebene Geschichten, die als eigene Seite auf der Website veröffentlicht werden.
2. **Kurzgeschichten** - Kurze Erfahrungsberichte, die als Karten auf der Startseite für Geschichten angezeigt werden.
3. **Videogeschichten** - Videos, die über die [Videogalerie](/videos/) eingereicht und auf der Geschichten-Seite präsentiert werden.

### Richtlinien für die Aufnahme {#listing-policy}

Ethereum.org ist eine neutrale, lehrreiche Ressource. Die Geschichten-Seite wird kuratiert, um:

- Den Zweck und die Technologie von Ethereum durch echte persönliche und Community-Erfahrungen **menschlicher zu machen**
- **Hervorzuheben**, wie Open-Source- und Sanctuary-Technologien einen echten Unterschied im Leben der Menschen machen
- Die Vielfalt der globalen Ethereum-Community zu **feiern**

Die Website listet keine Geschichten auf, die in erster Linie ein bestimmtes Produkt, einen bestimmten Token oder eine kommerzielle Dienstleistung bewerben. Alle Einreichungen werden vom ethereum.org-Team geprüft.

### Kriterien für die Aufnahme {#criteria-for-inclusion}

#### Zwingende Voraussetzungen {#must-haves}

- **Echte menschliche Geschichte** - Die Geschichte muss eine echte persönliche oder gemeinschaftliche Erfahrung beschreiben. Dies kann Ihre Erfahrungen mit Open-Source- und Sanctuary-Technologien umfassen, wie das Ethereum-Ökosystem Ihr Leben beeinflusst hat oder wie Sie und Ihre Community Ethereum nutzen. Fiktive, hypothetische oder rein technische Inhalte gehören in andere Bereiche (z. B. `/latest/` für technische Artikel aus der Ersteller-Landschaft).
- **Ethereum-Bezug** - Die Geschichte muss Ethereum, Ethereum-basierte Anwendungen (einschließlich L2s, Stablecoins auf Ethereum, dezentrale Anwendungen (Dapps) usw.) oder das breitere Open-Source- und Sanctuary-Tech-Ökosystem auf sinnvolle Weise einbeziehen. Allgemeine Geschichten über Krypto und Blockchains ohne spezifischen Ethereum-Fokus werden nicht akzeptiert.
- **Nicht werblich** - Die Geschichte sollte nicht in erster Linie ein bestimmtes Produkt, einen bestimmten Token, ein Unternehmen oder eine kommerzielle Dienstleistung bewerben. Die Erwähnung verwendeter Tools ist akzeptabel; die Geschichte als Produktempfehlung zu formulieren, ist es nicht.
- **Angemessene Inhalte** - Geschichten müssen dem [Verhaltenskodex](/community/code-of-conduct/) von ethereum.org entsprechen. Inhalte, die hasserfüllt, irreführend oder illegal sind, werden nicht akzeptiert.
- **Englisch zuerst** - Geschichten können in jeder [unterstützten Sprache](/contributing/translation-program/) eingereicht werden. Setzen Sie das Feld `lang` auf die Sprache, in der die Geschichte geschrieben ist (z. B. `en` für Englisch, `es` für Spanisch). Englische Einreichungen werden in andere Sprachen übersetzt, und nicht-englische Einreichungen können über das [Übersetzungsprogramm](/contributing/translation-program/) ins Englische übersetzt werden.

#### Wünschenswert {#nice-to-haves}

- **Geografische und thematische Vielfalt** - Geschichten aus unterrepräsentierten Regionen und über unterrepräsentierte Anwendungsfälle werden bevorzugt behandelt.
- **Multimedia** - Geschichten, die von Video, Audio oder Fotografie begleitet werden, sind erwünscht.
- **Open-Source- und Sanctuary-Werte** - Geschichten, die veranschaulichen, wie Open-Source- oder Sanctuary-Technologien Zensurresistenz, offenen Zugang, finanzielle Inklusion, Privatsphäre oder Sicherheit ermöglichen, werden besonders geschätzt.

### Wie man eine hervorgehobene Geschichte hinzufügt {#how-to-add-featured-story}

#### Option 1: Über das Einreichungsformular einreichen {#submit-via-intake-form}

Wenn Sie eine Geschichte teilen möchten, können Sie diese über das Einreichungsformular einreichen. Das ethereum.org-Team wird Ihre Einreichung prüfen und sich bei Ihnen melden.

<ButtonLink href="https://ethereumstory.paperform.co/">
  Reichen Sie Ihre Geschichte ein
</ButtonLink>

#### Option 2: Ein Issue auf GitHub eröffnen {#open-an-issue}

Wenn Sie eine Community-Geschichte für ethereum.org vorschlagen möchten und diese die Kriterien erfüllt, erstellen Sie ein Issue auf GitHub.

<ButtonLink href="https://github.com/ethereum/ethereum-org-website/issues/new?template=content-suggestion.yml">
  Ein Issue erstellen
</ButtonLink>

### Weitere Möglichkeiten, eine Geschichte beizusteuern {#other-ways-to-contribute}

Sie können auch eine **Kurzgeschichte** oder eine **Videogeschichte** vorschlagen:

- **Kurzgeschichten** erscheinen als Karten auf der [Startseite für Geschichten](/stories/). Diese können direkt auf ethereum.org gepostet werden oder auf Geschichten verlinken, die auf Twitter geteilt wurden.
- **Videogeschichten** sind Videos, die in der [Videogalerie](/videos/) mit der Kategorie `community-stories` versehen sind. Um eines hinzuzufügen, folgen Sie dem bestehenden Leitfaden zum [Hinzufügen von Videos](/contributing/adding-videos/) und fügen Sie `community-stories` in die Themen-Tags ein.

Wenn Sie eine der beiden Arten vorschlagen möchten, erstellen Sie ein Issue auf GitHub.

<ButtonLink href="https://github.com/ethereum/ethereum-org-website/issues/new?template=BLANK_ISSUE">
  Ein Issue erstellen
</ButtonLink>

### Wartung {#maintenance}

- Hervorgehobene Geschichten können aktualisiert werden, wenn sich sachliche Details ändern.
- Kurzgeschichten werden auf Angemessenheit und Richtigkeit überprüft.
- Die Geschichten-Seite übernimmt automatisch neue Videoeinträge, die mit `community-stories` getaggt sind.
