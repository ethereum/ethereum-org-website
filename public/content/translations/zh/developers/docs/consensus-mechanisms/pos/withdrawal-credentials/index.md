每个验证者都有一个**提款凭证**，它决定了他们质押的 ETH 和奖励可以如何以及在何处被提款。凭证类型由第一个字节指示：`0x00`、`0x01` 或 `0x02`。了解这些类型对于管理其质押的验证者来说非常重要。

## 0x00：沙佩拉升级前的凭证 {#0x00-credentials}

`0x00` 类型是沙佩拉升级（2023 年 4 月）之前的原始提款凭证格式。拥有此凭证类型的验证者没有设置执行层提款地址，这意味着他们的资金仍然锁定在共识层上。如果你仍然拥有 `0x00` 凭证，你必须升级到 `0x01` 或 `0x02` 才能接收任何提款。

## 0x01：旧版提款凭证 {#0x01-credentials}

`0x01` 类型是在沙佩拉升级中引入的，并成为想要设置执行层提款地址的验证者的标准。使用 `0x01` 凭证：

- 任何超过 32 ETH 的余额都会被**自动清扫**到你的提款地址
- 完全退出需要通过标准退出队列
- 超过 32 ETH 的奖励无法复利——它们会被定期清扫出去

**为什么有些验证者仍在使用 0x01：** 它更简单且为人熟知。许多验证者在沙佩拉升级之后进行质押，并且已经拥有这种类型，对于那些希望自动提取多余余额的人来说，它运作良好。

**为什么不推荐使用：** 使用 `0x01`，你将失去对超过 32 ETH 的奖励进行复利的能力。每一丁点多余的余额都会被自动清扫，这限制了你的验证者的盈利潜力，并需要单独管理已提取的资金。

## 0x02：复利提款凭证 {#0x02-credentials}

`0x02` 类型是在佩克特拉升级中引入的，是当今验证者的**推荐选择**。拥有 `0x02` 凭证的验证者有时被称为“复利验证者”。

使用 `0x02` 凭证：

- 超过 32 ETH 的奖励以 1 ETH 为增量进行**复利**，最高有效余额可达 2048 ETH
- 部分提款必须手动请求（自动清扫仅在超过 2048 ETH 阈值时发生）
- 验证者可以将多个 32 ETH 的验证者合并为一个更高余额的验证者
- 仍然支持通过标准退出队列进行完全退出

部分提款和合并都可以通过 [Launchpad 验证者操作](https://launchpad.ethereum.org/en/validator-actions)来执行。

**为什么验证者应该首选 0x02：** 它通过复利提供了更好的资金效率，对提款发生的时间有更多的控制权，并支持验证者合并。对于随着时间推移积累奖励的独立质押者来说，这意味着他们的有效余额——以及他们的奖励——可以增长到超过 32 ETH，而无需人工干预。

**重要提示：** 一旦你从 `0x01` 转换为 `0x02`，就无法回退。

有关转换为类型 2 凭证和 MaxEB 功能的详细指南，请参阅 [MaxEB 解释页面](/roadmap/pectra/maxeb/)。

## 我应该选择什么？ {#what-should-i-pick}

- **新验证者：** 选择 `0x02`。它是现代标准，具有更好的复利和灵活性。
- **现有的 0x01 验证者：** 如果你希望超过 32 ETH 的奖励进行复利，或者计划合并验证者，请考虑转换为 `0x02`。
- **现有的 0x00 验证者：** 立即升级——如果不更新你的凭证，你将无法提款。你必须首先转换为 `0x01`，然后才能转换为 `0x02`。

## 管理提款凭证的工具 {#withdrawal-credential-tools}

有几种工具支持选择或在凭证类型之间进行转换：

- **[以太坊质押 Launchpad](https://launchpad.ethereum.org/en/validator-actions)** - 用于存款和验证者管理的官方工具，包括凭证转换和合并
- **[佩克特拉质押管理器 (Pectra Staking Manager)](https://pectrastaking.com)** - 支持连接钱包进行转换和合并的 Web 用户界面
- **[佩克特拉验证者操作 CLI 工具 (Pectra Validator Ops CLI Tool)](https://github.com/Luganodes/Pectra-Batch-Contract)** - 用于批量转换的命令行工具
- **[Ethereal](https://github.com/wealdtech/ethereal)** - 用于以太坊操作（包括验证者管理）的 CLI 工具

有关合并工具的完整列表和详细的转换说明，请参阅 [MaxEB 合并工具](/roadmap/pectra/maxeb/#consolidation-tooling)。
