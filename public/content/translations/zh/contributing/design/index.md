设计是任何项目的重要组成部分，通过将你的时间和设计技能投入到 ethereum.org，你可以帮助改善我们访客的用户体验。为开源项目做贡献提供了一个在协作环境中获得相关经验和发展技能的机会。你将有机会与其他设计师、开发者和社区成员合作，他们每个人都有自己独特的视角和见解。

最终，这是建立一个多样化且令人印象深刻的作品集以展示你的设计技能的绝佳方式。

## 如何贡献？ {#how-to-contribute}

### <Emoji text=":one:" size={1} /> &nbsp;对早期设计原型提供反馈 {#design-critique}

我们有时需要帮助来测试我们的初步想法。这是一种无需任何技术知识即可做出贡献的绝佳方式。

1. 设计团队将在 [Discord](https://discord.com/invite/ethereum-org) 和 [GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) 上分享设计模型。
2. 你将在引导下浏览设计，并通过评论功能提供反馈。
3. 结果将分享在 GitHub 议题 (issue) 中，然后由团队关闭。

### <Emoji text=":two:" size={1} /> &nbsp;参与调查研究 {#answer-surveys}

通过以下方式对我们的网站提供反馈：

1. 访问 ethereum.org 并阅读页面。
2. 点击右下角的反馈小部件，回答与设计和内容相关的问题。
3. 重点关注自由格式的问题。

### <Emoji text=":three:" size={1} /> &nbsp;在网站上寻找与设计相关的议题并报告 {#report-design-issues}

ethereum.org 是一个快速发展的网站，拥有许多功能和内容。一些用户界面 (UI) 很容易过时或有待改进。如果你遇到任何此类情况，请报告它，以便引起我们的注意。

1. 浏览网站并关注其设计。
2. 如果你看到任何视觉或用户体验 (UX) 问题，请截图并做笔记。
3. 使用[错误报告](https://github.com/ethereum/ethereum-org-website/issues/new/choose)来报告发现的问题。

### <Emoji text=":four:" size={1} /> &nbsp;提出设计变更 {#propose-design-changes}

如果你乐于接受设计挑战，可以访问我们的 GitHub 议题看板并筛选[与设计相关的议题](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8)。

1. 浏览我们的网站并关注其设计，或者前往我们的 GitHub 仓库并查看带有 [“Design required”（需要设计）标签](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8)的议题。
2. 构思解决方案并进行设计。（最好使用我们的[设计系统](https://www.figma.com/community/file/1134414495420383395)）。
3. 在相应的 GitHub 议题中提出解决方案，或者[创建一个新议题。](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. 等待设计团队审核。

### <Emoji text=":five:" size={1} /> &nbsp;共同构建设计系统 {#contribute-to-design-system}

我们的设计系统让设计 ethereum.org 变得有趣且简单。如果你是一位经验丰富的设计师，你可以帮助我们为网站准备许多组件。

1. 从 GitHub 上的[设计系统看板](https://github.com/ethereum/ethereum-org-website/labels/design%20system)中选择一个要处理的议题，或者创建一个新议题。
2. 请求将选定的议题分配给你。
3. 开始在 Figma 中设计所需的组件。
4. 当你需要审核或指导时，在 GitHub 上与设计团队分享。
5. 设计团队将进行审核。
6. 设计团队会将更改合并到主文件中，并将该文件发布给社区。

### <Emoji text=":six:" size={1} /> &nbsp;在网站上编写与设计相关的内容 {#write-design-articles}

以太坊开发者社区非常强大，但设计社区略显落后。如果你是一位具备 Web3 知识的设计师，请考虑与更广泛的社区分享你的经验，以便我们共同成长和进步；我们有一个[关于以太坊设计的页面](/developers/docs/design-and-ux/)，你可以为其做出贡献。你也可以查看我们的[上架政策](/contributing/design/adding-design-resources)。

1. 构思 ethereum.org 上应该涵盖的、对该领域设计师有益的设计主题。
2. 前往我们的 GitHub 仓库并[提出一个议题](https://github.com/ethereum/ethereum-org-website/issues/new)来建议一个主题（先不要编写内容）。
3. 等待设计团队批准。
4. 获得批准后，编写内容。
5. 在相应的 GitHub 议题中提交它。

### <Emoji text=":seven:" size={1} /> &nbsp;绘制新插图 {#prepare-illustrations}

可视化是解释抽象主题最强大的工具之一。添加图表和信息图表具有巨大的潜力。毕竟，一图胜千言。

1. 访问我们的网站，看看哪些页面可以添加一些新的信息图表。
2. 确保插图风格与我们的[资产](/assets/)相符。
3. 前往我们的 GitHub 仓库并[提出一个议题](https://github.com/ethereum/ethereum-org-website/issues/new)来建议该插图。
4. 设计团队将对其进行审核。
5. 我们会创建一个新议题，要求开发者实现新图像。
