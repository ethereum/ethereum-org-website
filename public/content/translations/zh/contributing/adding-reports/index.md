## 添加报告 {#adding-a-report}

报告显示在 [ethereum.org/reports/](/reports/)，包含来自各种来源的与以太坊相关的研究、分析和出版物。任何人都可以推荐要添加的报告。

### 收录政策 {#listing-policy}

Ethereum.org 是一个中立的教育资源。报告页面的策划旨在：

- <strong>告知</strong>读者有关以太坊技术、经济学、安全性、治理以及更广泛的政策和监管环境的信息
- 在数据和分析方面**保持准确**
- <strong>保持</strong>与以太坊社区和生态系统的**相关性**

本网站不收录主要推广特定产品、代币或商业服务的报告。所有建议均由 ethereum.org 团队审核。

### 收录标准 {#criteria-for-inclusion}

#### 必备条件 {#must-haves}

- **实质性的以太坊内容** - 报告必须实质性地讨论以太坊技术、经济学、安全性、治理或生态系统，或者涵盖与以太坊相关的更广泛的政策、监管和金融环境。在更广泛的加密货币报告中顺便提及以太坊是不够的。
- **可靠的来源** - 首选来自在研究方面有良好记录的来源的报告。这包括学术机构、研究公司、行业团体、基金会和成熟的开发组织。如果发布者不太知名，请在你的 PR 描述中包含背景信息，解释其可信度和专业知识。
- **可公开访问** - 报告必须可以免费访问，没有付费墙。不接受需要注册才能查看的 PDF 报告。
- **准确的信息** - 技术和数据声明应该是可验证的。包含明显事实错误或主观意见的报告将不予收录。

#### 加分项 {#nice-to-haves}

- **时效性** - 首选较新的报告。除非具有历史意义，否则不太可能添加超过 2 年的报告。
- **原创研究** - 包含原创数据、分析或建模的报告优先于对现有公开信息的总结。

### 推荐报告 {#suggest-a-report}

如果你想为 ethereum.org 推荐报告并且该报告符合标准，请在 GitHub 上创建一个议题 (issue)。

<ButtonLink href="https://github.com/ethereum/ethereum-org-website/issues/new?template=content-suggestion.yml">
  推荐报告
</ButtonLink>

### 维护 {#maintenance}

报告链接会定期检查。失效链接或已撤回的出版物将被移除。如果你发现发布者已将报告移至新的 URL，请[创建议题](https://github.com/ethereum/ethereum-org-website/issues/new?template=content-suggestion.yml)或发送电子邮件至 [website@ethereum.org](mailto:website@ethereum.org)。
