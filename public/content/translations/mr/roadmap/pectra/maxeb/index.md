*थोडक्यात (tl;dr):* पेक्ट्रा हार्ड फोर्क इथेरियम प्रमाणकांना **Type 1** वरून **Type 2** पैसे काढण्याची अधिकारपत्रे (withdrawal credentials) मध्ये रूपांतरित करून उच्च कमाल प्रभावी शिल्लक आणि चक्रवाढ (compounding) निवडण्याची अनुमती देतो. हे करण्यासाठी अधिकृत साधन Launchpad आहे. ही प्रक्रिया पूर्ववत केली जाऊ शकत नाही.

## विहंगावलोकन {#overview}

### कोणावर परिणाम होईल? {#who-is-affected}

प्रमाणक चालवणारी कोणतीही व्यक्ती - ही बहुधा अशी व्यक्ती असते जिला त्यांच्या नियंत्रणाखालील प्रमाणकाचा निर्देशांक (उदा., [प्रमाणक #12345](https://beaconcha.in/validator/12345)) माहित असतो. जर तुम्ही प्रमाणक चालवण्यासाठी प्रोटोकॉल वापरत असाल (उदा., लिडो CSM किंवा Rocket Pool), तर ते maxEB ला कधी आणि समर्थन देतात की नाही हे पाहण्यासाठी तुम्हाला त्यांच्याशी संपर्क साधावा लागेल.

जर तुम्ही तरल स्टेकिंग टोकन (LST) (उदा., rETH किंवा stETH) वापरून स्टेकिंग करत असाल, तर कोणतीही कृती आवश्यक किंवा शिफारस केलेली नाही.

### "maxEB" म्हणजे काय? {#what-is-maxeb}

maxEB = प्रमाणकाची कमाल प्रभावी शिल्लक (MAXimum Effective Balance). पेक्ट्रा हार्ड फोर्कपर्यंत, प्रत्येक प्रमाणक जास्तीत जास्त 32 ETH वर कमाई करतो. पेक्ट्रा नंतर, प्रमाणकांना या बदलाची निवड करून 1 ETH च्या वाढीसह 32 आणि 2048 ETH मधील कोणत्याही शिल्लक रकमेवर कमाई करण्याचा पर्याय आहे.

### प्रमाणक कशी निवड करतो? {#how-does-a-validator-opt-in}

प्रमाणक **Type 1** वरून **Type 2** पैसे काढण्याची अधिकारपत्रे मध्ये रूपांतरित करून maxEB बदलाची निवड करतो. पेक्ट्रा हार्ड फोर्क लाईव्ह झाल्यानंतर हे [Launchpad (प्रमाणक कृती)](https://launchpad.ethereum.org/validator-actions) वर केले जाऊ शकते. **Type 0** → **Type 1** प्रमाणेच, **Type 1** → **Type 2** मध्ये रूपांतरित करणे ही एक अपरिवर्तनीय प्रक्रिया आहे.

### पैसे काढण्याची अधिकारपत्रे म्हणजे काय? {#whats-a-withdrawal-credential}

जेव्हा तुम्ही प्रमाणक चालवता, तेव्हा तुमच्याकडे पैसे काढण्याची अधिकारपत्रे असतात. हे तुमच्या डिपॉझिट डेटा json मध्ये आढळू शकतात किंवा तुम्ही ते तुमच्या प्रमाणकाच्या beaconcha.in [डिपॉझिट टॅब](https://beaconcha.in/validator/12345#deposits) वर पाहू शकता.

1. **Type 0** पैसे काढण्याची अधिकारपत्रे: जर तुमच्या प्रमाणकाची पैसे काढण्याची अधिकारपत्रे `0x00...` ने सुरू होत असतील, तर तुम्ही शापेला हार्ड फोर्कच्या आधी डिपॉझिट केले आहे आणि अद्याप पैसे काढण्याचा पत्ता सेट केलेला नाही.

![Type 0 withdrawal credential](./0x00-wd.png)

2. **Type 1** पैसे काढण्याची अधिकारपत्रे: जर तुमच्या प्रमाणकाची पैसे काढण्याची अधिकारपत्रे `0x01...` ने सुरू होत असतील, तर तुम्ही शापेला हार्ड फोर्कनंतर डिपॉझिट केले आहे किंवा तुमची **Type 0** अधिकारपत्रे आधीच **Type 1** अधिकारपत्रांमध्ये रूपांतरित केली आहेत.

 ![Type 1 withdrawal credential](./0x01-wd.png)

3. **Type 2** पैसे काढण्याची अधिकारपत्रे: हा नवीन पैसे काढण्याची अधिकारपत्रे प्रकार `0x02...` ने सुरू होईल आणि पेक्ट्रा नंतर सक्षम केला जाईल. **Type 2** पैसे काढण्याची अधिकारपत्रे असलेल्या प्रमाणकांना कधीकधी "**चक्रवाढ प्रमाणक (compounding validators)**" म्हटले जाते.

| **परवानगी आहे** | **परवानगी नाही** |
| --- | --- |
| ✅ Type 0 → Type 1 | ❌ Type 0 → Type 2 |
| ✅ Type 1 → Type 2 | ❌ Type 1 → Type 0 |
|  | ❌ Type 2 → Type 1 |
|  | ❌ Type 2 → Type 0 |

### धोके {#risks}

MaxEB प्रमाणकाला त्याची संपूर्ण शिल्लक दुसऱ्या प्रमाणकाला पाठवण्यास सक्षम करते. एकत्रीकरण (consolidation) विनंती सबमिट करणाऱ्या वापरकर्त्यांनी ते ज्या व्यवहारावर स्वाक्षरी करत आहेत त्याचा स्रोत आणि सामग्री सत्यापित केली पाहिजे. maxEB वैशिष्ट्यांचा लाभ घेण्यासाठी अधिकृत साधन Launchpad आहे. जर तुम्ही तृतीय-पक्ष साधन वापरण्याचे ठरवले, तर तुम्ही हे सत्यापित केले पाहिजे की:

- स्रोत प्रमाणकाची सार्वजनिक की (pubkey) आणि पैसे काढण्याचा पत्ता त्यांच्या नियंत्रणाखालील प्रमाणकाशी जुळतो
- लक्ष्य प्रमाणकाची सार्वजनिक की योग्य आहे आणि ती त्यांच्या मालकीची आहे
- जर त्यांचा दुसऱ्या प्रमाणकाला निधी पाठवण्याचा हेतू नसेल, तर विनंती हे रूपांतरण (conversion) आहे, एकत्रीकरण नाही
- व्यवहारावर योग्य पैसे काढण्याच्या पत्त्याद्वारे स्वाक्षरी केली जात आहे

तुम्ही वापरू इच्छित असलेल्या कोणत्याही तृतीय-पक्ष साधनाबद्दल [EthStaker समुदायाशी](https://ethstaker.org/about) चर्चा करण्याची आम्ही **प्रकर्षाने शिफारस करतो**. तुमचा दृष्टिकोन तपासून पाहण्यासाठी आणि चुका टाळण्यासाठी हे एक उपयुक्त ठिकाण आहे. जर तुम्ही दुर्भावनापूर्ण किंवा चुकीच्या पद्धतीने कॉन्फिगर केलेले साधन वापरले, तर **तुमची संपूर्ण प्रमाणक शिल्लक तुमच्या नियंत्रणात नसलेल्या प्रमाणकाला पाठवली जाऊ शकते** — आणि ती परत मिळवण्याचा कोणताही मार्ग नसेल.

## तांत्रिक तपशील {#technical-details}

### प्रवाह {#the-flow}

`ConsolidationRequest` प्रक्रियेचे दोन उपयोग असतील:

1. विद्यमान प्रमाणकाचे **Type 1** वरून **Type 2** प्रमाणकामध्ये रूपांतर करणे
2. इतर प्रमाणकांना विद्यमान **Type 2** प्रमाणकामध्ये एकत्रित करणे

**Type 1** चे **Type 2** प्रमाणकामध्ये रूपांतर करताना, *स्रोत* आणि *लक्ष्य* दोन्ही तोच प्रमाणक असेल ज्याचे तुम्ही रूपांतर करत आहात. या प्रक्रियेसाठी गॅस खर्च होईल आणि ती इतर एकत्रीकरण विनंत्यांच्या मागे रांगेत ठेवली जाईल. ही रांग डिपॉझिट रांगेपेक्षा **वेगळी** आहे आणि नवीन प्रमाणक डिपॉझिट्सचा तिच्यावर परिणाम होत नाही आणि ती [pectrified.com](https://pectrified.com/) वर पाहिली जाऊ शकते.

प्रमाणक एकत्रित करण्यासाठी, तुमच्याकडे एक *लक्ष्य प्रमाणक* असणे आवश्यक आहे ज्याकडे **Type 2** पैसे काढण्याची अधिकारपत्रे आहेत. हे एकत्रित केल्या जाणाऱ्या कोणत्याही प्रमाणक शिल्लक रकमेचे गंतव्यस्थान आहे आणि निर्देशांक जतन केला जातो.

### Type 2 मध्ये रूपांतरित करण्यासाठी आवश्यकता {#requirements-for-converting-to-type-2}

तुम्ही **Type 2** मध्ये रूपांतरित करत असलेल्या पहिल्या प्रमाणकासाठी हे आवश्यक असेल. या प्रमाणकाचा निर्देशांक जतन केला जातो आणि तो सक्रिय असतो. रूपांतरणासाठी, *स्रोत प्रमाणक* == *लक्ष्य प्रमाणक.*

प्रमाणक...

- सक्रिय असावा
- कडे **Type 1** पैसे काढण्याची अधिकारपत्रे असावीत
- निर्गमन स्थितीत (किंवा स्लॅशिंग झालेला) नसावा
- कडे प्रलंबित मॅन्युअली-ट्रिगर केलेली रक्कम काढणे नसावे (स्वीप्ससाठी लागू होत नाही)

![conversion illustration](./conversion.png)

### एकत्रीकरणासाठी आवश्यकता {#requirements-for-consolidating}

ही रूपांतरणासारखीच *समान प्रक्रिया* आहे परंतु जेव्हा *स्रोत प्रमाणक* *लक्ष्य प्रमाणकापेक्षा* वेगळा असतो तेव्हा ती वापरली जाते. लक्ष्य प्रमाणकाचा निर्देशांक जतन केला जातो आणि तो स्रोत प्रमाणकाकडून शिल्लक स्वीकारतो. स्रोत प्रमाणकाचा निर्देशांक `EXITED` स्थितीत ठेवला जातो.

या प्रकरणात, स्रोत प्रमाणकासाठी वरील सर्व समान आवश्यकता आहेत आणि त्याव्यतिरिक्त:

- किमान ~27.3 तास (एक `SHARD_COMMITTEE_PERIOD`) सक्रिय असावा

लक्ष्य प्रमाणक...

- कडे **Type 2** पैसे काढण्याची अधिकारपत्रे असावीत
- निर्गमन स्थितीत नसावा.

![consolidation illustration](./consolidation.png)

### एकत्रीकरण विनंती {#the-consolidation-request}

एकत्रीकरण विनंतीवर स्रोत प्रमाणकाशी संबंधित पैसे काढण्याच्या पत्त्याद्वारे स्वाक्षरी केली जाईल आणि त्यात हे असेल:

1. स्रोत प्रमाणकाचा पत्ता (उदा., `0x15F4B914A0cCd14333D850ff311d6DafbFbAa32b`)
2. स्रोत प्रमाणकाची सार्वजनिक की (उदा., `0xa1d1ad0714035353258038e964ae9675dc0252ee22cea896825c01458e1807bfad2f9969338798548d9858a571f7425c`)
3. त्या लक्ष्य प्रमाणकाची सार्वजनिक की

रूपांतरणामध्ये, 2 आणि 3 समान असतील. ही प्रक्रिया [Launchpad](https://launchpad.ethereum.org/) वर केली जाऊ शकते.

### स्वाक्षरी करण्याच्या आवश्यकता {#signing-requirements}

`ConsolidationRequest` सबमिट करण्यासाठी, **स्रोत प्रमाणकाच्या पैसे काढण्याच्या पत्त्याने** विनंतीवर स्वाक्षरी करणे आवश्यक आहे. हे प्रमाणक निधीवरील नियंत्रण सिद्ध करते.

### कशावर स्वाक्षरी केली जाते? {#what-is-signed}

`ConsolidationRequest` ऑब्जेक्टचे डोमेन-विभक्त [स्वाक्षरी मूळ (signing root)](https://github.com/ethereum/consensus-specs/blob/master/specs/phase0/beacon-chain.md#compute_signing_root) वापरले जाते.

- **डोमेन:** `DOMAIN_CONSOLIDATION_REQUEST`
- **स्वाक्षरी मूळ फील्ड्स:**
  - `source_pubkey`: `BLSPubkey`
  - `target_pubkey`: `BLSPubkey`
  - `source_address`: `ExecutionAddress`

परिणामी **BLS स्वाक्षरी** विनंतीसोबत सबमिट केली जाते.

टीप: स्वाक्षरी करणे पैसे काढण्याच्या पत्त्याद्वारे केले जाते, प्रमाणक की द्वारे नाही.

### अंशतः रक्कम काढणे {#partial-withdrawals}

**Type 1** अधिकारपत्रे असलेल्या प्रमाणकांना त्यांच्या अतिरिक्त शिल्लक रकमेचे (32 ETH वरील काहीही) त्यांच्या पैसे काढण्याच्या पत्त्यावर स्वयंचलित, गॅस-मुक्त स्वीप्स मिळतात. कारण **Type 2** प्रमाणकाला 1 ETH च्या वाढीसह शिल्लक चक्रवाढ करण्याची अनुमती देते, ते 2048 ETH पर्यंत पोहोचेपर्यंत स्वयंचलितपणे शिल्लक स्वीप करणार नाही. **Type 2** प्रमाणकांवरील अंशतः रक्कम काढणे मॅन्युअली ट्रिगर केले जाणे आवश्यक आहे आणि त्यासाठी गॅस खर्च होईल.

## एकत्रीकरण साधने {#consolidation-tooling}

एकत्रीकरण व्यवस्थापित करण्यासाठी अनेक साधने उपलब्ध आहेत. इथेरियम फाउंडेशनने तयार केलेले अधिकृत साधन [Launchpad](https://launchpad.ethereum.org/en/validator-actions) आहे. स्टेकिंग समुदायातील संस्थांनी तयार केलेली तृतीय-पक्ष साधने देखील आहेत जी Launchpad द्वारे प्रदान न केलेली वैशिष्ट्ये देऊ शकतात. जरी येथील साधनांचे इथेरियम फाउंडेशनद्वारे ऑडिट किंवा समर्थन केले जात नसले तरी, खालील साधने समुदायाच्या ज्ञात सदस्यांद्वारे तयार केलेली मुक्त स्रोत (open source) साधने आहेत.

## वारंवार विचारले जाणारे प्रश्न (FAQ) {#faq}

### निवड केल्याने माझे प्रस्ताव नशीब किंवा बक्षिसे बदलतात का? {#change-luck-or-rewards}

नाही. निवड केल्याने तुमची प्रस्ताव संधी कमी होत नाही - तुमची कर्तव्ये आणि प्रस्ताव निवड समान राहते. उदाहरणार्थ, जर तुमच्याकडे दोन 32 ETH प्रमाणक विरुद्ध एक 64 ETH प्रमाणक असेल, तर ब्लॉक प्रस्तावित करण्यासाठी निवडले जाण्याची आणि बक्षिसे मिळवण्याची तुमची एकूण संधी समान असेल.

### निवड केल्याने माझा स्लॅशिंग धोका बदलतो का? {#change-slashing-risk}

लहान किंवा अव्यावसायिक ऑपरेटरसाठी, थोडक्यात उत्तर नाही असे आहे. सविस्तर उत्तर असे आहे की, जलद सूचनांसह प्रति नोड अनेक प्रमाणक चालवणाऱ्या व्यावसायिक ऑपरेटरसाठी, कमी प्रमाणकांमध्ये एकत्रीकरण केल्याने स्लॅशिंगला प्रतिक्रिया देण्याची आणि कॅस्केड घटना टाळण्याची त्यांची क्षमता कमी होऊ शकते. हा धोका भरून काढण्यासाठी सर्व प्रमाणकांसाठी प्रारंभिक स्लॅशिंग *दंड* 1 ETH (प्रति 32 ETH) वरून 0.0078125 ETH (प्रति 32 ETH) पर्यंत लक्षणीयरीत्या कमी करण्यात आला आहे.

### रूपांतरित करण्यासाठी मला माझ्या प्रमाणकातून निर्गमन करावे लागेल का? {#exit-validator}

नाही. तुम्ही निर्गमन न करता जागेवरच रूपांतरित करू शकता.

### रूपांतरित / एकत्रित करण्यासाठी किती वेळ लागेल? {#how-long}

किमान 27.3 तास परंतु एकत्रीकरण देखील रांगेच्या अधीन आहे. ही रांग डिपॉझिट आणि रक्कम काढण्याच्या रांगांपासून स्वतंत्र आहे आणि त्यांच्यामुळे प्रभावित होत नाही.

### मी माझा प्रमाणक निर्देशांक ठेवू शकतो का? {#keep-validator-index}

होय. जागेवरच रूपांतरण केल्याने तोच प्रमाणक निर्देशांक राहतो. जर तुम्ही अनेक प्रमाणक एकत्रित केले, तर तुम्ही फक्त *लक्ष्य प्रमाणकाचा* निर्देशांक ठेवू शकाल.

### मी अटेस्टेशन्स (attestations) चुकवेन का? {#miss-attestations}

दुसऱ्या प्रमाणकामध्ये एकत्रीकरण करताना, स्रोत प्रमाणक निर्गमन करतो आणि लक्ष्य प्रमाणकावर शिल्लक सक्रिय होण्यापूर्वी ~27 तासांचा प्रतीक्षा कालावधी असतो. हा कालावधी **कामगिरी मेट्रिक्सवर परिणाम करत नाही**.

### मला दंड आकारला जाईल का? {#incur-penalties}

नाही. जोपर्यंत तुमचा प्रमाणक ऑनलाइन आहे, तोपर्यंत तुम्हाला दंड आकारला जाणार नाही.

### एकत्रित केल्या जाणाऱ्या प्रमाणकांचे पैसे काढण्याचे पत्ते जुळणे आवश्यक आहे का? {#withdrawal-addresses-match}

नाही. परंतु *स्रोताने* स्वतःच्या पत्त्यावरून विनंती अधिकृत करणे आवश्यक आहे.

### रूपांतरणानंतर माझी बक्षिसे चक्रवाढ होतील का? {#rewards-compound}

होय. **Type 2** अधिकारपत्रांसह, 32 ETH वरील बक्षिसे स्वयंचलितपणे पुन्हा स्टेक केली जातात — परंतु त्वरित नाही. एका लहान बफरमुळे (ज्याला [*हिस्टेरेसिस (hysteresis)*](https://eth2book.info/capella/part2/incentives/balances/#hysteresis) म्हणतात), अतिरिक्त रक्कम पुन्हा स्टेक होण्यापूर्वी तुमची शिल्लक **सुमारे 1.25 ETH अधिक** पोहोचणे आवश्यक आहे. त्यामुळे 33.0 ETH वर चक्रवाढ होण्याऐवजी, ते 33.25 (प्रभावी शिल्लक = 33 ETH), नंतर 34.25 (प्रभावी शिल्लक = 34 ETH), आणि असेच पुढे होते.

### रूपांतरणानंतरही मला स्वयंचलित स्वीप्स मिळू शकतात का? {#automatic-sweep}

स्वयंचलित स्वीप्स केवळ 2048 वरील अतिरिक्त शिल्लक रकमेसह होतील. इतर सर्व अंशतः रक्कम काढण्यासाठी, तुम्हाला ते मॅन्युअली ट्रिगर करावे लागतील.

### मी माझा विचार बदलून Type 2 वरून Type 1 वर परत जाऊ शकतो का? {#go-back-to-type1}

नाही. **Type 2** मध्ये रूपांतरित करणे अपरिवर्तनीय आहे.

### जर मला अनेक प्रमाणक एकत्रित करायचे असतील, तर मला प्रथम प्रत्येकाला Type 2 मध्ये रूपांतरित करावे लागेल का? {#consolidate-multiple-validators}

नाही! एका प्रमाणकाला Type 2 मध्ये रूपांतरित करा आणि नंतर त्याचा लक्ष्य म्हणून वापर करा. त्या Type 2 लक्ष्यामध्ये एकत्रित केलेले इतर सर्व प्रमाणक Type 1 किंवा Type 2 असू शकतात.

### माझा प्रमाणक ऑफलाइन आहे किंवा 32 ETH च्या खाली आहे - तरीही मी त्याचे रूपांतर करू शकतो का? {#offline-or-below-32eth}

होय. जोपर्यंत तो सक्रिय आहे (निर्गमन केलेला नाही) आणि तुम्ही त्याच्या पैसे काढण्याच्या पत्त्यासह स्वाक्षरी करू शकता, तोपर्यंत तुम्ही त्याचे रूपांतर करू शकता.
