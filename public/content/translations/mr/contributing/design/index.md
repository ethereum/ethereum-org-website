कोणत्याही प्रकल्पाचा डिझाइन हा एक महत्त्वाचा भाग असतो आणि तुमचा वेळ आणि डिझाइन कौशल्ये ethereum.org ला देऊन, तुम्ही आमच्या अभ्यागतांसाठी वापरकर्ता अनुभव सुधारण्यास मदत करू शकता. ओपन-सोर्स प्रकल्पामध्ये योगदान दिल्याने संबंधित अनुभव मिळवण्याची आणि सहयोगी वातावरणात तुमची कौशल्ये विकसित करण्याची संधी मिळते. तुम्हाला इतर डिझायनर्स, डेव्हलपर्स आणि समुदाय सदस्यांसोबत काम करण्याची संधी मिळेल, ज्यांचे स्वतःचे अद्वितीय दृष्टिकोन आणि अंतर्दृष्टी असेल.

शेवटी, तुमची डिझाइन कौशल्ये प्रदर्शित करणारा वैविध्यपूर्ण आणि प्रभावी पोर्टफोलिओ तयार करण्याचा हा एक उत्तम मार्ग आहे.

## योगदान कसे द्यावे? {#how-to-contribute}

### <Emoji text=":one:" size={1} /> &nbsp;सुरुवातीच्या डिझाइन प्रोटोटाइपवर अभिप्राय द्या {#design-critique}

आम्हाला कधीकधी आमच्या कच्च्या कल्पनांची चाचणी करण्यासाठी मदतीची आवश्यकता असते. कोणत्याही तांत्रिक ज्ञानाशिवाय योगदान देण्याचा हा एक उत्तम मार्ग आहे.

1. डिझाइन टीम [डिस्कॉर्ड्](https://discord.com/invite/ethereum-org) आणि [GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) वर मॉकअप डिझाइन शेअर करेल.
2. कमेंट्स फंक्शनद्वारे अभिप्राय देण्यासाठी तुम्हाला डिझाइन्सबद्दल मार्गदर्शन केले जाईल.
3. परिणाम GitHub इश्यूमध्ये शेअर केला जाईल आणि नंतर टीमद्वारे बंद केला जाईल.

### <Emoji text=":two:" size={1} /> &nbsp;सर्वेक्षण संशोधनात सहभागी व्हा {#answer-surveys}

आमच्या वेबसाइटवर याद्वारे अभिप्राय द्या:

1. ethereum.org ला भेट देऊन आणि पेजेस वाचून.
2. तळाशी उजव्या कोपर्यात असलेल्या फीडबॅक विजेटवर क्लिक करून आणि डिझाइन आणि सामग्री-संबंधित प्रश्नांची उत्तरे देऊन.
3. मुक्त स्वरूपाच्या प्रश्नांवर लक्ष केंद्रित करा.

### <Emoji text=":three:" size={1} /> &nbsp;वेबसाइटवर डिझाइन संबंधित समस्या शोधा आणि त्यांची तक्रार करा {#report-design-issues}

ethereum.org ही अनेक वैशिष्ट्ये आणि सामग्री असलेली वेगाने वाढणारी वेबसाइट आहे. काही UI सहजपणे जुने होऊ शकतात किंवा सुधारले जाऊ शकतात. तुम्हाला असे काही आढळल्यास, कृपया त्याची तक्रार करा जेणेकरून ते आमच्या लक्षात येईल.

1. वेबसाइट पहा आणि तिच्या डिझाइनकडे लक्ष द्या.
2. तुम्हाला काही व्हिज्युअल किंवा UX समस्या दिसल्यास स्क्रीनशॉट्स आणि नोट्स घ्या.
3. [बग रिपोर्ट](https://github.com/ethereum/ethereum-org-website/issues/new/choose) वापरून आढळलेल्या समस्यांची तक्रार करा.

### <Emoji text=":four:" size={1} /> &nbsp;डिझाइन बदलांचा प्रस्ताव द्या {#propose-design-changes}

तुम्हाला डिझाइन आव्हाने स्वीकारणे सोयीचे वाटत असल्यास, तुम्ही आमच्या GitHub इश्यूज बोर्डला भेट देऊ शकता आणि [डिझाइन-संबंधित इश्यूज](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) फिल्टर करू शकता.

1. आमच्या वेबसाइटवर जा आणि तिच्या डिझाइनकडे लक्ष द्या किंवा आमच्या GitHub रिपॉझिटरीवर जा आणि [“Design required” टॅग](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) असलेल्या इश्यूजचे पुनरावलोकन करा.
2. उपायावर विचार करा आणि ते डिझाइन करा. (शक्यतो आमची [डिझाइन सिस्टीम](https://www.figma.com/community/file/1134414495420383395) वापरून).
3. संबंधित GitHub इश्यूमध्ये उपायाचा प्रस्ताव द्या किंवा [नवीन तयार करा.](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. डिझाइन टीमच्या पुनरावलोकनाची प्रतीक्षा करा.

### <Emoji text=":five:" size={1} /> &nbsp;एकत्रितपणे डिझाइन सिस्टीम तयार करा {#contribute-to-design-system}

आमची डिझाइन सिस्टीम ethereum.org चे डिझाइन करणे मजेदार आणि सोपे बनवते. जर तुम्ही अनुभवी डिझायनर असाल, तर तुम्ही आम्हाला वेबसाइटसाठी अनेक घटक तयार करण्यात मदत करू शकता.

1. GitHub वरील [डिझाइन सिस्टीम बोर्ड](https://github.com/ethereum/ethereum-org-website/labels/design%20system) वरून काम करण्यासाठी एखादा इश्यू निवडा किंवा नवीन तयार करा.
2. निवडलेला इश्यू तुम्हाला नियुक्त करण्याची विनंती करा.
3. Figma मध्ये विनंती केलेल्या घटकाचे डिझाइन सुरू करा.
4. तुम्हाला पुनरावलोकन किंवा मार्गदर्शनाची आवश्यकता असल्यास ते GitHub वर डिझाइन टीमसोबत शेअर करा.
5. डिझाइन टीम पुनरावलोकन करेल.
6. डिझाइन टीम मुख्य फाइलमध्ये बदल समाविष्ट करेल आणि फाइल समुदायासाठी प्रकाशित करेल.

### <Emoji text=":six:" size={1} /> &nbsp;वेबसाइटवर डिझाइन-संबंधित सामग्री लिहा {#write-design-articles}

इथेरियम डेव्हलपर समुदाय मजबूत आहे, परंतु डिझाइन समुदाय थोडा मागे पडत आहे. जर तुम्ही Web3 चे ज्ञान असलेले डिझायनर असाल, तर कृपया तुमचे शिक्षण मोठ्या समुदायासोबत शेअर करण्याचा विचार करा जेणेकरून आपण सर्वजण एकत्र वाढू आणि सुधारू शकू; आमच्याकडे [इथेरियमसाठी डिझाइन करण्यावर एक पेज](/developers/docs/design-and-ux/) आहे ज्यामध्ये तुम्ही योगदान देऊ शकता. तुम्ही आमची [लिस्टिंग धोरणे](/contributing/design/adding-design-resources) देखील तपासू शकता.

1. ethereum.org वर कव्हर केल्या जाणाऱ्या आणि या क्षेत्रातील डिझायनर्ससाठी फायदेशीर ठरणाऱ्या डिझाइन विषयांवर विचार करा.
2. आमच्या GitHub रिपॉझिटरीवर जा आणि विषय प्रस्तावित करणारा [इश्यू तयार करा](https://github.com/ethereum/ethereum-org-website/issues/new) (अद्याप सामग्री लिहू नका).
3. डिझाइन टीमच्या मंजुरीची प्रतीक्षा करा.
4. मंजूर झाल्यावर, सामग्री लिहा.
5. संबंधित GitHub इश्यूमध्ये ते सबमिट करा.

### <Emoji text=":seven:" size={1} /> &nbsp;नवीन चित्रे काढा {#prepare-illustrations}

अमूर्त विषय स्पष्ट करण्यासाठी व्हिज्युअलायझेशन हे सर्वात शक्तिशाली साधनांपैकी एक आहे. आकृत्या आणि इन्फोग्राफिक्स जोडून प्रचंड क्षमता निर्माण होते. शेवटी, एक चित्र हजार शब्दांचे काम करू शकते.

1. आमच्या वेबसाइटवर जा आणि अशी पेजेस पहा जिथे काही नवीन इन्फोग्राफिक्स जोडले जाऊ शकतात.
2. चित्राची शैली आमच्या [अॅसेट्सशी](/assets/) जुळत असल्याची खात्री करा.
3. आमच्या GitHub रिपॉझिटरीवर जा आणि चित्र प्रस्तावित करणारा [इश्यू तयार करा](https://github.com/ethereum/ethereum-org-website/issues/new).
4. डिझाइन टीम त्याचे पुनरावलोकन करेल.
5. नवीन इमेज लागू करण्यासाठी डेव्हलपरला विचारण्यासाठी आम्ही एक नवीन इश्यू तयार करू.
