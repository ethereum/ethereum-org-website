**Ethereum on Arm ही एक सानुकूल Linux इमेज आहे जी Raspberry Pi ला इथेरियम नोडमध्ये बदलू शकते.**

Raspberry Pi ला इथेरियम नोडमध्ये बदलण्यासाठी Ethereum on Arm वापरण्याकरिता, खालील हार्डवेअरची शिफारस केली जाते:

- Raspberry 4 (मॉडेल B 8GB), Odroid M1 किंवा Rock 5B (8GB/16GB RAM) बोर्ड
- MicroSD कार्ड (किमान 16 GB क्लास 10)
- किमान 2 TB SSD USB 3.0 डिस्क किंवा USB ते SATA केस असलेली SSD.
- वीज पुरवठा (Power supply)
- इथरनेट केबल
- पोर्ट फॉरवर्डिंग (अधिक माहितीसाठी क्लायंट्स पहा)
- हीटसिंक आणि फॅन असलेली केस
- USB कीबोर्ड, मॉनिटर आणि HDMI केबल (मायक्रो-HDMI) (पर्यायी)

## ARM वर इथेरियम का चालवावे? {#why-run-ethereum-on-arm}

ARM बोर्ड हे अतिशय परवडणारे, लवचिक आणि छोटे संगणक आहेत. इथेरियम नोड्स चालवण्यासाठी ते उत्तम पर्याय आहेत कारण ते स्वस्तात विकत घेता येतात, त्यांचे सर्व संसाधने फक्त नोडवर केंद्रित करण्यासाठी कॉन्फिगर केले जाऊ शकतात, ज्यामुळे ते कार्यक्षम बनतात, ते कमी प्रमाणात वीज वापरतात आणि शारीरिकदृष्ट्या लहान असल्यामुळे ते कोणत्याही घरात सहजपणे बसू शकतात. नोड्स सुरू करणे देखील खूप सोपे आहे कारण Raspberry Pi चे MicroSD फक्त पूर्व-निर्मित (prebuilt) इमेजसह फ्लॅश केले जाऊ शकते, ज्यासाठी कोणतेही सॉफ्टवेअर डाउनलोड किंवा तयार करण्याची आवश्यकता नसते.

## हे कसे काम करते? {#how-does-it-work}

Raspberry Pi चे मेमरी कार्ड पूर्व-निर्मित इमेजसह फ्लॅश केले जाते. या इमेजमध्ये इथेरियम नोड चालवण्यासाठी आवश्यक असलेले सर्व काही असते. फ्लॅश केलेल्या कार्डसह, वापरकर्त्याला फक्त Raspberry Pi चालू करणे आवश्यक आहे. नोड चालवण्यासाठी आवश्यक असलेल्या सर्व प्रक्रिया आपोआप सुरू होतात. हे काम करते कारण मेमरी कार्डमध्ये Linux-आधारित ऑपरेटिंग सिस्टम (OS) असते ज्यावर सिस्टम-स्तरीय प्रक्रिया आपोआप चालवल्या जातात ज्या युनिटला इथेरियम नोडमध्ये बदलतात.

लोकप्रिय Raspberry Pi Linux OS "Raspbian" वापरून इथेरियम चालवता येत नाही कारण Raspbian अजूनही 32-बिट आर्किटेक्चर वापरते ज्यामुळे इथेरियम वापरकर्त्यांना मेमरी समस्या येतात आणि सहमती क्लायंट 32-बिट बायनरींना समर्थन देत नाहीत. यावर मात करण्यासाठी, Ethereum on Arm टीमने "Armbian" नावाच्या मूळ 64-बिट OS वर स्थलांतर केले.

**इमेजेस सर्व आवश्यक पायऱ्यांची काळजी घेतात**, वातावरण सेट करण्यापासून आणि SSD डिस्क फॉरमॅट करण्यापासून ते इथेरियम सॉफ्टवेअर स्थापित आणि चालवण्यापर्यंत तसेच ब्लॉकचेन समक्रमण सुरू करण्यापर्यंत.

## अंमलबजावणी आणि सहमती क्लायंट्सवरील नोंद {#note-on-execution-and-consensus-clients}

Ethereum on Arm इमेजमध्ये सेवा म्हणून पूर्व-निर्मित अंमलबजावणी आणि सहमती क्लायंट्स समाविष्ट आहेत. इथेरियम नोडसाठी दोन्ही क्लायंट्स समक्रमित आणि चालू असणे आवश्यक आहे. तुम्हाला फक्त इमेज डाउनलोड आणि फ्लॅश करणे आणि नंतर सेवा सुरू करणे आवश्यक आहे. इमेजमध्ये खालील अंमलबजावणी क्लायंट्स पूर्व-लोड केलेले आहेत:

- गेथ
- नेदरमाइंड
- बेसू

आणि खालील सहमती क्लायंट्स:

- लाइटहाऊस
- निंबस
- प्रिझम
- टेकू

तुम्ही चालवण्यासाठी प्रत्येकी एक निवडला पाहिजे - सर्व अंमलबजावणी क्लायंट्स सर्व सहमती क्लायंट्सशी सुसंगत आहेत. जर तुम्ही स्पष्टपणे क्लायंट निवडला नाही, तर नोड त्याच्या डीफॉल्ट्सवर - गेथ आणि लाइटहाऊस - परत जाईल आणि जेव्हा बोर्ड चालू होईल तेव्हा त्यांना आपोआप चालवेल. तुम्ही तुमच्या राउटरवर पोर्ट 30303 उघडणे आवश्यक आहे जेणेकरून गेथ पीअर्स शोधू शकेल आणि त्यांच्याशी कनेक्ट होऊ शकेल.

## इमेज डाउनलोड करणे {#downloading-the-image}

Raspberry Pi 4 इथेरियम इमेज ही एक "प्लग आणि प्ले" इमेज आहे जी अंमलबजावणी आणि सहमती क्लायंट्स दोन्ही आपोआप स्थापित आणि सेट करते, त्यांना एकमेकांशी बोलण्यासाठी आणि इथेरियम नेटवर्कशी कनेक्ट होण्यासाठी कॉन्फिगर करते. वापरकर्त्याला फक्त एका सोप्या कमांडचा वापर करून त्यांच्या प्रक्रिया सुरू करणे आवश्यक आहे.

[Ethereum on Arm](https://ethereumonarm-my.sharepoint.com/:u:/p/dlosada/Ec_VmUvr80VFjf3RYSU-NzkBmj2JOteDECj8Bibde929Gw?download=1) वरून Raspberry Pi इमेज डाउनलोड करा आणि SHA-256 हॅश सत्यापित करा:

```sh
# डाउनलोड केलेली इमेज असलेल्या डिरेक्टरीमधून
shasum -a 256 ethonarm_22.04.00.img.zip
# हॅश आउटपुट असायला हवे: fb497e8f8a7388b62d6e1efbc406b9558bee7ef46ec7e53083630029c117444f
```

नोंद घ्या की Rock 5B आणि Odroid M1 बोर्डसाठी इमेजेस Ethereum-on-Arm च्या [डाउनलोड्स पृष्ठावर](https://ethereum-on-arm-documentation.readthedocs.io/en/latest/) उपलब्ध आहेत.

## MicroSD फ्लॅश करणे {#flashing-the-microsd}

Raspberry Pi साठी वापरले जाणारे MicroSD कार्ड प्रथम डेस्कटॉप किंवा लॅपटॉपमध्ये घातले पाहिजे जेणेकरून ते फ्लॅश केले जाऊ शकेल. त्यानंतर, खालील टर्मिनल कमांड्स डाउनलोड केलेली इमेज SD कार्डवर फ्लॅश करतील:

```shell
# MicroSD कार्डचे नाव तपासा
sudo fdisk -l

>> sdxxx
```

नाव योग्य असणे खरोखर महत्त्वाचे आहे कारण पुढील कमांडमध्ये `dd` समाविष्ट आहे जे इमेज कार्डवर ढकलण्यापूर्वी कार्डवरील विद्यमान सामग्री पूर्णपणे पुसून टाकते. पुढे जाण्यासाठी, झिप केलेली इमेज असलेल्या डिरेक्टरीमध्ये जा:

```shell
# इमेज अनझिप आणि फ्लॅश करा
unzip ethonarm_22.04.00.img.zip
sudo dd bs=1M if=ethonarm_22.04.00.img of=/dev/<sdxx> conv=fdatasync status=progress
```

कार्ड आता फ्लॅश झाले आहे, त्यामुळे ते Raspberry Pi मध्ये घातले जाऊ शकते.

## नोड सुरू करा {#start-the-node}

Raspberry Pi मध्ये SD कार्ड घातल्यानंतर, इथरनेट केबल आणि SSD कनेक्ट करा आणि नंतर वीज चालू करा. OS बूट होईल आणि Raspberry Pi ला इथेरियम नोडमध्ये बदलणारी पूर्व-कॉन्फिगर केलेली कार्ये आपोआप करण्यास सुरुवात करेल, ज्यामध्ये क्लायंट सॉफ्टवेअर स्थापित करणे आणि तयार करणे समाविष्ट आहे. यास बहुधा 10-15 मिनिटे लागतील.

एकदा सर्वकाही स्थापित आणि कॉन्फिगर झाल्यानंतर, ssh कनेक्शनद्वारे किंवा बोर्डला मॉनिटर आणि कीबोर्ड जोडलेला असल्यास थेट टर्मिनल वापरून डिव्हाइसमध्ये लॉग इन करा. लॉग इन करण्यासाठी `ethereum` खाते वापरा, कारण यामध्ये नोड सुरू करण्यासाठी आवश्यक परवानग्या आहेत.

```shell
User: ethereum
Password: ethereum
```

डीफॉल्ट अंमलबजावणी क्लायंट, गेथ, आपोआप सुरू होईल. तुम्ही खालील टर्मिनल कमांड वापरून नोंदी तपासून याची पुष्टी करू शकता:

```sh
sudo journalctl -u geth -f
```

सहमती क्लायंट स्पष्टपणे सुरू करणे आवश्यक आहे. हे करण्यासाठी, प्रथम तुमच्या राउटरवर पोर्ट 9000 उघडा जेणेकरून लाइटहाऊस पीअर्स शोधू शकेल आणि त्यांच्याशी कनेक्ट होऊ शकेल. त्यानंतर लाइटहाऊस सेवा सक्षम करा आणि सुरू करा:

```sh
sudo systemctl enable lighthouse-beacon
sudo systemctl start lighthouse-beacon
```

नोंदी वापरून क्लायंट तपासा:

```sh
sudo journalctl -u lighthouse-beacon
```

नोंद घ्या की सहमती क्लायंट काही मिनिटांत समक्रमित होईल कारण तो चेकपॉईंट समक्रमण वापरतो. अंमलबजावणी क्लायंटला जास्त वेळ लागेल - संभाव्यतः अनेक तास, आणि जोपर्यंत सहमती क्लायंटचे समक्रमण पूर्ण होत नाही तोपर्यंत तो सुरू होणार नाही (याचे कारण असे की अंमलबजावणी क्लायंटला समक्रमित करण्यासाठी लक्ष्याची आवश्यकता असते, जे समक्रमित सहमती क्लायंट प्रदान करतो).

गेथ आणि लाइटहाऊस सेवा चालू आणि समक्रमित झाल्यामुळे, तुमचा Raspberry Pi आता एक इथेरियम नोड आहे! गेथच्या JavaScript कन्सोलचा वापर करून इथेरियम नेटवर्कशी संवाद साधणे सर्वात सामान्य आहे, जे पोर्ट 8545 वर गेथ क्लायंटशी जोडले जाऊ शकते. Curl सारख्या विनंती टूलचा वापर करून JSON ऑब्जेक्ट्स म्हणून फॉरमॅट केलेल्या कमांड्स सबमिट करणे देखील शक्य आहे. [गेथ दस्तऐवजीकरणामध्ये](https://geth.ethereum.org/) अधिक पहा.

गेथ हे Grafana डॅशबोर्डवर मेट्रिक्स नोंदवण्यासाठी पूर्व-कॉन्फिगर केलेले आहे जे ब्राउझरमध्ये पाहिले जाऊ शकते. अधिक प्रगत वापरकर्ते `ipaddress:3000` वर नेव्हिगेट करून, `user: admin` आणि `passwd: ethereum` पास करून त्यांच्या नोडच्या आरोग्यावर लक्ष ठेवण्यासाठी या वैशिष्ट्याचा वापर करू इच्छितात.

## प्रमाणक {#validators}

सहमती क्लायंटमध्ये पर्यायीरित्या प्रमाणक देखील जोडला जाऊ शकतो. प्रमाणक सॉफ्टवेअर तुमच्या नोडला एकमतामध्ये सक्रियपणे सहभागी होण्याची परवानगी देते आणि नेटवर्कला क्रिप्टो-आर्थिक सुरक्षा प्रदान करते. या कामासाठी तुम्हाला ETH मध्ये बक्षीस मिळते. प्रमाणक चालवण्यासाठी, तुमच्याकडे प्रथम 32 ETH असणे आवश्यक आहे, जे ठेव करारामध्ये जमा केले जाणे आवश्यक आहे. [लॉन्चपॅडवरील](https://launchpad.ethereum.org/) चरण-दर-चरण मार्गदर्शकाचे अनुसरण करून ठेव जमा केली जाऊ शकते. हे डेस्कटॉप/लॅपटॉपवर करा, परंतु कीज (keys) व्युत्पन्न करू नका — हे थेट Raspberry Pi वर केले जाऊ शकते.

Raspberry Pi वर टर्मिनल उघडा आणि ठेव कीज व्युत्पन्न करण्यासाठी खालील कमांड चालवा:

```
sudo apt-get update
sudo apt-get install staking-deposit-cli
cd && deposit new-mnemonic --num_validators 1
```

(किंवा एअरगॅप्ड मशीनवर चालवण्यासाठी [staking-deposit-cli](https://github.com/ethereum/staking-deposit-cli) डाउनलोड करा, आणि `deposit new-mnemnonic` कमांड चालवा)

निमोनिक (mnemonic) वाक्प्रचार सुरक्षित ठेवा! वरील कमांडने नोडच्या कीस्टोअरमध्ये दोन फाइल्स व्युत्पन्न केल्या: प्रमाणक कीज आणि ठेव डेटा फाइल. ठेव डेटा लॉन्चपॅडमध्ये अपलोड करणे आवश्यक आहे, त्यामुळे तो Raspberry Pi वरून डेस्कटॉप/लॅपटॉपवर कॉपी करणे आवश्यक आहे. हे ssh कनेक्शन किंवा इतर कोणत्याही कॉपी/पेस्ट पद्धतीचा वापर करून केले जाऊ शकते.

एकदा ठेव डेटा फाइल लॉन्चपॅड चालवणाऱ्या संगणकावर उपलब्ध झाल्यानंतर, ती लॉन्चपॅड स्क्रीनवरील `+` वर ड्रॅग आणि ड्रॉप केली जाऊ शकते. ठेव करारावर व्यवहार पाठवण्यासाठी स्क्रीनवरील सूचनांचे अनुसरण करा.

पुन्हा Raspberry Pi वर, प्रमाणक सुरू केला जाऊ शकतो. यासाठी प्रमाणक कीज आयात करणे, बक्षिसे गोळा करण्यासाठी पत्ता सेट करणे आणि नंतर पूर्व-कॉन्फिगर केलेली प्रमाणक प्रक्रिया सुरू करणे आवश्यक आहे. खालील उदाहरण लाइटहाऊससाठी आहे—इतर सहमती क्लायंट्ससाठी सूचना [Ethereum on Arm दस्तऐवजांवर](https://ethereum-on-arm-documentation.readthedocs.io/en/latest/) उपलब्ध आहेत:

```shell
# प्रमाणक कीज आयात करा
lighthouse account validator import --directory=/home/ethereum/validator_keys

# बक्षीस पत्ता सेट करा
sudo sed -i 's/<ETH_ADDRESS>' /etc/ethereum/lighthouse-validator.conf

# प्रमाणक सुरू करा
sudo systemctl start lighthouse-validator
```

अभिनंदन, आता तुमच्याकडे Raspberry Pi वर चालणारा पूर्ण इथेरियम नोड आणि प्रमाणक आहे!

## अधिक तपशील {#more-details}

या पृष्ठाने Raspberry Pi वापरून गेथ-लाइटहाऊस नोड आणि प्रमाणक कसे सेट करावे याचे विहंगावलोकन दिले. अधिक तपशीलवार सूचना [Ethereum-on-Arm वेबसाइटवर](https://ethereum-on-arm-documentation.readthedocs.io/en/latest/) उपलब्ध आहेत.
