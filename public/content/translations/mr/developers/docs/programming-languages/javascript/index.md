इथेरियम इकोसिस्टममध्ये JavaScript ही सर्वात लोकप्रिय भाषांपैकी एक आहे. किंबहुना, इथेरियमला शक्य तितक्या प्रमाणात JavaScript मध्ये आणण्यासाठी एक समर्पित [टीम](https://github.com/ethereumjs) आहे.

[स्टॅकच्या सर्व स्तरांवर](/developers/docs/ethereum-stack/) JavaScript (किंवा त्यासारखे काहीतरी) लिहिण्याच्या संधी आहेत.

## इथेरियमशी संवाद साधा {#interact-with-ethereum}

### JavaScript API लायब्ररीज {#javascript-api-libraries}

जर तुम्हाला ब्लॉकचेनला क्वेरी करण्यासाठी, ट्रान्झॅक्शन्स पाठवण्यासाठी आणि बरेच काही करण्यासाठी JavaScript लिहायचे असेल, तर हे करण्याचा सर्वात सोयीस्कर मार्ग म्हणजे [JavaScript API लायब्ररी](/developers/docs/apis/javascript/) वापरणे. हे APIs डेव्हलपर्सना [इथेरियम नेटवर्कमधील नोड्सशी](/developers/docs/nodes-and-clients/) सहजपणे संवाद साधण्याची परवानगी देतात.

तुम्ही या लायब्ररीजचा वापर इथेरियमवरील स्मार्ट कॉन्ट्रॅक्ट्सशी संवाद साधण्यासाठी करू शकता, त्यामुळे असे विकेंद्रित ॲप्लिकेशन (dapp) तयार करणे शक्य आहे जिथे तुम्ही आधीपासून अस्तित्वात असलेल्या कॉन्ट्रॅक्ट्सशी संवाद साधण्यासाठी फक्त JavaScript वापरता.

**हे तपासा**

- [Web3.js](https://web3js.readthedocs.io)
- [Ethers.js](https://ethers.org) – _यामध्ये JavaScript आणि TypeScript मधील इथेरियम वॉलेट अंमलबजावणी आणि युटिलिटीज समाविष्ट आहेत._
- [viem](https://viem.sh) – _इथेरियमसाठी एक TypeScript इंटरफेस जो इथेरियमशी संवाद साधण्यासाठी लो-लेव्हल स्टेटलेस प्रिमिटिव्ह्ज प्रदान करतो._
- [Drift](https://ryangoree.github.io/drift/) – _Web3 लायब्ररीजमध्ये सहज इथेरियम डेव्हलपमेंटसाठी अंगभूत कॅशिंग, हुक्स आणि टेस्ट मॉक्स असलेली एक TypeScript मेटा-लायब्ररी._

### स्मार्ट कॉन्ट्रॅक्ट्स {#smart-contracts}

जर तुम्ही JavaScript डेव्हलपर असाल आणि तुम्हाला तुमचे स्वतःचे स्मार्ट कॉन्ट्रॅक्ट लिहायचे असेल, तर तुम्हाला [Solidity](https://solidity.readthedocs.io) ची माहिती करून घेणे उपयुक्त ठरेल. ही सर्वात लोकप्रिय स्मार्ट कॉन्ट्रॅक्ट भाषा आहे आणि ती सिंटॅक्सच्या दृष्टीने JavaScript सारखीच आहे, ज्यामुळे ती शिकणे सोपे होऊ शकते.

[स्मार्ट कॉन्ट्रॅक्ट्स](/developers/docs/smart-contracts/) बद्दल अधिक माहिती.

## प्रोटोकॉल समजून घ्या {#understand-the-protocol}

### इथेरियम व्हर्च्युअल मशीन {#the-ethereum-virtual-machine}

[इथेरियमच्या व्हर्च्युअल मशीनची](/developers/docs/evm/) एक JavaScript अंमलबजावणी आहे. ती नवीनतम फोर्क नियमांना समर्थन देते. फोर्क नियम म्हणजे नियोजित अपग्रेड्सचा परिणाम म्हणून EVM मध्ये केलेले बदल.

हे विविध JavaScript पॅकेजेसमध्ये विभागले गेले आहे जे तुम्ही अधिक चांगल्या प्रकारे समजून घेण्यासाठी तपासू शकता:

- खाती (Accounts)
- ब्लॉक्स
- स्वतः ब्लॉकचेन
- ट्रान्झॅक्शन्स
- आणि बरेच काही...

हे तुम्हाला "खात्याची डेटा रचना काय आहे?" यासारख्या गोष्टी समजून घेण्यास मदत करेल.

जर तुम्हाला कोड वाचायला आवडत असेल, तर आमचे डॉक्युमेंट्स वाचण्याऐवजी ही JavaScript एक उत्तम पर्याय असू शकते.

**EVM तपासा**  
[`@ethereumjs/evm`](https://github.com/ethereumjs/ethereumjs-monorepo/tree/master/packages/evm)

### नोड्स आणि क्लायंट्स {#nodes-and-clients}

एक EthereumJS क्लायंट सक्रिय डेव्हलपमेंटमध्ये आहे जो तुम्हाला समजणाऱ्या भाषेत, म्हणजेच JavaScript मध्ये, इथेरियम क्लायंट्स कसे काम करतात हे सखोलपणे समजून घेण्यास मदत करतो!

**क्लायंट तपासा**  
[`@ethereumjs/client`](https://github.com/ethereumjs/ethereumjs-monorepo/tree/master/packages/client)

## इतर प्रोजेक्ट्स {#other-projects}

इथेरियम JavaScript च्या जगात इतरही अनेक गोष्टी घडत आहेत, ज्यामध्ये खालील गोष्टींचा समावेश आहे:

- वॉलेट युटिलिटीजच्या लायब्ररीज.
- इथेरियम कीज (keys) जनरेट, इम्पोर्ट आणि एक्सपोर्ट करण्यासाठी टूल्स.
- `merkle-patricia-tree` ची अंमलबजावणी – इथेरियम येलो पेपरमध्ये वर्णन केलेली एक डेटा रचना.

तुम्हाला ज्यामध्ये सर्वात जास्त रस आहे ते [EthereumJS रेपो](https://github.com/ethereumjs) वर सखोलपणे तपासा
