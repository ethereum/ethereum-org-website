इथेरियम मोठ्या प्रमाणात डेटा तयार करते, साठवते आणि हस्तांतरित करते. कोणालाही तुलनेने सामान्य ग्राहक-श्रेणीच्या हार्डवेअरवर [नोड चालवण्याची](/run-a-node/) अनुमती देण्यासाठी हा डेटा प्रमाणित आणि मेमरी-कार्यक्षम मार्गांनी फॉरमॅट केलेला असणे आवश्यक आहे. हे साध्य करण्यासाठी, इथेरियम स्टॅकवर अनेक विशिष्ट डेटा स्ट्रक्चर्स वापरले जातात.

## पूर्व आवश्यकता {#prerequisites}

तुम्हाला इथेरियम आणि [क्लायंट सॉफ्टवेअर](/developers/docs/nodes-and-clients/) च्या मूलभूत गोष्टी समजल्या पाहिजेत. नेटवर्किंग लेयर आणि [इथेरियम श्वेतपत्रिका](/whitepaper/) यांची माहिती असण्याची शिफारस केली जाते.

## डेटा स्ट्रक्चर्स {#data-structures}

### पॅट्रिशिया मर्कल ट्राईज {#patricia-merkle-tries}

पॅट्रिशिया मर्कल ट्राईज (Patricia Merkle Tries) हे असे स्ट्रक्चर्स आहेत जे की-व्हॅल्यू जोड्यांना एका निश्चित आणि क्रिप्टोग्राफिकदृष्ट्या प्रमाणित ट्राई (trie) मध्ये एन्कोड करतात. यांचा इथेरियमच्या अंमलबजावणी स्तरावर मोठ्या प्रमाणावर वापर केला जातो.

[पॅट्रिशिया मर्कल ट्राईज बद्दल अधिक माहिती](/developers/docs/data-structures-and-encoding/patricia-merkle-trie)

### रिकर्सिव्ह लेंथ प्रीफिक्स {#recursive-length-prefix}

रिकर्सिव्ह लेंथ प्रीफिक्स (RLP) ही एक क्रमिकरण पद्धत आहे जी इथेरियमच्या अंमलबजावणी स्तरावर मोठ्या प्रमाणावर वापरली जाते.

[RLP बद्दल अधिक माहिती](/developers/docs/data-structures-and-encoding/rlp)

### सिम्पल सीरियलाइज {#simple-serialize}

सिम्पल सीरियलाइज (SSZ) हे इथेरियमच्या सहमती स्तरावरील प्रमुख क्रमिकरण स्वरूप आहे कारण ते मर्कलायझेशनशी सुसंगत आहे.

[SSZ बद्दल अधिक माहिती](/developers/docs/data-structures-and-encoding/ssz)
