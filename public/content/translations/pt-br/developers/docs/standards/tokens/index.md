## Introdução {#introduction}

Muitos padrões de desenvolvimento do [Ethereum](/) focam em interfaces de token. Esses padrões ajudam a garantir que os contratos inteligentes permaneçam compuníveis, de modo que, quando um novo projeto emite um token, ele permaneça compatível com as corretoras e aplicativos descentralizados existentes.

Os padrões de token definem como os tokens se comportam e interagem em todo o ecossistema Ethereum. Eles facilitam a construção por parte dos desenvolvedores sem precisar reinventar a roda, garantindo que os tokens funcionem perfeitamente com carteiras, corretoras e plataformas de finanças descentralizadas (DeFi). Seja em jogos, governança ou outros casos de uso, esses padrões fornecem consistência e tornam o Ethereum mais interconectado.

## Padrões de token {#token-standards}

Aqui estão alguns dos padrões de token mais populares no Ethereum:

- [ERC-20](/developers/docs/standards/tokens/erc-20/) - Uma interface padrão para tokens fungíveis (intercambiáveis), como tokens de voto, tokens de staking ou moedas virtuais.

### Padrões de NFT {#nft-standards}

- [ERC-721](/developers/docs/standards/tokens/erc-721/) - Uma interface padrão para tokens não fungíveis, como uma escritura de uma obra de arte ou uma música.
- [ERC-1155](/developers/docs/standards/tokens/erc-1155/) - O ERC-1155 permite negociações mais eficientes e o agrupamento de transações – economizando custos. Esse padrão de token permite a criação tanto de tokens de utilidade (como $BNB ou $BAT) quanto de Tokens Não Fungíveis, como os CryptoPunks.

A lista completa de propostas [ERC](https://eips.ethereum.org/erc).
