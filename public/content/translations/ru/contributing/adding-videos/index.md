# Добавление видео {#adding-videos}

В [галерее видео ethereum.org](/videos/) представлены видео об Эфириуме и экосистеме Эфириума от авторов из сообщества и надежных источников. Любой желающий может предложить видео для добавления.

## Правила размещения {#listing-policy}

Ethereum.org — это нейтральный образовательный ресурс. Галерея видео курируется с целью:

- **Обучать** пользователей технологиям Эфириума, его экосистеме и сообществу
- **Сохранять точность** технического контента
- **Оставаться актуальной** для сообщества Эфириума

Сайт не размещает видео, которые в первую очередь продвигают определенный продукт, токен или коммерческую услугу.


## Критерии включения
## Критерии для включения {#criteria-for-inclusion}

### Обязательные требования {#must-haves}

- **Ориентация на Эфириум** — видео должно быть в первую очередь об Эфириуме, его технологиях, экосистеме или сообществе. Видео на общие темы про блокчейн допускаются только в том случае, если они существенно дополняют или относятся к образовательной странице на сайте, либо упоминают Эфириум.
- **Образовательная ценность** — видео должно рассказывать зрителям что-то новое об Эфириуме или отмечать достижения глобального сообщества Эфириума. Рекламный или маркетинговый контент не принимается.
- **Достоверная информация** — технический контент должен быть фактически правильным и актуальным. Устаревшие видео об отмененных функциях могут быть удалены.
- **Качество производства** — видео должно иметь достаточно четкое качество звука и изображения.
- **Общедоступность** — видео должно быть размещено на открытом ресурсе или доступной платформе, такой как YouTube, и быть в свободном доступе без платного доступа или требования регистрации.

### Желательные требования {#nice-to-haves}

- **Наличие расшифровки** — видео с текстовой расшифровкой улучшают доступность и SEO. Если у вас ее нет, команда ethereum.org может помочь с ее созданием.
- **Из надежного источника** — контент от признанных преподавателей, исследователей и авторитетных источников имеет приоритет.
- **Актуальность вне времени** — контент, который остается актуальным с течением времени, предпочтительнее материалов, привязанных к конкретному моменту.


## Как добавить видео
## Как добавить видео {#how-to-add-a-video}

### Вариант 1: Открыть issue {#open-an-issue}

Если вы хотите предложить видео, но не хотите создавать файлы самостоятельно, откройте issue на GitHub с подробной информацией о видео, и один из участников поможет вам его добавить.

### Вариант 2: Открыть pull request {#open-a-pull-request}

Если вы хотите добавить видео самостоятельно, выполните следующие шаги:

#### Шаг 1: Создайте файл видео {#step-1}

Создайте новую директорию и файл `index.md` по пути:

```
public/content/videos/{your-video-slug}/index.md
```

Слаг (slug) должен быть безопасным для URL, в нижнем регистре и использовать дефисы (например, `blockchain-101-visual-demo`).

#### Шаг 2: Добавьте frontmatter {#step-2}

Добавьте следующий YAML frontmatter в ваш `index.md`:

```yaml
---
title: "Your Video Title"
description: "A brief 1–3 sentence summary of the video."
lang: en
youtubeId: "dQw4w9WgXcQ"
uploadDate: 2025-01-15
duration: "12:30"
educationLevel: beginner
topic:
  - "your-topic"
  - "another-topic"
format: explainer
author: Channel Name
---
```

**Описание полей:**

| Поле | Обязательно | Описание |
|---|---|---|
| `title` | Да | Название видео |
| `description` | Да | Краткое описание из 1–3 предложений |
| `lang` | Да | Пока всегда `en` |
| `youtubeId` | Да | ID видео на YouTube (из URL после `v=`) |
| `uploadDate` | Да | Дата оригинальной загрузки в формате `YYYY-MM-DD` |
| `duration` | Да | Длительность видео в формате `H:MM:SS` или `M:SS` |
| `educationLevel` | Да | `beginner`, `intermediate` или `advanced` |
| `topic` | Да | Массив тематических тегов для фильтрации в галерее |
| `format` | Да | `explainer`, `presentation`, `interview`, `tutorial` или `panel` |
| `author` | Да | Имя автора или название канала |
| `breadcrumb` | Нет | Пользовательская короткая метка для навигационной цепочки (хлебных крошек) |
| `customThumbnailUrl` | Нет | Пользовательский URL миниатюры (по умолчанию используется миниатюра YouTube) |

#### Шаг 3: Добавьте расшифровку (рекомендуется) {#step-3}

Под frontmatter `---` добавьте текстовую расшифровку видео в формате markdown:

```markdown
---
title: "..."
# ... остальная часть frontmatter
---

Краткое введение в содержание видео.

### Название раздела (0:00)

Текст расшифровки для этого раздела...

### Следующий раздел (5:30)

Дополнительный текст расшифровки...
```

Используйте заголовки `###` с временными метками, чтобы отметить основные разделы. Это делает расшифровку удобной для беглого чтения и улучшает SEO.

Если у вас нет расшифровки, вы можете оставить тело документа пустым, и команда сгенерирует ее.

#### Шаг 4: Выберите тематические теги {#step-4}

Выберите тематические теги, которые соответствуют существующим категориям, используемым в галерее. Текущие категории и их теги включают:

- **Как работает Эфириум**: `how-ethereum-works`, `consensus`, `blockchain`, `cryptography`, `accounts`, `ethereum`, `intro`, `transactions`, `pos`, `smart-contracts`
- **Обновления сети**: `network-upgrades`, `upgrades`, `pectra`, `dencun`, `eip-4844`, `blobs`, `fusaka`
- **Дорожная карта и приоритеты**: `roadmap-and-priorities`, `pbs`, `mev`
- **Масштабирование и уровень 2 (l2)**: `scaling-and-layer-2`, `scaling`, `layer-2`, `rollups`, `optimistic-rollups`, `zk-rollups`
- **Варианты использования**: `use-cases`, `defi`, `finance`, `nfts`, `erc-721`, `erc-1155`, `lending`, `dapps`, `restaking`, `eigenlayer`, `dao`, `identity`, `desci`, `refi`
- **Приватность и безопасность**: `privacy-and-security`, `privacy`, `authentication`
- **Истории сообщества**: `community-stories`, `contributing`, `translations`, `community`

Чтобы ваше видео появилось в разделе категории галереи, включите хотя бы один ключевой тег категории (название жирным шрифтом в формате kebab-case, например, `use-cases` или `scaling-and-layer-2`). Видео без распознанного тега категории будут отображаться только в разделе «Все» и в результатах поиска.

Вы также можете использовать новые теги — они будут доступны для будущих группировок по категориям.

#### Шаг 5: Отправьте ваш PR {#step-5}

Откройте pull request с вашими изменениями в ветку `dev`. Команда рассмотрит вашу заявку и предоставит обратную связь.


## Обслуживание

Размещенные видео регулярно проверяются, чтобы убедиться, что они:

- По-прежнему соответствуют критериям размещения
- Содержат точную и актуальную информацию
- Имеют рабочие ссылки на хостинг/Ютуб

Если вы заметили проблему с размещенным видео, [создайте issue](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+✨,content+🖋️&template=suggest_video.yaml) или отправьте письмо на [website@ethereum.org](mailto:website@ethereum.org).
