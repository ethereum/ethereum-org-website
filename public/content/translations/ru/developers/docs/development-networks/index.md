При создании приложения для [Эфириума](/) со смарт-контрактами вы захотите запустить его в локальной сети, чтобы проверить, как оно работает, прежде чем развернуть его.

Подобно тому, как вы можете запустить локальный сервер на своем компьютере для веб-разработки, вы можете использовать сеть для разработки, чтобы создать локальный экземпляр блокчейна для тестирования вашего децентрализованного приложения (dapp). Эти сети для разработки Эфириума предоставляют функции, которые позволяют выполнять итерации намного быстрее, чем в публичной тестовой сети (например, вам не нужно заниматься получением ETH из крана тестовой сети).

## Предварительные требования {#prerequisites}

Вы должны понимать [основы стека Эфириума](/developers/docs/ethereum-stack/) и [сетей Эфириума](/developers/docs/networks/), прежде чем погружаться в сети для разработки.

## Что такое сеть для разработки? {#what-is-a-development-network}

Сети для разработки — это, по сути, клиенты Эфириума (реализации Эфириума), разработанные специально для локальной разработки.

**Почему бы просто не запустить стандартный узел Эфириума локально?**

Вы _могли бы_ [запустить узел](/developers/docs/nodes-and-clients/#running-your-own-node), но поскольку сети для разработки созданы специально для разработки, они часто оснащены такими удобными функциями, как:

- Детерминированное заполнение вашего локального блокчейна данными (например, учетными записями с балансами ETH)
- Мгновенное создание блоков с каждой полученной транзакцией, по порядку и без задержек
- Расширенные функции отладки и ведения журнала

## Доступные инструменты {#available-projects}

**Примечание**: Большинство [фреймворков для разработки](/developers/docs/frameworks/) включают встроенную сеть для разработки. Мы рекомендуем начать с фреймворка, чтобы [настроить локальную среду разработки](/developers/local-environment/).

### Hardhat Network {#hardhat-network}

Локальная сеть Эфириума, предназначенная для разработки. Она позволяет вам развернуть ваши контракты, запускать тесты и отлаживать код.

Hardhat Network встроена в Hardhat, среду разработки Эфириума для профессионалов.

- [Веб-сайт](https://hardhat.org/)
- [GitHub](https://github.com/NomicFoundation/hardhat)

### Локальные сигнальные цепочки {#local-beacon-chains}

Некоторые клиенты консенсуса имеют встроенные инструменты для запуска локальных сигнальных цепочек в целях тестирования. Доступны инструкции для Лайтхаус, Нимбус и Лодстар:

- [Локальная тестовая сеть с использованием Лодстар](https://chainsafe.github.io/lodestar/contribution/advanced-topics/setting-up-a-testnet#post-merge-local-testnet/)
- [Локальная тестовая сеть с использованием Лайтхаус](https://lighthouse-book.sigmaprime.io/setup.html#local-testnets)

### Публичные тестовые цепочки Эфириума {#public-beacon-testchains}

Существуют также две поддерживаемые публичные тестовые реализации Эфириума: Sepolia и Hoodi. Рекомендуемая тестовая сеть с долгосрочной поддержкой — Hoodi, в которой любой желающий может свободно выступать в роли валидатора. Sepolia использует набор валидаторов с разрешенным доступом, что означает отсутствие общего доступа для новых валидаторов в этой тестовой сети.

- [Панель запуска стейкинга Hoodi](https://hoodi.launchpad.ethereum.org/)

### Пакет Kurtosis для Эфириума {#kurtosis}

Kurtosis — это система сборки для многоконтейнерных тестовых сред, которая позволяет разработчикам локально запускать воспроизводимые экземпляры сетей блокчейна.

Пакет Kurtosis для Эфириума можно использовать для быстрого создания параметризуемой, хорошо масштабируемой и приватной тестовой сети Эфириума поверх Docker или Kubernetes. Пакет поддерживает все основные клиенты уровня исполнения (EL) и уровня консенсуса (CL). Kurtosis корректно обрабатывает все локальные сопоставления портов и подключения сервисов для репрезентативной сети, которая будет использоваться в рабочих процессах валидации и тестирования, связанных с базовой инфраструктурой Эфириума.

- [Сетевой пакет Эфириума](https://github.com/kurtosis-tech/ethereum-package)
- [Веб-сайт](https://www.kurtosis.com/)
- [GitHub](https://github.com/kurtosis-tech/kurtosis)
- [Документация](https://docs.kurtosis.com/)
