У каждого валидатора есть **реквизиты для вывода**, которые определяют, как и куда можно вывести застейканный ETH и вознаграждения. Тип реквизитов указывается первым байтом: `0x00`, `0x01` или `0x02`. Понимание этих типов важно для валидаторов, управляющих своим стейком.

## 0x00: Реквизиты до обновления Шапелла {#0x00-credentials}

Тип `0x00` — это исходный формат реквизитов для вывода, существовавший до обновления Шапелла (апрель 2023 года). У валидаторов с этим типом реквизитов не установлен адрес для вывода на уровне исполнения, что означает, что их средства остаются заблокированными на уровне консенсуса. Если у вас все еще есть реквизиты `0x00`, вы должны обновить их до `0x01` или `0x02`, прежде чем сможете получать какие-либо выводы.

## 0x01: Устаревшие реквизиты для вывода {#0x01-credentials}

Тип `0x01` был представлен в обновлении Шапелла и стал стандартом для валидаторов, которые хотели установить адрес для вывода на уровне исполнения. С реквизитами `0x01`:

- Любой баланс свыше 32 ETH **автоматически переводится** на ваш адрес для вывода
- Полные выходы проходят через стандартную очередь на выход
- Вознаграждения свыше 32 ETH не могут капитализироваться — они периодически выводятся

**Почему некоторые валидаторы все еще используют 0x01:** Это проще и привычнее. Многие валидаторы внесли депозит после обновления Шапелла и уже имеют этот тип, и он отлично работает для тех, кто хочет автоматического вывода избыточного баланса.

**Почему это не рекомендуется:** С `0x01` вы теряете возможность капитализировать вознаграждения свыше 32 ETH. Любой избыток автоматически выводится, что ограничивает потенциал заработка вашего валидатора и требует отдельного управления выведенными средствами.

## 0x02: Капитализируемые реквизиты для вывода {#0x02-credentials}

Тип `0x02` был представлен в обновлении Пектра и сегодня является **рекомендуемым выбором** для валидаторов. Валидаторов с реквизитами `0x02` иногда называют «капитализирующими валидаторами».

С реквизитами `0x02`:

- Вознаграждения свыше 32 ETH **капитализируются** с шагом в 1 ETH до максимального эффективного баланса в 2048 ETH
- Частичные выводы должны запрашиваться вручную (автоматические переводы происходят только при превышении порога в 2048 ETH)
- Валидаторы могут объединить несколько валидаторов по 32 ETH в одного валидатора с более высоким балансом
- Полные выходы по-прежнему поддерживаются через стандартную очередь на выход

Как частичные выводы, так и объединения могут быть выполнены через [действия валидатора на Launchpad](https://launchpad.ethereum.org/en/validator-actions).

**Почему валидаторам следует предпочесть 0x02:** Это обеспечивает лучшую эффективность капитала за счет капитализации, больший контроль над тем, когда происходят выводы, и поддерживает объединение валидаторов. Для соло-стейкеров, которые накапливают вознаграждения со временем, это означает, что их эффективный баланс — а следовательно, и их вознаграждения — могут расти сверх 32 ETH без ручного вмешательства.

**Важно:** Как только вы конвертируете `0x01` в `0x02`, вы не сможете сделать откат.

Для получения подробного руководства по переходу на реквизиты типа 2 и функции MaxEB см. [страницу с объяснением MaxEB](/roadmap/pectra/maxeb/).

## Что мне выбрать? {#what-should-i-pick}

- **Новые валидаторы:** Выбирайте `0x02`. Это современный стандарт с лучшей капитализацией и гибкостью.
- **Существующие валидаторы 0x01:** Рассмотрите возможность перехода на `0x02`, если вы хотите, чтобы вознаграждения капитализировались свыше 32 ETH, или планируете объединить валидаторов.
- **Существующие валидаторы 0x00:** Обновитесь немедленно — вы не сможете вывести средства без обновления ваших реквизитов. Сначала вы должны конвертировать их в `0x01`, а затем сможете конвертировать в `0x02`.

## Инструменты для управления реквизитами для вывода {#withdrawal-credential-tools}

Несколько инструментов поддерживают выбор или конвертацию между типами реквизитов:

- **[Ethereum Staking Launchpad](https://launchpad.ethereum.org/en/validator-actions)** — официальный инструмент для депозитов и управления валидаторами, включая конвертацию реквизитов и объединение
- **[Pectra Staking Manager](https://pectrastaking.com)** — веб-интерфейс с поддержкой подключения кошелька для конвертации и объединения
- **[Pectra Validator Ops CLI Tool](https://github.com/Luganodes/Pectra-Batch-Contract)** — инструмент командной строки для пакетной конвертации
- **[Ethereal](https://github.com/wealdtech/ethereal)** — инструмент командной строки для операций в Эфириуме, включая управление валидаторами

Полный список инструментов для объединения и подробные инструкции по конвертации см. в разделе [Инструменты объединения MaxEB](/roadmap/pectra/maxeb/#consolidation-tooling).
