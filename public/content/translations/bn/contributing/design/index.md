# ethereum.org-এ ডিজাইনে অবদান {#design-contributions}

ডিজাইন যেকোনো প্রজেক্টের একটি গুরুত্বপূর্ণ অংশ, এবং ethereum.org-এ আপনার সময় ও ডিজাইনের দক্ষতা কাজে লাগিয়ে আপনি আমাদের ভিজিটরদের ইউজার এক্সপেরিয়েন্স উন্নত করতে সাহায্য করতে পারেন। একটি ওপেন-সোর্স প্রজেক্টে অবদান রাখা প্রাসঙ্গিক অভিজ্ঞতা অর্জন এবং একটি সহযোগিতামূলক পরিবেশে আপনার দক্ষতা বিকাশের সুযোগ প্রদান করে। আপনি অন্যান্য ডিজাইনার, ডেভেলপার এবং কমিউনিটির সদস্যদের সাথে কাজ করার সুযোগ পাবেন, যাদের প্রত্যেকের নিজস্ব অনন্য দৃষ্টিভঙ্গি এবং অন্তর্দৃষ্টি রয়েছে।

পরিশেষে, এটি একটি বৈচিত্র্যময় এবং চিত্তাকর্ষক পোর্টফোলিও তৈরি করার একটি দুর্দান্ত উপায় যা আপনার ডিজাইনের দক্ষতা প্রদর্শন করে।

## কীভাবে অবদান রাখবেন?

### <Emoji text=":one:" size={1} /> &nbsp;প্রাথমিক ডিজাইন প্রোটোটাইপগুলোর উপর মতামত দিন {#design-critique}

আমাদের মাঝে মাঝে আমাদের প্রাথমিক ধারণাগুলো পরীক্ষা করার জন্য সাহায্যের প্রয়োজন হয়। কোনো প্রযুক্তিগত জ্ঞান ছাড়াই অবদান রাখার এটি একটি দুর্দান্ত উপায়।

1. ডিজাইন টিম [Discord](https://discord.com/invite/ethereum-org) এবং [GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8)-এ একটি মকআপ ডিজাইন শেয়ার করবে।
2. কমেন্ট ফাংশনের মাধ্যমে মতামত প্রদানের জন্য আপনাকে ডিজাইনগুলোর বিষয়ে গাইড করা হবে।
3. ফলাফলটি GitHub ইস্যুতে শেয়ার করা হবে এবং তারপর টিমের দ্বারা বন্ধ করা হবে।

### <Emoji text=":two:" size={1} /> &nbsp;জরিপ গবেষণায় অংশগ্রহণ করুন {#answer-surveys}

আমাদের ওয়েবসাইটে মতামত প্রদান করুন এভাবে:

1. ethereum.org ভিজিট করে এবং পেজগুলো পড়ে।
2. নিচের ডানদিকের কোণায় থাকা ফিডব্যাক উইজেটে ক্লিক করে এবং ডিজাইন ও কন্টেন্ট-সম্পর্কিত প্রশ্নের উত্তর দিয়ে।
3. ফ্রি ফরম্যাট প্রশ্নগুলোর উপর ফোকাস করুন।

### <Emoji text=":three:" size={1} /> &nbsp;ওয়েবসাইটে ডিজাইন সম্পর্কিত সমস্যাগুলো খুঁজুন এবং রিপোর্ট করুন {#report-design-issues}

ethereum.org অনেক ফিচার এবং কন্টেন্ট সহ একটি দ্রুত বর্ধনশীল ওয়েবসাইট। এর কিছু UI সহজেই সেকেলে হয়ে যেতে পারে বা উন্নত করা যেতে পারে। আপনি যদি এমন কোনো উদাহরণ দেখতে পান, তবে অনুগ্রহ করে এটি রিপোর্ট করুন যাতে এটি আমাদের নজরে আসে।

1. ওয়েবসাইটটি ঘুরে দেখুন এবং এর ডিজাইনের দিকে মনোযোগ দিন।
2. আপনি যদি কোনো ভিজ্যুয়াল বা UX সমস্যা দেখতে পান তবে স্ক্রিনশট এবং নোট নিন।
3. একটি [বাগ রিপোর্ট](https://github.com/ethereum/ethereum-org-website/issues/new/choose) ব্যবহার করে প্রাপ্ত সমস্যাগুলো রিপোর্ট করুন।

### <Emoji text=":four:" size={1} /> &nbsp;ডিজাইন পরিবর্তনের প্রস্তাব দিন {#propose-design-changes}

আপনি যদি ডিজাইনের চ্যালেঞ্জ নিতে স্বাচ্ছন্দ্যবোধ করেন, তবে আপনি আমাদের GitHub ইস্যু বোর্ডে যেতে পারেন এবং [ডিজাইন-সম্পর্কিত ইস্যুগুলোর](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) জন্য ফিল্টার করতে পারেন।

1. আমাদের ওয়েবসাইটটি ঘুরে দেখুন এবং এর ডিজাইনের দিকে মনোযোগ দিন অথবা আমাদের GitHub রিপোজিটরিতে যান এবং [“Design required” ট্যাগ](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) যুক্ত ইস্যুগুলো পর্যালোচনা করুন।
2. সমাধানের বিষয়ে ধারণা তৈরি করুন এবং এটি ডিজাইন করুন। (আদর্শভাবে আমাদের [ডিজাইন সিস্টেম](https://www.figma.com/community/file/1134414495420383395) ব্যবহার করে)।
3. সংশ্লিষ্ট GitHub ইস্যুতে সমাধানের প্রস্তাব দিন অথবা [একটি নতুন ইস্যু তৈরি করুন।](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. ডিজাইন টিমের পর্যালোচনার জন্য অপেক্ষা করুন।

### <Emoji text=":five:" size={1} /> &nbsp;একসাথে ডিজাইন সিস্টেম তৈরি করুন {#Contribute-to-design-system}

আমাদের ডিজাইন সিস্টেম ethereum.org-এর ডিজাইনকে মজাদার এবং সহজ করে তোলে। আপনি যদি একজন অভিজ্ঞ ডিজাইনার হন, তবে আপনি ওয়েবসাইটের জন্য অনেক কম্পোনেন্ট প্রস্তুত করতে আমাদের সাহায্য করতে পারেন।

1. GitHub-এ [ডিজাইন সিস্টেম বোর্ড](https://github.com/ethereum/ethereum-org-website/labels/design%20system) থেকে কাজ করার জন্য একটি ইস্যু নির্বাচন করুন অথবা একটি নতুন ইস্যু তৈরি করুন।
2. নির্বাচিত ইস্যুটি আপনাকে অ্যাসাইন করার জন্য অনুরোধ করুন।
3. Figma-তে অনুরোধ করা কম্পোনেন্টটির ডিজাইন শুরু করুন।
4. আপনার যখন পর্যালোচনা বা নির্দেশনার প্রয়োজন হবে তখন এটি GitHub-এ ডিজাইন টিমের সাথে শেয়ার করুন।
5. ডিজাইন টিম এটি পর্যালোচনা করবে।
6. ডিজাইন টিম মূল ফাইলে পরিবর্তনগুলো অন্তর্ভুক্ত করবে এবং ফাইলটি কমিউনিটির জন্য প্রকাশ করবে।

### <Emoji text=":six:" size={1} /> &nbsp;ওয়েবসাইটে ডিজাইন-সম্পর্কিত কন্টেন্ট লিখুন {#write-design-articles}

ইথিরিয়াম ডেভেলপার কমিউনিটি শক্তিশালী, কিন্তু ডিজাইন কমিউনিটি কিছুটা পিছিয়ে পড়ছে। আপনি যদি ওয়েব3 জ্ঞানসম্পন্ন একজন ডিজাইনার হন, তবে অনুগ্রহ করে বৃহত্তর কমিউনিটির সাথে আপনার শিক্ষা শেয়ার করার কথা বিবেচনা করুন যাতে আমরা সবাই একসাথে বেড়ে উঠতে এবং উন্নতি করতে পারি; আমাদের [ইথিরিয়ামের জন্য ডিজাইন করার উপর একটি পেজ](/developers/docs/design-and-ux/) রয়েছে যেখানে আপনি অবদান রাখতে পারেন। আপনি আমাদের [লিস্টিং পলিসিগুলোও](/contributing/design/adding-design-resources) দেখতে পারেন।

1. ethereum.org-এ কভার করা উচিত এবং এই স্পেসের ডিজাইনারদের জন্য উপকারী হবে এমন ডিজাইন টপিকগুলোর বিষয়ে ধারণা তৈরি করুন।
2. আমাদের GitHub রিপোজিটরিতে যান এবং একটি টপিকের প্রস্তাব দিয়ে [একটি ইস্যু তৈরি করুন](https://github.com/ethereum/ethereum-org-website/issues/new) (এখনই কন্টেন্ট লিখবেন না)।
3. ডিজাইন টিমের অনুমোদনের জন্য অপেক্ষা করুন।
4. অনুমোদিত হলে, কন্টেন্টটি লিখুন।
5. এটি সংশ্লিষ্ট GitHub ইস্যুতে জমা দিন।

### <Emoji text=":seven:" size={1} /> &nbsp;নতুন ইলাস্ট্রেশন আঁকুন {#prepare-illustrations}

বিমূর্ত বিষয়গুলো ব্যাখ্যা করার জন্য ভিজ্যুয়ালাইজেশন হলো অন্যতম শক্তিশালী টুল। ডায়াগ্রাম এবং ইনফোগ্রাফিক্স যুক্ত করার মাধ্যমে বিশাল সম্ভাবনা রয়েছে। সর্বোপরি, একটি ছবি হাজার শব্দের সমান কথা বলতে পারে।

1. আমাদের ওয়েবসাইটে যান এবং এমন পেজগুলো দেখুন যেখানে কিছু নতুন ইনফোগ্রাফিক্স যুক্ত করা যেতে পারে।
2. নিশ্চিত করুন যে ইলাস্ট্রেশনের স্টাইলটি আমাদের [অ্যাসেটগুলোর](/assets/) সাথে সামঞ্জস্যপূর্ণ।
3. আমাদের GitHub রিপোজিটরিতে যান এবং ইলাস্ট্রেশনের প্রস্তাব দিয়ে [একটি ইস্যু তৈরি করুন](https://github.com/ethereum/ethereum-org-website/issues/new)।
4. ডিজাইন টিম এটি পর্যালোচনা করবে।
5. নতুন ছবিটি বাস্তবায়ন করার জন্য একজন ডেভেলপারকে বলার উদ্দেশ্যে আমরা একটি নতুন ইস্যু তৈরি করি।
