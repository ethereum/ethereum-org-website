## Beacon Chain என்றால் என்ன? {#what-is-the-beacon-chain}

Beacon Chain என்பது 2020 ஆம் ஆண்டு அறிமுகப்படுத்தப்பட்ட மூல proof-of-stake blockchain ஆகும். இது Ethereum Mainnet இல் proof-of-stake செயல்பாட்டை இயக்கு முன்னர், அதன் தரத்தை மற்றும் நிலைத்தன்மையை உறுதி செய்ய உருவாக்கப்பட்டது. அதனால், இது ஆரம்பத்தில் proof-of-work Ethereum உடன் இணைந்து இயங்கியது. Beacon Chain தொடக்கத்தில் “காலி” (empty) பிளாக்கள் கொண்டது.Ethereum இல் proof-of-work நிறுத்தி proof-of-stake இயக்கவேண்டும் என்றால், Beacon Chain க்கு transaction தரவை execution clients இருந்து ஏற்றவும், பிளாக்களில் தொகுத்து, proof-of-stake அடிப்படையிலான consensus முறையில் blockchain ஆக அமைக்கவும் உத்தரவிட வேண்டும். அதே நேரத்தில், பழைய Ethereum clients தங்கள் mining, block propagation மற்றும் consensus logic ஐ நிறுத்தி, அனைத்தையும் Beacon Chain க்கு மாற்றின. இந்த நிகழ்வு [தி மெர்ஜ்](/roadmap/merge/) என்று அறியப்பட்டது. The Merge நடக்கும்போது இரண்டு blockchain களும் இருந்தது இல்லாமல், ஒரே proof-of-stake Ethereum உருவானது. இப்போது ஒவ்வொரு node க்கும் இரண்டு விதமான clients தேவை. Consensus layer (Beacon Chain) – block gossip மற்றும் consensus logic களை கையாளும் peer-to-peer network.Execution layer (மூல Ethereum clients) – transactions ஐ execute செய்யும், gossip செய்யும் மற்றும் Ethereum state ஐ நிர்வகிக்கும். இரு layer களும் Engine API மூலம் பரஸ்பரம் தொடர்பு கொள்ள முடியும்.

## Beacon Chain என்ன செய்கிறது? பீக்கன் செயின் என்ன செய்கிறது? {#what-does-the-beacon-chain-do}

பீக்கன் செயின் என்பது, எத்தேரியம் [ஸ்டேக்கர்கள்](/staking/) உண்மையான எத்தேரியம் பிளாக்குகளைச் சரிபார்க்கத் தொடங்குவதற்கு முன்பு, அந்த ஸ்டேக்கர்களின் நெட்வொர்க்கை நடத்தி ஒருங்கிணைத்த கணக்குகளின் பேரேட்டிற்கு (ledger) வழங்கப்படும் பெயர். இது பரிவர்த்தனைகளை செயல்படுத்தவோ அல்லது ஸ்மார்ட் கன்ட்ராக்ட் தொடர்புகளை கையாளவோ செய்யாது, ஏனெனில் அது எக்சிகியூஷன் லேயரில் செய்யப்படுகிறது.
பீகன் செயின் பிளாக் மற்றும் அட்டெஸ்டேஷன் கையாளுதல், ஃபோர்க் சாய்ஸ் ஆல்கொரிதம் இயக்குதல் மற்றும் விருதுகள் மற்றும் தண்டனைகளை நிர்வகிப்பது போன்றவற்றிற்கே பொறுப்பானது.
எங்கள் [முனைக் கட்டமைப்புப் பக்கத்தில்](/developers/docs/nodes-and-clients/node-architecture/#node-comparison) மேலும் படிக்கவும்.

## பீக்கன் செயின் தாக்கம் {#beacon-chain-features}

### ஸ்டேக்கிங்கை அறிமுகப்படுத்துதல் {#introducing-staking}

பீக்கன் செயின், எத்தேரியத்திற்கு [ப்ரூஃப்-ஆஃப்-ஸ்டேக்கை](/developers/docs/consensus-mechanisms/pos/) அறிமுகப்படுத்தியது. இது எதீரியத்தை பாதுகாப்பாக வைக்க உதவுகிறது மற்றும் வேலிடேட்டர்கள் கூட அதிக ETH சம்பாதிக்க உதவுகிறது. நடைமுறையில், ஸ்டேக்கிங் என்பது புதிய பிளாக்களை உருவாக்கும் மற்றும் சரிபார்க்கும் வெலிடேட்டர் சாப்ட்வேர் இயக்குவதற்காக ETH-ஐ ஸ்டேக் செய்யவேண்டும். ஸ்டேக்கிங் மைனிங்கால் முன்பு செய்யப்பட்ட செயலுக்குச் சமமான நோக்கத்தைக் கொண்டது, ஆனால் பல விதங்களில் வேறுபடுகிறது.

ஸ்டேக்கிங், [மைனிங்கால்](/developers/docs/consensus-mechanisms/pow/mining/) முன்பு செய்யப்பட்ட செயலுக்குச் சமமான நோக்கத்தைக் கொண்டது, ஆனால் பல விதங்களில் வேறுபடுகிறது. மைனிங் அதிக அளவிலான ஹார்ட்வேர் மற்றும் எரிசக்தி பயன்பாட்டை தேவையாக்கியது, இது பொருளாதார அளவுகளில் மையமாக்கலுக்கு வழிவகுத்தது. மேலும், மைனிங் எந்தவொரு சொத்து பிடித்தையும் பூட்டி வைக்கவில்லை, ஆகையால் ஒரு தாக்கத்துக்குப் பிறகு தீய நடத்தை காட்டும்வர்களை தண்டிக்க முடிவில்லை.

ப்ரூஃப்-ஆஃப்-ஸ்டேக்குக்கு மாறுவதால், எதீரியம் ப்ரூஃப்-ஆஃப்-வொர்க்குடன் ஒப்பிடும்போது, முக்கியமாக பாதுகாப்பானதும் மையமற்றதும் ஆனது. நெட்வொர்க்கில் அதிக பேர் பங்கேற்கும் போது, அது மேலும் மையமற்றதும் தாக்குதல்களில் இருந்து பாதுகாப்பானதும் ஆகிறது.

### ஷார்டிங்கிற்காக அமைத்தல் {#setting-up-for-sharding}

பீகன் செயின் மொத்த எதீரியம் மெய்ன்நெட்டுடன் இணைந்த பிறகு, எதீரியம் சமூகமே நெட்வொர்க் ஸ்கேலிங்கைப் பற்றி ஆராய தொடங்கியது.

ப்ரூஃப்-ஆஃப்-ஸ்டேக்கிற்கு ஒரு நன்மை, ஒவ்வொரு தருணத்திலும் ETH-ஐ ஸ்டேக் செய்த அனைத்து அங்கீகாரம் பெற்ற பிளாக் தயாரிப்பாளர்களின் பதிவேட்டை வைத்திருப்பதாகும். இந்த பதிவேடு, குறிப்பிட்ட நெட்வொர்க் பொறுப்புகளை நம்பகமாக பிரித்து கையாளும் திறனை உருவாக்குகிறது.

இதன் பொறுப்பு ப்ரூஃப்-ஆஃப்-வொர்க் முறைமையுடன் மாறுபடுகிறது, அங்கு மைனர்கள் நெட்வொர்க்கிற்கு எந்தவொரு கடமையும் இல்லாமல், விரைவில் மைனிங் நிறுத்தி, தங்களது நோட் சாப்ட்வேரை நிரந்தரமாக அணைத்து விடலாம். இதற்கு எந்த தண்டனையும் இல்லை. மேலும், பறந்த பிளாக் முன்மொழியுநர்களின் பதிவு இல்லை, மற்றும் நெட்வொர்க் பொறுப்புகளை பாதுகாப்பாக பிரிக்க நம்பகமான வழி கிடையாது.

## மேம்பாடுகளுக்கு இடையிலான உறவு {#relationship-between-upgrades}

எதீரியம் மேம்பாடுகள் அனைத்தும் ஒன்றிற்கு ஒன்று தொடர்புடையவையாக உள்ளன. எனவே, பீகன் செயின் மற்ற மேம்பாடுகளுக்கு எவ்வாறு பாதிப்பை ஏற்படுத்துகிறது என்பதை மீண்டும் பார்ப்போம். எனவே, பீகன் செயின் மற்ற மேம்பாடுகளுக்கு எவ்வாறு பாதிப்பை ஏற்படுத்துகிறது என்பதை மீண்டும் பார்ப்போம்.
