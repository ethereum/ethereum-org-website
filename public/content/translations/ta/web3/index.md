# Web3 அறிமுகம் {#introduction}

<div className="mt-4">
<ListenToPlayer slug="/web3/" />
</div>

மையப்படுத்தல் (Centralization) கோடிக்கணக்கான மக்களை World Wide Web-இல் இணைக்க உதவியுள்ளதுடன், அது இயங்கும் நிலையான, வலுவான உள்கட்டமைப்பையும் உருவாக்கியுள்ளது. அதே நேரத்தில், ஒரு சில மையப்படுத்தப்பட்ட நிறுவனங்கள் World Wide Web-இன் பெரும் பகுதிகளைத் தங்கள் கட்டுப்பாட்டில் வைத்துள்ளன, எதை அனுமதிக்க வேண்டும், எதை அனுமதிக்கக் கூடாது என்பதை ஒருதலைப்பட்சமாக முடிவு செய்கின்றன.

இந்தச் சிக்கலுக்கு Web3 தான் பதில். பெரிய தொழில்நுட்ப நிறுவனங்களால் ஏகபோகமாக ஆளப்படும் Web-க்கு பதிலாக, Web3 பரவலாக்கலை (decentralization) ஏற்றுக்கொள்கிறது, மேலும் இது அதன் பயனர்களால் உருவாக்கப்பட்டு, இயக்கப்பட்டு, சொந்தமாக்கப்பட்டுள்ளது. Web3 அதிகாரத்தை நிறுவனங்களின் கைகளில் அல்லாமல் தனிநபர்களின் கைகளில் கொடுக்கிறது.
Web3 பற்றிப் பேசுவதற்கு முன், நாம் எப்படி இங்கு வந்தோம் என்பதை ஆராய்வோம்.

## ஆரம்பகால Web {#early-internet}

பெரும்பாலான மக்கள் Web-ஐ நவீன வாழ்க்கையின் தொடர்ச்சியான தூணாகவே கருதுகிறார்கள்—அது கண்டுபிடிக்கப்பட்டது, அன்றிலிருந்து அப்படியே இருக்கிறது என்று நினைக்கிறார்கள். இருப்பினும், இன்று நாம் அறிந்த Web, முதலில் கற்பனை செய்யப்பட்டதிலிருந்து முற்றிலும் மாறுபட்டது. இதைப் பற்றி நன்கு புரிந்துகொள்ள, Web-இன் குறுகிய வரலாற்றை Web 1.0 மற்றும் Web 2.0 எனப் பிரிப்பது உதவியாக இருக்கும்.

### Web 1.0: படிக்க-மட்டும் (Read-Only) (1990-2004) {#web1}

1989-இல், ஜெனீவாவில் உள்ள CERN-இல், Tim Berners-Lee World Wide Web ஆக மாறவிருக்கும் நெறிமுறைகளை (protocols) உருவாக்குவதில் மும்முரமாக இருந்தார். அவரது யோசனை என்ன? பூமியில் எங்கிருந்தும் தகவல்களைப் பகிர்ந்துகொள்ள அனுமதிக்கும் திறந்த, பரவலாக்கப்பட்ட நெறிமுறைகளை உருவாக்குவது.

Berners-Lee-இன் படைப்பின் முதல் தொடக்கம், இப்போது 'Web 1.0' என்று அழைக்கப்படுகிறது, இது தோராயமாக 1990 முதல் 2004 வரை நிகழ்ந்தது. Web 1.0 என்பது முக்கியமாக நிறுவனங்களுக்குச் சொந்தமான நிலையான (static) இணையதளங்களாக இருந்தது, மேலும் பயனர்களிடையே கிட்டத்தட்ட எந்தத் தொடர்பும் இல்லை - தனிநபர்கள் எப்போதாவதுதான் உள்ளடக்கத்தை உருவாக்கினர் - இதனால் இது படிக்க-மட்டும் (read-only) web என்று அழைக்கப்பட்டது.

![Web 1.0-ஐக் குறிக்கும் கிளையண்ட்-சர்வர் கட்டமைப்பு](./web1.png)

### Web 2.0: படிக்க-எழுத (Read-Write) (2004-இப்போது வரை) {#web2}

சமூக ஊடகத் தளங்களின் வருகையுடன் Web 2.0 காலகட்டம் 2004-இல் தொடங்கியது. படிக்க-மட்டும் என்பதற்குப் பதிலாக, web படிக்க-எழுத (read-write) எனப் பரிணாமம் அடைந்தது. நிறுவனங்கள் பயனர்களுக்கு உள்ளடக்கத்தை வழங்குவதற்குப் பதிலாக, பயனர்கள் உருவாக்கும் உள்ளடக்கத்தைப் பகிரவும், பயனர்களுக்கிடையேயான தொடர்புகளில் ஈடுபடவும் தளங்களை வழங்கத் தொடங்கின. அதிகமான மக்கள் ஆன்லைனில் வந்ததால், ஒரு சில முன்னணி நிறுவனங்கள் web-இல் உருவாக்கப்படும் ட்ராஃபிக் மற்றும் மதிப்பின் பெரும் பகுதியைக் கட்டுப்படுத்தத் தொடங்கின. Web 2.0 விளம்பர அடிப்படையிலான வருவாய் மாதிரியையும் உருவாக்கியது. பயனர்கள் உள்ளடக்கத்தை உருவாக்க முடிந்தாலும், அவர்கள் அதைச் சொந்தமாக்கிக் கொள்ளவோ அல்லது அதன் மூலம் பணம் சம்பாதிப்பதன் மூலம் பயனடையவோ இல்லை.

![Web 2.0-ஐக் குறிக்கும் கிளையண்ட்-சர்வர் கட்டமைப்பு](./web2.png)

## Web 3.0: படிக்க-எழுத-சொந்தமாக்க (Read-Write-Own) {#web3}

'Web 3.0' என்ற கருத்தாக்கம் 2014-இல் [Ethereum](/) தொடங்கப்பட்ட சிறிது நேரத்திலேயே அதன் இணை நிறுவனர் Gavin Wood என்பவரால் உருவாக்கப்பட்டது. பல ஆரம்பகால கிரிப்டோ பயனர்கள் உணர்ந்த ஒரு பிரச்சனைக்கு Gavin ஒரு தீர்வை வார்த்தைகளில் கூறினார்: Web-க்கு அதிகப்படியான நம்பிக்கை தேவைப்பட்டது. அதாவது, இன்று மக்கள் அறிந்த மற்றும் பயன்படுத்தும் பெரும்பாலான Web, பொதுமக்களின் நலனுக்காகச் செயல்படும் ஒரு சில தனியார் நிறுவனங்களை நம்பியிருப்பதையே அடிப்படையாகக் கொண்டுள்ளது.

![Web3-ஐக் குறிக்கும் பரவலாக்கப்பட்ட நோடு கட்டமைப்பு](./web3.png)

### Web3 என்றால் என்ன? {#what-is-web3}

Web3 என்பது புதிய, சிறந்த இணையத்தின் பார்வைக்கான ஒரு பொதுவான சொல்லாக மாறிவிட்டது. அடிப்படையில், Web3 பிளாக்செயின்கள், கிரிப்டோகரன்சிகள் மற்றும் NFT-களைப் பயன்படுத்தி பயனர்களுக்கு உரிமையின் வடிவத்தில் அதிகாரத்தைத் திரும்ப அளிக்கிறது. [Twitter-இல் 2020-இல் வெளியான ஒரு பதிவு](https://twitter.com/himgajria/status/1266415636789334016) இதைச் சிறப்பாகக் கூறியது: Web1 படிக்க-மட்டும் (read-only), Web2 படிக்க-எழுத (read-write), Web3 படிக்க-எழுத-சொந்தமாக்க (read-write-own) என்பதாக இருக்கும்.

#### Web3-இன் முக்கியக் கருத்துகள் {#core-ideas}

Web3 என்றால் என்ன என்பதற்கு ஒரு உறுதியான வரையறையை வழங்குவது சவாலானது என்றாலும், சில அடிப்படைக் கொள்கைகள் அதன் உருவாக்கத்தை வழிநடத்துகின்றன.

- **Web3 பரவலாக்கப்பட்டது (decentralized):** இணையத்தின் பெரும் பகுதிகள் மையப்படுத்தப்பட்ட நிறுவனங்களால் கட்டுப்படுத்தப்பட்டு சொந்தமாக்கப்படுவதற்குப் பதிலாக, உரிமை அதன் உருவாக்குநர்கள் மற்றும் பயனர்களிடையே விநியோகிக்கப்படுகிறது.
- **Web3 அனுமதியற்றது (permissionless):** Web3-இல் பங்கேற்க அனைவருக்கும் சமமான அணுகல் உள்ளது, மேலும் யாரும் ஒதுக்கப்படுவதில்லை.
- **Web3 நேட்டிவ் பேமெண்ட்களைக் கொண்டுள்ளது:** வங்கிகள் மற்றும் பேமெண்ட் ப்ராசஸர்களின் காலாவதியான உள்கட்டமைப்பை நம்பியிருப்பதற்குப் பதிலாக, ஆன்லைனில் பணத்தைச் செலவழிக்கவும் அனுப்பவும் இது கிரிப்டோகரன்சியைப் பயன்படுத்துகிறது.
- **Web3 நம்பிக்கையற்றது (trustless):** நம்பகமான மூன்றாம் தரப்பினரை நம்பியிருப்பதற்குப் பதிலாக, இது ஊக்கத்தொகைகள் மற்றும் பொருளாதார வழிமுறைகளைப் பயன்படுத்திச் செயல்படுகிறது.

### Web3 ஏன் முக்கியமானது? {#why-is-web3-important}

Web3-இன் சிறந்த அம்சங்கள் தனிமைப்படுத்தப்பட்டவை அல்ல, மேலும் அவை நேர்த்தியான வகைகளுக்குள் பொருந்தாது என்றாலும், எளிமைக்காக அவற்றைப் புரிந்துகொள்வதை எளிதாக்க நாங்கள் அவற்றைப் பிரிக்க முயன்றுள்ளோம்.

#### உரிமை (Ownership) {#ownership}

Web3 உங்கள் டிஜிட்டல் சொத்துகளின் உரிமையை முன்னோடியில்லாத வகையில் உங்களுக்கு வழங்குகிறது. எடுத்துக்காட்டாக, நீங்கள் ஒரு web2 கேம் விளையாடுகிறீர்கள் என்று வைத்துக்கொள்வோம். நீங்கள் கேமில் ஒரு பொருளை வாங்கினால், அது நேரடியாக உங்கள் கணக்குடன் இணைக்கப்படும். கேம் உருவாக்குநர்கள் உங்கள் கணக்கை நீக்கினால், நீங்கள் இந்தப் பொருட்களை இழப்பீர்கள். அல்லது, நீங்கள் கேம் விளையாடுவதை நிறுத்தினால், உங்கள் கேம் பொருட்களில் நீங்கள் முதலீடு செய்த மதிப்பை இழப்பீர்கள்.

Web3 [பூஞ்சையற்ற டோக்கன்கள் (NFTs)](/glossary/#nft) மூலம் நேரடி உரிமையை அனுமதிக்கிறது. கேமின் உருவாக்குநர்கள் உட்பட யாருக்கும் உங்கள் உரிமையைப் பறிக்க அதிகாரம் இல்லை. மேலும், நீங்கள் விளையாடுவதை நிறுத்தினால், உங்கள் கேம் பொருட்களை திறந்த சந்தைகளில் விற்கலாம் அல்லது வர்த்தகம் செய்யலாம் மற்றும் அவற்றின் மதிப்பைத் திரும்பப் பெறலாம். இது எப்படிச் செயல்படுகிறது என்பதைப் பார்க்க [ஆன்செயின் கேமிங்கை (onchain gaming)](/gaming/) ஆராயுங்கள்.

#### தணிக்கை எதிர்ப்பு (Censorship resistance) {#censorship-resistance}

தளங்களுக்கும் உள்ளடக்க உருவாக்குநர்களுக்கும் இடையிலான அதிகாரப் பகிர்வு பெருமளவில் சமநிலையற்றதாக உள்ளது.

OnlyFans என்பது 1 மில்லியனுக்கும் அதிகமான உள்ளடக்க உருவாக்குநர்களைக் கொண்ட பயனர்களால் உருவாக்கப்பட்ட வயது வந்தோருக்கான உள்ளடக்கத் தளமாகும், இவர்களில் பலர் இந்தத் தளத்தைத் தங்கள் முதன்மை வருமான ஆதாரமாகப் பயன்படுத்துகின்றனர். ஆகஸ்ட் 2021-இல், OnlyFans வெளிப்படையான பாலியல் உள்ளடக்கத்தைத் தடை செய்யும் திட்டங்களை அறிவித்தது. இந்த அறிவிப்பு தளத்தில் உள்ள உருவாக்குநர்களிடையே சீற்றத்தைத் தூண்டியது, அவர்கள் உருவாக்க உதவிய ஒரு தளத்தில் தங்களின் வருமானம் பறிக்கப்படுவதாக உணர்ந்தனர். எதிர்ப்புகளுக்குப் பிறகு, முடிவு விரைவாக மாற்றப்பட்டது. உருவாக்குநர்கள் இந்தப் போரில் வெற்றி பெற்றாலும், இது Web 2.0 உருவாக்குநர்களுக்கான ஒரு சிக்கலை எடுத்துக்காட்டுகிறது: நீங்கள் ஒரு தளத்தை விட்டு வெளியேறினால், நீங்கள் சேர்த்த நற்பெயரையும் பின்தொடர்பவர்களையும் இழக்கிறீர்கள்.

Web3-இல், உங்கள் தரவு பிளாக்செயினில் வாழ்கிறது. நீங்கள் ஒரு தளத்தை விட்டு வெளியேற முடிவு செய்யும் போது, உங்கள் நற்பெயரை உங்களுடன் எடுத்துச் செல்லலாம், அதை உங்கள் மதிப்புகளுடன் மிகவும் தெளிவாக ஒத்துப்போகும் மற்றொரு இடைமுகத்தில் (interface) செருகலாம்.

Web 2.0-இல் உள்ளடக்க உருவாக்குநர்கள் விதிகளை மாற்ற மாட்டார்கள் என்று தளங்களை நம்ப வேண்டும், ஆனால் தணிக்கை எதிர்ப்பு என்பது Web3 தளத்தின் இயல்பான அம்சமாகும்.

#### பரவலாக்கப்பட்ட தன்னாட்சி அமைப்புகள் (DAOs) {#daos}

Web3-இல் உங்கள் தரவைச் சொந்தமாக்கிக் கொள்வதுடன், ஒரு நிறுவனத்தில் உள்ள பங்குகளைப் போலச் செயல்படும் டோக்கன்களைப் பயன்படுத்தி, தளத்தை ஒரு கூட்டமைப்பாகவும் நீங்கள் சொந்தமாக்கிக் கொள்ளலாம். ஒரு தளத்தின் பரவலாக்கப்பட்ட உரிமையை ஒருங்கிணைக்கவும், அதன் எதிர்காலம் குறித்த முடிவுகளை எடுக்கவும் DAO-கள் உங்களை அனுமதிக்கின்றன.

DAO-கள் தொழில்நுட்ப ரீதியாக ஒப்புக்கொள்ளப்பட்ட [ஸ்மார்ட் ஒப்பந்தங்கள் (smart contracts)](/glossary/#smart-contract) என வரையறுக்கப்படுகின்றன, அவை வளங்களின் (டோக்கன்கள்) தொகுப்பின் மீதான பரவலாக்கப்பட்ட முடிவெடுப்பதை தானியங்குபடுத்துகின்றன. டோக்கன்களைக் கொண்ட பயனர்கள் வளங்கள் எவ்வாறு செலவிடப்படுகின்றன என்பது குறித்து வாக்களிக்கின்றனர், மேலும் குறியீடு தானாகவே வாக்களிப்பின் முடிவைச் செயல்படுத்துகிறது.

இருப்பினும், மக்கள் பல Web3 சமூகங்களை DAO-கள் என்று வரையறுக்கின்றனர். இந்தச் சமூகங்கள் அனைத்தும் குறியீட்டின் மூலம் வெவ்வேறு நிலைகளில் பரவலாக்கல் மற்றும் ஆட்டோமேஷனைக் கொண்டுள்ளன. தற்போது, DAO-கள் என்றால் என்ன, அவை எதிர்காலத்தில் எவ்வாறு உருவாகலாம் என்பதை நாங்கள் ஆராய்ந்து வருகிறோம்.

### அடையாளம் (Identity) {#identity}

பாரம்பரியமாக, நீங்கள் பயன்படுத்தும் ஒவ்வொரு தளத்திற்கும் ஒரு கணக்கை உருவாக்குவீர்கள். எடுத்துக்காட்டாக, உங்களிடம் Twitter கணக்கு, YouTube கணக்கு மற்றும் Reddit கணக்கு இருக்கலாம். உங்கள் காட்சிப் பெயர் (display name) அல்லது சுயவிவரப் படத்தைய (profile picture) மாற்ற வேண்டுமா? நீங்கள் அதை ஒவ்வொரு கணக்கிலும் செய்ய வேண்டும். சில சந்தர்ப்பங்களில் நீங்கள் சமூக உள்நுழைவுகளைப் (social sign-ins) பயன்படுத்தலாம், ஆனால் இது ஒரு பழக்கமான சிக்கலை முன்வைக்கிறது—தணிக்கை. ஒரே கிளிக்கில், இந்தத் தளங்கள் உங்கள் முழு ஆன்லைன் வாழ்க்கையிலிருந்தும் உங்களை வெளியேற்ற முடியும். இன்னும் மோசமான விஷயம் என்னவென்றால், ஒரு கணக்கை உருவாக்க தனிப்பட்ட முறையில் அடையாளம் காணக்கூடிய தகவல்களுடன் நீங்கள் அவர்களை நம்ப வேண்டும் என்று பல தளங்கள் கோருகின்றன.

Ethereum முகவரி மற்றும் [Ethereum Name Service (ENS)](/glossary/#ens) சுயவிவரத்துடன் உங்கள் டிஜிட்டல் அடையாளத்தைக் கட்டுப்படுத்த அனுமதிப்பதன் மூலம் Web3 இந்தச் சிக்கல்களைத் தீர்க்கிறது. Ethereum முகவரியைப் பயன்படுத்துவது, பாதுகாப்பான, தணிக்கை-எதிர்ப்பு மற்றும் அநாமதேயமான தளங்கள் முழுவதும் ஒற்றை உள்நுழைவை (single login) வழங்குகிறது.

### நேட்டிவ் பேமெண்ட்கள் (Native payments) {#native-payments}

Web2-இன் பேமெண்ட் உள்கட்டமைப்பு வங்கிகள் மற்றும் பேமெண்ட் ப்ராசஸர்களை நம்பியுள்ளது, இது வங்கிக் கணக்குகள் இல்லாதவர்கள் அல்லது தவறான நாட்டின் எல்லைகளுக்குள் வசிப்பவர்களைத் தவிர்க்கிறது.
Web3 பிரவுசரில் நேரடியாகப் பணத்தை அனுப்ப [ETH](/glossary/#ether) போன்ற டோக்கன்களைப் பயன்படுத்துகிறது மற்றும் நம்பகமான மூன்றாம் தரப்பினர் யாரும் தேவையில்லை.

## Web3 வரம்புகள் {#web3-limitations}

Web3 அதன் தற்போதைய வடிவத்தில் பல நன்மைகளைக் கொண்டிருந்தாலும், அது செழிக்க சுற்றுச்சூழல் அமைப்பு (ecosystem) தீர்க்க வேண்டிய பல வரம்புகள் இன்னும் உள்ளன.

### அணுகல்தன்மை (Accessibility) {#accessibility}

Sign-in with Ethereum போன்ற முக்கியமான Web3 அம்சங்கள், எவரும் பூஜ்ஜியச் செலவில் பயன்படுத்த ஏற்கனவே கிடைக்கின்றன. ஆனால், பரிவர்த்தனைகளின் ஒப்பீட்டுச் செலவு இன்னும் பலருக்குத் தடையாக உள்ளது. அதிகப் பரிவர்த்தனைக் கட்டணங்கள் காரணமாக, குறைந்த செல்வம் கொண்ட, வளரும் நாடுகளில் Web3 பயன்படுத்தப்படுவதற்கான வாய்ப்புகள் குறைவு. Ethereum-இல், இந்தச் சவால்கள் [வழிகாட்டி (roadmap)](/roadmap/) மற்றும் [லேயர் 2 ஸ்கேலிங் தீர்வுகள் (layer 2 scaling solutions)](/glossary/#layer-2) மூலம் தீர்க்கப்படுகின்றன. தொழில்நுட்பம் தயாராக உள்ளது, ஆனால் Web3-ஐ அனைவருக்கும் அணுகக்கூடியதாக மாற்ற லேயர் 2-இல் அதிக அளவிலான தத்தெடுப்பு (adoption) நமக்குத் தேவை.

### பயனர் அனுபவம் (User experience) {#user-experience}

Web3-ஐப் பயன்படுத்துவதற்கான தொழில்நுட்ப நுழைவுத் தடை தற்போது மிக அதிகமாக உள்ளது. பயனர்கள் பாதுகாப்பு கவலைகளைப் புரிந்துகொள்ள வேண்டும், சிக்கலான தொழில்நுட்ப ஆவணங்களைப் புரிந்துகொள்ள வேண்டும் மற்றும் உள்ளுணர்வு இல்லாத பயனர் இடைமுகங்களை (unintuitive user interfaces) வழிநடத்த வேண்டும். குறிப்பாக, [வாலட் வழங்குநர்கள்](/wallets/find-wallet/) இதைத் தீர்க்கப் பணியாற்றி வருகின்றனர், ஆனால் Web3 பெருமளவில் ஏற்றுக்கொள்ளப்படுவதற்கு முன்பு இன்னும் அதிக முன்னேற்றம் தேவை.

### கல்வி (Education) {#education}

Web3 புதிய முன்னுதாரணங்களை அறிமுகப்படுத்துகிறது, இதற்கு Web2.0-இல் பயன்படுத்தப்பட்டதை விட வித்தியாசமான மன மாதிரிகளைக் (mental models) கற்றுக்கொள்ள வேண்டும். 1990-களின் பிற்பகுதியில் Web1.0 பிரபலமடைந்து வந்தபோது இதேபோன்ற ஒரு கல்வி உந்துதல் நிகழ்ந்தது; world wide web-இன் ஆதரவாளர்கள் எளிய உருவகங்கள் (தகவல் நெடுஞ்சாலை, பிரவுசர்கள், இணையத்தில் உலாவுதல்) முதல் [தொலைக்காட்சி ஒளிபரப்புகள்](https://www.youtube.com/watch?v=SzQLI7BxfYI) வரை பொதுமக்களுக்குக் கற்பிக்கப் பல கல்வி நுட்பங்களைப் பயன்படுத்தினர். Web3 கடினமானது அல்ல, ஆனால் அது வித்தியாசமானது. Web2 பயனர்களுக்கு இந்த Web3 முன்னுதாரணங்களைத் தெரிவிக்கும் கல்வி முயற்சிகள் அதன் வெற்றிக்கு இன்றியமையாதவை.

முக்கியமான Ethereum உள்ளடக்கத்தை முடிந்தவரை பல மொழிகளில் மொழிபெயர்ப்பதை நோக்கமாகக் கொண்டு, எங்கள் [மொழிபெயர்ப்புத் திட்டம் (Translation Program)](/contributing/translation-program/) மூலம் Web3 கல்விக்கு Ethereum.org பங்களிக்கிறது.

### மையப்படுத்தப்பட்ட உள்கட்டமைப்பு (Centralized infrastructure) {#centralized-infrastructure}

Web3 சுற்றுச்சூழல் அமைப்பு இளமையானது மற்றும் விரைவாக உருவாகி வருகிறது. இதன் விளைவாக, இது தற்போது முக்கியமாக மையப்படுத்தப்பட்ட உள்கட்டமைப்பையே (GitHub, Twitter, Discord போன்றவை) நம்பியுள்ளது. பல Web3 நிறுவனங்கள் இந்த இடைவெளிகளை நிரப்ப விரைகின்றன, ஆனால் உயர்தர, நம்பகமான உள்கட்டமைப்பை உருவாக்க நேரம் எடுக்கும்.

## ஒரு பரவலாக்கப்பட்ட எதிர்காலம் {#decentralized-future}

Web3 ஒரு இளமையான மற்றும் வளர்ந்து வரும் சுற்றுச்சூழல் அமைப்பாகும். Gavin Wood 2014-இல் இந்த வார்த்தையை உருவாக்கினார், ஆனால் இந்த யோசனைகளில் பல சமீபத்தில்தான் நனவாகியுள்ளன. கடந்த ஆண்டில் மட்டும், கிரிப்டோகரன்சி மீதான ஆர்வம், லேயர் 2 ஸ்கேலிங் தீர்வுகளுக்கான மேம்பாடுகள், புதிய நிர்வாக வடிவங்களுடனான பாரிய சோதனைகள் மற்றும் டிஜிட்டல் அடையாளத்தில் புரட்சிகள் ஆகியவற்றில் கணிசமான எழுச்சி ஏற்பட்டுள்ளது.

Web3 மூலம் சிறந்த Web-ஐ உருவாக்குவதன் தொடக்கத்தில் மட்டுமே நாம் இருக்கிறோம், ஆனால் அதை ஆதரிக்கும் உள்கட்டமைப்பை நாம் தொடர்ந்து மேம்படுத்தும்போது, Web-இன் எதிர்காலம் பிரகாசமாகத் தெரிகிறது.
