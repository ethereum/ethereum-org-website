# Web3-க்கான அறிமுகம் {#introduction}

<div className="mt-4">
<ListenToPlayer slug="/web3/" />
</div>

மையமயமாக்கல் உலகளாவிய இணையத்தில் கோடிக்கணக்கான மக்களை வரவேற்க உதவியது மற்றும் அதன் மேல் வாழ்க்கையை நடத்த உதவும் நிலைத்த, வலுவான பிணையத்தை உருவாக்கியுள்ளது. ஒரே நேரத்தில், சில மையமயமாக்கப்பட்ட நிறுவனங்கள் உலகளாவிய இணையத்தின் பெரியபகுதியைக் கைப்பற்றியுள்ளனன, என்னை அனுமதிக்க வேண்டும், என்னைஅனுமதிக்கக் கூடாதுு என்று ஒருதலையாக முடிவு செய்கின்றன.

Web3 இத்தகைய சிக்கலுக்கான பதிலாகும். பெரிய தொழில்நுட்ப நிறுவனங்களால் ஆக்கிரமிக்கப்பட்ட இணையத்திற்குப் பதிலாக, Web3 மையமற்ற தன்மையை ஏற்கிறது மற்றும் அதன் பயனாளர்களால் கட்டப்பட்டு, இயக்கப்பட்டு, உரிமையுள்ளதாக உள்ளது. Web3 சக்தியை நிறுவனங்களின் கையிலிருந்து தனிநபர்களின் கைகளுக்கு மாற்றுகிறது.
Web3 பற்றிப் பேசுவதற்கு முன், நாங்கள் இங்கு எப்படி வந்தோம் என்பதைக் கண்டுபிடிக்கலாம்.

## ஆரம்பகால இணையம் {#early-internet}

இன்றைய இணையம் நவீன வாழ்க்கையின் தொடர்ச்சியான தூணாகவே அனைவரும் நினைக்கிறார்கள் - இது கண்டுபிடிக்கப்பட்டது மற்றும் தொடர்ந்து இருந்து வருகிறது. இருப்பினும், இன்றைக்கு நாங்கள் அறிந்திருக்கும் இணையம் ஆரம்பத்தில் கற்பனை செய்ததிலிருந்து மிகவும் மாறுபட்டது. இதைச் சிறப்பாகப் புரிந்துகொள்ள, இணையத்தின் குறுகிய வரலாற்றைச் சற்றே சிதறியகாலங்களில் பிரிக்கலாம்—Web 1.0 மற்றும் Web 2.0.

### Web 1.0: வாசிப்பு மட்டும் (1990-2004) {#web1}

1989 இல், செர்ன், ஜெனிவாவில், டிம் பெர்னர்ஸ்-லீ உலகளாவிய இணையமாக மாறிய சிக்கல்களை உருவாக்குவதில் தீவிரமாக இருந்தார். அவரது யோசனை என்ன? தகவல் பகிர்வுகளை எங்கிருந்தும் பகிரும் திறன் கொண்ட திறந்த, மையமற்ற நெறிமுறைகளை உருவாக்க.

பெர்னர்ஸ்-லீயின் படைப்பின் முதல் உருவாக்கம், தற்போது 'Web 1.0' என்று அழைக்கப்படுகிறது, சுமார் 1990 முதல் 2004 வரை நிகழ்ந்தது. Web 1.0 கணினி நிறுவனங்கள் தங்களுக்குள் உள்ள வெப்சைட்டுகளே மையமாக இருந்தது, மற்றும் பயனாளர்களுக்கு இடையே குறைந்த அளவு இடைமுகங்கள் மட்டுமே இருந்தன.

![வாடிக்கையாளர்-சேவையக கட்டமைப்பு, Web 1.0-ஐக் குறிக்கிறது](./web1.png)

### Web 2.0: வாசிப்பு-எழுதுதல் (2004-தற்போது) {#web2}

Web 2.0 காலம் 2004 இல் சமூக ஊடக தளங்களின் உருவாகியதுடன் துவங்கியது. வாசிப்பு மட்டுமில்லாமல், இணையம் வாசிப்பு-எழுதுதல் என மாறியது. நிறுவனங்கள் பயனாளர்களுக்கு உள்ளடக்கங்களை வழங்கியதற்கு பதிலாக, பயனாளர் உருவாக்கப்பட்ட உள்ளடக்கங்களைப் பகிர்வதற்கும் பயனர்-பயனர் இடைமுகங்களில் ஈடுபடுவதற்கும் தளங்களை வழங்க ஆரம்பித்தனர். மேலும், மக்கள் இணையத்திற்குள் அதிக அளவில் இணையத் தொடங்கியதால், சில முன்னணி நிறுவனங்கள் இணையத்தில் உற்பத்தியாகியிருக்கும் மிகப் பரவலான அளவு திரவியங்களைக் கட்டுப்படுத்தத் தொடங்கினர். Web 2.0 விளம்பர-சார்ந்த வருமான மாதிரியை உருவாக்கியது. பயனாளர்கள் உள்ளடக்கங்களை உருவாக்க முடிந்தபோதும், அவற்றின் நிலைத்த நிலைத்த நிலம் பயன்படுத்தப்படுவதில் அவர்கள் உரிமை பெறவில்லை.

![வாடிக்கையாளர்-சேவையக கட்டமைப்பு, Web 2.0-ஐக் குறிக்கிறது](./web2.png)

## Web 3.0: வாசிப்பு-எழுதுதல்-உரிமை {#web3}

'Web 3.0' என்ற கோட்பாட்டை [Ethereum](/what-is-ethereum/) இணை நிறுவனர் கேவின் வுட் 2014-இல் Ethereum தொடங்கப்பட்ட சிறிது காலத்திலேயே உருவாக்கினார். கேவின் பல ஆரம்பகால க்ரிப்டோ உற்பத்தி எட்டிகள் அனுபவித்த பிரச்சினைக்குத் தீர்வை வார்த்தைகளில் வெளிப்படுத்தினார்: இணையம் மிகவும் நம்பிக்கை தேவை. அதாவது, மக்கள் அறிந்திருக்கும் இணையத்தின் பெரும்பாலான பகுதி சில தனியார் நிறுவனங்கள் பொது நலனில் செயல்படுகின்றன என்ற நம்பிக்கையில் மட்டுமே நம்பிக்கை.

![பரவலாக்கப்பட்ட முனை கட்டமைப்பு, Web3-ஐக் குறிக்கிறது](./web3.png)

### Web3 என்றால் என்ன? {#what-is-web3}

Web3 என்பதற்கான காட்சியைப் பரிசோதிக்கும் புதிய, சிறந்த இணையத்தை உருவாக்கும் முழுமையான ஆவணம். இதன் மூலக்கூறு என்ன? Web3 வல்லாண்மையைப் பயனாளர்களின் கைகளில் மாற்றுகிறது. 2020 இல் ஒரு ட்விட்டர் பதிவில் அதனைச் சிறப்பாக வெளிப்படுத்தியது: Web1 வாசிப்பு மட்டும், Web2 வாசிப்பு-எழுதுதல், Web3 வாசிப்பு-எழுதுதல்-உரிமை ஆகும்.

#### Web3 இன் முக்கிய கருத்துக்கள் {#core-ideas}

Web3 என்ன என்பது ஒரு உறுதியான விளக்கத்தை வழங்குவது சவாலானது, இருந்தாலும், அதன் உருவாக்கத்தை வழிநடத்தும் சில முக்கியக் கொள்கைகள் உள்ளன.

- **Web3 பரவலாக்கப்பட்டது:** இணையத்தின் பெரும் பகுதிகள் மையப்படுத்தப்பட்ட நிறுவனங்களால் கட்டுப்படுத்தப்பட்டு சொந்தமாக்கப்படுவதற்குப் பதிலாக, உரிமை அதன் உருவாக்குநர்கள் மற்றும் பயனர்களிடையே பகிர்ந்தளிக்கப்படுகிறது.
- **Web3 அனுமதியற்றது:** Web3-இல் பங்கேற்க அனைவருக்கும் சமமான அணுகல் உள்ளது, யாரும் விலக்கப்படுவதில்லை.
- **Web3 உள்ளார்ந்த பணம் செலுத்தும் முறைகளைக் கொண்டுள்ளது:** இது வங்கிகள் மற்றும் கட்டணச் செயலிகளின் காலாவதியான உள்கட்டமைப்பைச் சார்ந்திருப்பதற்குப் பதிலாக, இணையத்தில் பணத்தைச் செலவழிக்கவும் அனுப்பவும் கிரிப்டோகரன்சியைப் பயன்படுத்துகிறது.
- **Web3 நம்பிக்கை தேவைப்படாதது:** இது நம்பகமான மூன்றாம் தரப்பினரைச் சார்ந்திருப்பதற்குப் பதிலாக, ஊக்கத்தொகைகள் மற்றும் பொருளாதார வழிமுறைகளைப் பயன்படுத்தி செயல்படுகிறது.

### Web3 ஏன் முக்கியமானது? {#why-is-web3-important}

Web3 இன் முக்கிய அம்சங்கள் தனித்தன்மை வாய்ந்தவை அல்ல மற்றும் சீரான பிரிவுகளில் பொருந்துவதில்லை, எளிமைப்படுத்துவதற்காக, அவற்றைப் புரிந்துகொள்ள எளிதாகப் பிரித்து விவாதிக்க முயற்சிக்கிறோம்.

#### உரிமை {#ownership}

Web3 உங்களுக்கு உங்கள் டிஜிட்டல் சொத்துக்களின் முன்னெப்போதும் இல்லாதவாறு உரிமையை வழங்குகிறது. எடுத்துக்காட்டாக, நீங்கள் ஒரு Web2 கேமை விளையாடுகிறீர்கள் என்று கூறுங்கள். நீங்கள் ஒரு கேமில் உள்ள உருப்படியை வாங்கினால், அது நேரடியாக உங்கள் கணக்குடன் தொடர்பு கொண்டிருக்கும். கேமின் உருவாக்குநர்கள் உங்கள் கணக்கை நீக்கினால், நீங்கள் அந்த உருப்படிகளை இழந்து விடுவீர்கள். அல்லது, நீங்கள் கேமிங் செய்யாமல் விட்டால், உங்கள் கேமில் உள்ள உருப்படிகளுக்கு முதலீடு செய்த மதிப்பை இழக்கிறீர்கள்.

Web3, [பதிலீடு செய்ய முடியாத டோக்கன்கள் (NFTகள்)](/glossary/#nft) மூலம் நேரடி உரிமையை அனுமதிக்கிறது. கேமின் உருவாக்குநர்கள் உட்பட, யாருக்கும் உங்கள் உரிமையை அகற்ற இயலாது. மேலும், நீங்கள் கேமிங் செய்யாமல் விட்டால், உங்கள் கேமில் உள்ள உருப்படிகளைத் திறந்த சந்தைகளில் விற்க அல்லது பரிமாறலாம் மற்றும் அதன் மதிப்பை மீட்டுக்கொள்ளலாம். இதைச் செயல்பாட்டில் காண [ஆன்செயின் கேமிங்கை](/gaming/) ஆராயுங்கள்.

#### தணிக்கை எதிர்ப்பு {#censorship-resistance}

மேடைகள் மற்றும் உள்ளடக்க உருவாக்குநர்கள் ஆகியோருக்கிடையிலான அதிகாரத்தின் சீர்குலைப்பு பரந்த அளவிலானது.

எடுத்துக்காட்டாக, OnlyFans என்பது பயனர் உருவாக்கிய ஆபாசமான உள்ளடக்கத்திற்கான தளமாகும், இது 1 மில்லியனுக்கும் அதிகமான உள்ளடக்க உருவாக்குநர்களைக் கொண்டுள்ளது, இதில் பலர் தங்களின் பிரதான வருமான மூலமாக இந்தத் தளத்தைப் பயன்படுத்துகின்றனர். 2021 ஆகஸ்டில், OnlyFans பாலியல் நெறிமுறைகள் தொடர்பான உள்ளடக்கத்தைத் தடை செய்யத் திட்டமிட்டது. இந்த அறிவிப்பு தளத்தில் உள்ள உருவாக்குநர்களிடையே கடுமையான எதிர்ப்பை ஏற்படுத்தியது, அவர்கள் தாங்கள் உருவாக்கிய ஒரு தளத்தில் வருமானத்தை இழப்பதாக நினைத்தனர். எதிர்ப்பின் பின்னர், முடிவு விரைவாகத் திரும்பிச் சென்றது. உருவாக்குநர்கள் இந்தப் போராட்டத்தில் வெற்றி பெற்றாலும், இது Web 2.0 உருவாக்குநர்களுக்கான ஒரு பிரச்சனையை முன்னிலைப்படுத்துகிறது: நீங்கள் ஒரு தளத்தை விட்டு வெளியேறினால், நீங்கள் சேர்த்த செல்வாக்கையும் ஆதரவையும் இழந்து விடுவீர்கள்.

Web3 இல், உங்கள் தரவுகள் பிளாக்செயினில் வாழ்கின்றன. நீங்கள் ஒரு தளத்தை விட்டு வெளியேற முடிவெடுத்தால், உங்கள் செல்வாக்கை உங்களுடன் கொண்டு செல்லலாம், மேலும் உங்கள் மதிப்புகளுடன் தெளிவாக ஒத்துப்போகும் மற்றொரு இடைமுகத்தில் அதை நுழைக்கலாம்.

Web 2.0 உள்ளடக்க உருவாக்குநர்கள் விதிகளை மாற்றமாட்டார்கள் என்று தளங்களை நம்புவதைக் கோருகிறது, ஆனால் தணிக்கை எதிர்ப்பு என்பது ஒரு Web3 தளத்தின் உள்ளார்ந்த அம்சமாகும்.

#### பரவலாக்கப்பட்ட தன்னாட்சி அமைப்புகள் (DAOs) {#daos}

Web3 இல் உங்கள் தரவின் உரிமையை உடையிருப்பதற்குப் பொருத்தமாக, நீங்கள் டோக்கன்கள் மூலம் ஒரு நிறுவனம்போலப் பங்குகளுடன் தளத்தைக் குழுமமாகக் கொண்டிருக்கலாம். DAOs மூலம் நீங்கள் ஒரு தளத்தின் பரவலாக்கப்பட்ட உரிமையை ஒருங்கிணைக்கலாம் மற்றும் அதன் எதிர்காலத்தைப் பற்றிய முடிவுகளை எடுக்கலாம்.

DAOs தொழில்நுட்ப ரீதியாக, ஒப்புக்கொள்ளப்பட்ட [ஸ்மார்ட் ஒப்பந்தங்கள்](/glossary/#smart-contract) என வரையறுக்கப்படுகின்றன, அவை வளங்களின் (டோக்கன்கள்) ஒரு தொகுப்பின் மீது பரவலாக்கப்பட்ட முடிவெடுப்பதை தானியக்கமாக்குகின்றன. டோக்கன் உடைய பயனர்கள் எப்படி வளங்கள் செலவழிக்கப்படுகின்றன என்பதில் வாக்களிக்கின்றனர், மேலும் வாக்களிக்கின்ற முடிவைத் தானாகவே நடைமுறைக்குக் கொண்டு வரும்.

இருப்பினும், மக்கள் பல Web3 சமூகங்களை DAO-க்களாக வரையறுக்கின்றனர். இந்த சமூகங்கள் அனைத்தும் குறியீட்டின் மூலம் வெவ்வேறு அளவிலான பரவலாக்கம் மற்றும் தானியக்கத்தைக் கொண்டுள்ளன. எதிர்காலத்தில் DAOs எப்படி பரிணாமம் அடையும் என்பது குறித்த ஆய்வு தற்போது நடைபெற்று வருகிறது.

### அடையாளம் {#identity}

பாரம்பரியமாக, நீங்கள் பயன்படுத்தும் ஒவ்வொரு தளத்திற்கும் ஒரு கணக்கை உருவாக்கியிருக்க வேண்டும். எடுத்துக்காட்டாக, நீங்கள் ஒரு Twitter கணக்கைப் பயன்படுத்தலாம், YouTube கணக்கைப் பயன்படுத்தலாம், மேலும் ஒரு Reddit கணக்கைப் பயன்படுத்தலாம். உங்கள் காட்சிப் பெயரை அல்லது ப்ரொஃபைல் படத்தை மாற்ற வேண்டுமா? ஒவ்வொரு கணக்கிலும் அதைச் செய்ய வேண்டும். சில நிலைகளில், நீங்கள் சமூக கையாளுதலைப் பயன்படுத்தலாம், ஆனால் இது ஒரு வழக்கமான பிரச்சினையை முன்னிலைப்படுத்துகிறது—மூடுபனி. ஒரு கிளிக் மூலம், இந்தத் தளங்கள் உங்கள் முழு இணைய வாழ்வைத் தடுக்க முடியும். மேலும் மோசமாக, பல தளங்கள் ஒரு கணக்கை உருவாக்க உங்களை அடையாளம் காணக்கூடிய தகவல்களை நம்ப உங்களுக்கிடையே வைத்து நம்பிக்கை செலுத்தச் செய்யும்.

ஒரு Ethereum முகவரி மற்றும் [Ethereum பெயர் சேவை (ENS)](/glossary/#ens) சுயவிவரம் மூலம் உங்கள் டிஜிட்டல் அடையாளத்தைக் கட்டுப்படுத்த அனுமதிப்பதன் மூலம் Web3 இந்தப் சிக்கல்களைத் தீர்க்கிறது. ஒரு Ethereum முகவரியைக் கொண்டிருப்பது, தளங்களுக்குப் போட்டியாக ஒரு தனித்துவமான உள்நுழைவு வழங்குகிறது, மேலும் இது பாதுகாப்பானது, மூடுபனி எதிர்ப்பு மற்றும்பெயர் இல்லாமல் உள்ளது.

### உள்ளார்ந்த பணம் செலுத்துதல்கள் {#native-payments}

Web2 இன் கட்டண முகாமைத்துவம் வங்கிகள் மற்றும் கட்டண செயலாக்கிகள் மீதான நம்பகத்தன்மையை மையமாகக் கொண்டது, அதுவே வங்கிக் கணக்குகள் இல்லாதவர்களை அல்லது தவறான நாட்டின் எல்லைகளுக்குள் வாழும் நபர்களை விலக்குகிறது.
Web3 உலாவியில் நேரடியாகப் பணம் அனுப்ப [ETH](/glossary/#ether) போன்ற டோக்கன்களைப் பயன்படுத்துகிறது, இதற்கு நம்பகமான மூன்றாம் தரப்பினர் தேவையில்லை.

## Web3 வரம்புகள் {#web3-limitations}

தற்போதைய Web3 இன் பல நன்மைகளுக்குப் பிறகும், இந்தப் பரிமாணத்தின் வளர்ச்சிக்காகச் சரிபார்க்க வேண்டிய இன்னும் பல வரம்புகள் உள்ளன.

### அணுகல்தன்மை {#accessibility}

பொருத்தமான Web3 அம்சங்கள், Sign-in with Ethereum போன்றவை, ஏற்கனவே இலவசமாகப் பயன்படுத்தத் தயாராக உள்ளன. ஆனால், பரிமாற்றங்களின் தொடர்புடைய செலவு பலருக்கு தடை செய்யும் வகையில் உள்ளது. குறைந்த வருமானம் கொண்ட, வளரும் நாடுகளில் Web3 பயன்படுத்தப்படுவதற்கான சாத்தியம் குறைவாக உள்ளது, காரணமாக உயர்ந்த பரிமாற்றச் செலவுகள். Ethereum-இல், இந்தச் சவால்கள் [செயல்திட்டம்](/roadmap/) மற்றும் [லேயர் 2 அளவிடுதல் தீர்வுகள்](/glossary/#layer-2) மூலம் தீர்க்கப்படுகின்றன. தொழில்நுட்பம் தயாராக உள்ளது, ஆனால் Web3 ஐ அனைவருக்கும் அணுக முடியும் வகையில் Layer 2 இல் உயர் நிலை கடைபிடிப்புகள் தேவைப்படுகிறது.

### பயனர் அனுபவம் {#user-experience}

Web3 பயன்படுத்துவதற்கான தொழில்நுட்பத் தடைகள் தற்போது மிகவும் உயரமாக உள்ளன. பயனர்கள் பாதுகாப்பு கவலைகளைப் புரிந்து கொள்ள வேண்டும், சிக்கலான தொழில்நுட்ப ஆவணங்களைப் புரிந்துகொள்ள வேண்டும், மேலும் தொடங்க இயலாத பயனர் இடைமுகங்களைத் தேட வேண்டும். [வாலட் வழங்குநர்கள்](/wallets/find-wallet/), குறிப்பாக, இதைத் தீர்க்க உழைத்து வருகின்றனர், ஆனால் Web3 பெருமளவில் ஏற்றுக்கொள்ளப்படுவதற்கு முன்பு மேலும் முன்னேற்றம் தேவை.

### கல்வி {#education}

Web3 புதிய நிகழ்ச்சிகளைக் கொண்டு வருகிறது, அவை Web2.0 க்கு பயன்படுத்தப்பட்டுள்ள மனவியல் மாதிரிகளுக்கு மாறான விதமாகக் கற்றல் தேவைப்படும். 1990-களின் பிற்பகுதியில் Web1.0 பிரபலமடைந்து வந்தபோதும் இதேபோன்ற ஒரு கல்வி முன்னெடுப்பு நடந்தது; உலகளாவிய வலையின் ஆதரவாளர்கள் பொதுமக்களுக்குக் கல்வி கற்பிக்க, எளிய உருவகங்கள் (தகவல் நெடுஞ்சாலை, உலாவிகள், இணையத்தில் உலாவுதல்) முதல் [தொலைக்காட்சி ஒளிபரப்புகள்](https://www.youtube.com/watch?v=SzQLI7BxfYI) வரை பல கல்வி நுட்பங்களைப் பயன்படுத்தினர். Web3 கற்றல் சிரமமானது அல்ல, ஆனால் இது வேறுபட்டது. Web3 க்கான இந்தச் சமயங்களை Web2 பயனர்களுக்கு அறிவிக்கும் கல்வித் திட்டங்கள் அதன் வெற்றிக்கு முக்கியமானவை.

Ethereum.org எங்களது [மொழிபெயர்ப்புத் திட்டத்தின்](/contributing/translation-program/) மூலம் Web3 கல்விக்குப் பங்களிக்கிறது, முக்கியமான Ethereum உள்ளடக்கத்தை முடிந்தவரை பல மொழிகளில் மொழிபெயர்ப்பதை நோக்கமாகக் கொண்டுள்ளது.

### மையப்படுத்தப்பட்ட உள்கட்டமைப்பு {#centralized-infrastructure}

Web3 இன்பகுதி இளம் மற்றும் விரைவாகப் பரிணாமம் அடைவதாக உள்ளது.   பல Web3 நிறுவனங்கள் இந்த இடைவெளிகளை நிரப்ப ஆவலுடன் உள்ளன, ஆனால் உயர்தரமான, நம்பகமான கட்டமைப்பை உருவாக்குவது நேரம் எடுக்கும்.

## ஒரு பரவலாக்கப்பட்ட எதிர்காலம் {#decentralized-future}

Web3 என்பது ஒரு இளம் மற்றும் பரிணாம வளர்ச்சி அடையும் பரிமாணமாகும். கேவின் வுட் 2014 இல் Web3 என்பதைக் குறித்த வார்த்தையை உருவாக்கினார், ஆனால் இந்த எண்ணங்கள் சமீபத்திய காலங்களில் தான் உண்மைமையடைந்துள்ளன. கடந்த ஆண்டில் மட்டும், cryptocurrency இல் பெரும் ஆர்வத்தை அதிகரித்துள்ளது, Layer 2 அளவுரு தீர்வுகளில் முன்னேற்றங்கள், புதிய முறை நிர்வாகங்களின் பரிசோதனைகள் மற்றும் டிஜிட்டல் அடையாளத்தில் புரட்சிகள் ஏற்படுத்தியுள்ளது.

Web3 உடன் ஒரு சிறந்த இணையத்தை உருவாக்குவதற்கான தொடக்கத்தில் மட்டுமே நாம் உள்ளோம், ஆனால் அதை ஆதரிக்கின்ற கட்டமைப்பை மேம்படுத்துவதற்குத் தொடர்வதற்குப் பிறகும், இணையத்தின் எதிர்காலம் பிரகாசமாகத் தெரிகிறது.
