ஒவ்வொரு சரிபார்ப்பானும் ஒரு **திரும்பப்பெறல் சான்றைக்** கொண்டுள்ளது, இது அவர்களின் பங்குவைக்கப்பட்ட ETH மற்றும் வெகுமதிகளை எப்படி, எங்கு திரும்பப் பெறலாம் என்பதைத் தீர்மானிக்கிறது. சான்று வகையானது முதல் பைட்டால் குறிக்கப்படுகிறது: `0x00`, `0x01`, அல்லது `0x02`. தங்கள் பங்குவைப்பை நிர்வகிக்கும் சரிபார்ப்பான்களுக்கு இந்த வகைகளைப் புரிந்துகொள்வது முக்கியமாகும்.

## 0x00: ஷபெல்லாவுக்கு முந்தைய சான்றுகள் {#0x00-credentials}

`0x00` வகையானது ஷபெல்லா மேம்படுத்தலுக்கு (ஏப்ரல் 2023) முந்தைய அசல் திரும்பப்பெறல் சான்று வடிவமாகும். இந்தச் சான்று வகையைக் கொண்ட சரிபார்ப்பான்களுக்கு செயலாக்க அடுக்கு திரும்பப்பெறல் முகவரி எதுவும் அமைக்கப்படவில்லை, அதாவது அவர்களின் நிதிகள் கருத்தொருமிப்பு அடுக்கிலேயே பூட்டப்பட்டிருக்கும். உங்களிடம் இன்னும் `0x00` சான்றுகள் இருந்தால், நீங்கள் எந்தவொரு திரும்பப் பெறுதலையும் பெறுவதற்கு முன்பு `0x01` அல்லது `0x02` க்கு மேம்படுத்த வேண்டும்.

## 0x01: மரபுவழி திரும்பப்பெறல் சான்றுகள் {#0x01-credentials}

`0x01` வகையானது ஷபெல்லா மேம்படுத்தலுடன் அறிமுகப்படுத்தப்பட்டது, மேலும் செயலாக்க அடுக்கு திரும்பப்பெறல் முகவரியை அமைக்க விரும்பிய சரிபார்ப்பான்களுக்கான தரநிலையாக மாறியது. `0x01` சான்றுகளுடன்:

- 32 ETH-க்கு மேலான எந்தவொரு இருப்பும் உங்கள் திரும்பப்பெறல் முகவரிக்கு **தானாகவே மாற்றப்படும்**
- முழுமையான வெளியேற்றங்கள் நிலையான வெளியேற்ற வரிசையின் வழியாகச் செல்லும்
- 32 ETH-க்கு மேலான வெகுமதிகள் கூட்டப்படாது—அவை அவ்வப்போது வெளியேற்றப்படும்

**சில சரிபார்ப்பான்கள் இன்னும் 0x01-ஐப் பயன்படுத்துவது ஏன்:** இது எளிமையானது மற்றும் பழக்கமானது. பல சரிபார்ப்பான்கள் ஷபெல்லாவுக்குப் பிறகு வைப்பிலிட்டனர், மேலும் ஏற்கனவே இந்த வகையைக் கொண்டுள்ளனர், மேலும் அதிகப்படியான இருப்பைத் தானாகவே திரும்பப் பெற விரும்புவோருக்கு இது நன்றாக வேலை செய்கிறது.

**இது ஏன் பரிந்துரைக்கப்படவில்லை:** `0x01` உடன், 32 ETH-க்கு மேலான வெகுமதிகளைக் கூட்டிக்கொள்ளும் திறனை நீங்கள் இழக்கிறீர்கள். ஒவ்வொரு அதிகப்படியான பகுதியும் தானாகவே வெளியேற்றப்படுகிறது, இது உங்கள் சரிபார்ப்பானின் சம்பாதிக்கும் திறனைக் கட்டுப்படுத்துகிறது மற்றும் திரும்பப் பெறப்பட்ட நிதியைத் தனியாக நிர்வகிக்க வேண்டிய அவசியத்தை ஏற்படுத்துகிறது.

## 0x02: கூட்டுத் திரும்பப்பெறல் சான்றுகள் {#0x02-credentials}

`0x02` வகையானது பெக்ட்ரா மேம்படுத்தலுடன் அறிமுகப்படுத்தப்பட்டது, மேலும் இது இன்று சரிபார்ப்பான்களுக்கு **பரிந்துரைக்கப்படும் தேர்வாகும்**. `0x02` சான்றுகளைக் கொண்ட சரிபார்ப்பான்கள் சில நேரங்களில் "கூட்டுச் சரிபார்ப்பான்கள்" என்று அழைக்கப்படுகிறார்கள்.

`0x02` சான்றுகளுடன்:

- 32 ETH-க்கு மேலான வெகுமதிகள் அதிகபட்சமாக 2048 ETH பயனுறு இருப்பு வரை 1 ETH அதிகரிப்புகளில் **கூட்டப்படும்**
- பகுதி திரும்பப் பெறுதல்களை கைமுறையாகக் கோர வேண்டும் (தானியங்கி மாற்றங்கள் 2048 ETH வரம்பிற்கு மேல் மட்டுமே நிகழும்)
- சரிபார்ப்பான்கள் பல 32 ETH சரிபார்ப்பான்களை அதிக இருப்புள்ள ஒற்றைச் சரிபார்ப்பானாக ஒருங்கிணைக்க முடியும்
- முழுமையான வெளியேற்றங்கள் இன்னும் நிலையான வெளியேற்ற வரிசையின் மூலம் ஆதரிக்கப்படுகின்றன

பகுதி திரும்பப் பெறுதல்கள் மற்றும் ஒருங்கிணைப்புகள் இரண்டையும் [லாஞ்ச்பேட் சரிபார்ப்பான் செயல்கள் (Launchpad Validator Actions)](https://launchpad.ethereum.org/en/validator-actions) வழியாகச் செய்ய முடியும்.

**சரிபார்ப்பான்கள் ஏன் 0x02-ஐ விரும்ப வேண்டும்:** இது கூட்டுச் செயல்பாட்டின் மூலம் சிறந்த மூலதனத் திறனையும், திரும்பப் பெறுதல்கள் எப்போது நிகழும் என்பதன் மீது கூடுதல் கட்டுப்பாட்டையும் வழங்குகிறது, மேலும் சரிபார்ப்பான் ஒருங்கிணைப்பை ஆதரிக்கிறது. காலப்போக்கில் வெகுமதிகளைக் குவிக்கும் தனிப் பங்குவைப்பாளர்களுக்கு, இதன் பொருள் அவர்களின் பயனுறு இருப்பு—அதனால் அவர்களின் வெகுமதிகள்—கைமுறை தலையீடு இல்லாமல் 32 ETH-க்கு அப்பால் வளர முடியும் என்பதாகும்.

**முக்கியமானது:** நீங்கள் `0x01` இலிருந்து `0x02` க்கு மாற்றியவுடன், உங்களால் மீண்டும் மீளமைக்க முடியாது.

வகை 2 சான்றுகளுக்கு மாற்றுவது மற்றும் MaxEB அம்சம் குறித்த விரிவான வழிகாட்டிக்கு, [MaxEB விளக்கப் பக்கத்தைப்](/roadmap/pectra/maxeb/) பார்க்கவும்.

## நான் எதைத் தேர்ந்தெடுக்க வேண்டும்? {#what-should-i-pick}

- **புதிய சரிபார்ப்பான்கள்:** `0x02` ஐத் தேர்ந்தெடுக்கவும். இது சிறந்த கூட்டுச் செயல்பாடு மற்றும் நெகிழ்வுத்தன்மையைக் கொண்ட நவீன தரநிலையாகும்.
- **தற்போதுள்ள 0x01 சரிபார்ப்பான்கள்:** 32 ETH-க்கு மேல் வெகுமதிகள் கூட்டப்பட வேண்டும் என நீங்கள் விரும்பினால் அல்லது சரிபார்ப்பான்களை ஒருங்கிணைக்கத் திட்டமிட்டால், `0x02` க்கு மாற்றுவதைக் கருத்தில் கொள்ளவும்.
- **தற்போதுள்ள 0x00 சரிபார்ப்பான்கள்:** உடனடியாக மேம்படுத்தவும்—உங்கள் சான்றுகளைப் புதுப்பிக்காமல் உங்களால் திரும்பப் பெற முடியாது. நீங்கள் முதலில் `0x01` க்கு மாற்ற வேண்டும், பின்னர் நீங்கள் `0x02` க்கு மாற்றலாம்.

## திரும்பப்பெறல் சான்றுகளை நிர்வகிப்பதற்கான கருவிகள் {#withdrawal-credential-tools}

பல கருவிகள் சான்று வகைகளைத் தேர்ந்தெடுப்பதை அல்லது அவற்றுக்கிடையே மாற்றுவதை ஆதரிக்கின்றன:

- **[எத்திரியம் பங்குவைப்பு லாஞ்ச்பேட் (Ethereum Staking Launchpad)](https://launchpad.ethereum.org/en/validator-actions)** - சான்று மாற்றங்கள் மற்றும் ஒருங்கிணைப்புகள் உட்பட, வைப்புகள் மற்றும் சரிபார்ப்பான் நிர்வாகத்திற்கான அதிகாரப்பூர்வ கருவி
- **[பெக்ட்ரா பங்குவைப்பு மேலாளர் (Pectra Staking Manager)](https://pectrastaking.com)** - மாற்றங்கள் மற்றும் ஒருங்கிணைப்புக்கான பணப்பை-இணைப்பு (wallet-connect) ஆதரவுடன் கூடிய வலை UI
- **[பெக்ட்ரா சரிபார்ப்பான் ஆப்ஸ் CLI கருவி (Pectra Validator Ops CLI Tool)](https://github.com/Luganodes/Pectra-Batch-Contract)** - தொகுதி மாற்றங்களுக்கான கட்டளை வரி கருவி
- **[எத்திரியல் (Ethereal)](https://github.com/wealdtech/ethereal)** - சரிபார்ப்பான் நிர்வாகம் உட்பட எத்திரியம் செயல்பாடுகளுக்கான CLI கருவி

ஒருங்கிணைப்புக் கருவிகளின் முழுமையான பட்டியல் மற்றும் விரிவான மாற்று வழிமுறைகளுக்கு, [MaxEB ஒருங்கிணைப்புக் கருவிகளைப்](/roadmap/pectra/maxeb/#consolidation-tooling) பார்க்கவும்.
