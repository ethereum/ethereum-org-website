Web2 என்பது இன்றைய இணையத்தின் பெரும்பாலோரும் அறிந்த பதிப்பை குறிக்கிறது. நீங்கள் வழங்கும் தனிப்பட்ட தரவிற்கு பதிலாக சேவைகள் வழங்கும் நிறுவனங்கள் ஆட்கொள்கின்ற இணையம். எதீரியம் தொடர்பான Web3 என்பது பிளாக்செயினில் இயங்கும் decentralized செயலிகள் (dapps) குறிக்கும். இந்த செயலிகள் யாரும் தங்களுடைய தனிப்பட்ட தரவை பணமாக மாற்றாமல் பங்கேற்க அனுமதிக்கின்றன.

மேலும் ஆரம்ப நிலை பயனாளிகளுக்கு ஏற்ற ஆதாரத்தை தேடுகிறீர்களா? எங்கள் [web3 அறிமுகத்தைப்](/web3/) பார்க்கவும்.

## Web3 நன்மைகள் {#web3-benefits}

எதீரியத்தின் இயற்கையான decentralization காரணமாக, பல Web3 டெவலப்பர்கள் dapps உருவாக்க தேர்ந்தெடுத்துள்ளனர்:

- நெட்வொர்க்கில் உள்ள யாரும் சேவையை பயன்படுத்த அனுமதி பெறலாம் – அல்லது வேறு சொல்லாக, அனுமதி தேவை இல்லை.
- யாரும் உங்களை தடுக்கும் அல்லது சேவையை அணுகுவதிலிருந்து மறுக்கும் முடியாது.
- செலவுகள் மூலக்கணம் (native token) Ether (ETH) மூலம் உட்பிரவேசமாக உள்ளன.
- எதீரியம் turing-complete ஆகும், அதாவது நீங்கள் பலவிதமான நிரல்களை உருவாக்கலாம்.

## Web3 வரம்புகள் {#web3-limitations}

Web3 இப்போது சில வரம்புகள் கொண்டுள்ளது:

- Scalability – web3 இல் பரிவர்த்தனைகள் (transactions) மெதுவாக உள்ளன, ஏனெனில் அவை விரிவாக்கமில்லாதவை. நிலைமாற்றங்கள், உதாரணமாக, ஒரு பணம் செலுத்தல், ஒரு நோடால் செயலாக்கப்பட வேண்டும் மற்றும் முழு வலையமைப்பில் பரவ வேண்டும்.
- UX – web3 செயலிகளுடன் தொடர்பு கொள்வது கூடுதல் படிகள், மென்பொருள், மற்றும் கல்வியை தேவைப்படுத்தலாம். இது ஏற்றுக்கொள்ளுதலுக்கு தடையாக இருக்கலாம்.
- Accessibility – நவீன இணைய உலாவிகளில் ஒருங்கிணைப்பு இல்லாததால் web3 பெரும்பாலான பயனர்களுக்கு குறைவாக அணுகக்கூடியது.
- Cost – வெற்றிகரமான dapp கள் பெரும்பாலும் தங்கள் குறியீட்டின் மிக சிறிய பகுதிகளை மட்டுமே blockchain இல் வைக்கின்றன, ஏனெனில் அது செலவானது.

## மையப்படுத்தல் மற்றும் பரவலாக்கம் {#centralization-vs-decentralization}

கீழே உள்ள அட்டவணையில், மையமையாக்கப்பட்ட மற்றும் விரிவாக்கமில்லாத டிஜிட்டல் வலையமைப்புகளின் சில பொதுவான நன்மைகள் மற்றும் குறைகள் பட்டியலிடப்பட்டுள்ளன.

| Centralized Systems                                                                                                                                                                                                                                               | Decentralized Systems                                                                                                                                                                                                                                                               |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| குறைந்த வலையமைப்பு விட்டம் (அனைத்து பங்கேற்பாளர்கள் மைய அதிகாரத்திற்கு இணைக்கப்பட்டுள்ளனர்); தகவல் விரைவாக பரவுகிறது, ஏனெனில் பரவல் அதிக கணினி வளங்களைக் கொண்ட மைய அதிகாரத்தால் கையாளப்படுகிறது.                               | வலையமைப்பில் மிகவும் தொலைவில் உள்ள பங்கேற்பாளர்கள் ஒருவருக்கொருவர் பலத் தூரங்களில் இருக்கலாம். வலையமைப்பின் ஒரு பக்கத்திலிருந்து தகவல் ஒப்பிலிருந்து மற்ற பக்கத்திற்கு செல்ல அதிக நேரம் ஆகலாம்.                                                     |
| பொதுவாக உயர் செயல்திறன் (மேலதிக throughput, குறைவான மொத்த கணினி வளங்கள் பயன்படுத்தப்படுகின்றன) மற்றும் செயலாக்கம் எளிதாக.                                                                                                      | பொதுவாக குறைந்த செயல்திறன் (குறைந்த throughput, அதிக மொத்த கணினி வளங்கள் பயன்படுத்தப்படுகின்றன) மற்றும் செயலாக்கம் சிக்கலானது.                                                                                                                   |
| தகவல்களில் முரண்பாடு ஏற்பட்டால் தீர்வு தெளிவாக மற்றும் எளிதாக இருக்கும்: இறுதி உண்மை மூலமாக மைய அதிகாரம் உள்ளது.                                                                                                                  | ஒரு சிக்கலான நெறிமுறை (protocol) தேவைப்படுகிறது, பீர்களால் நிலை தொடர்பான முரண்பாடுகளை உருவாக்கினால், பங்கேற்பாளர்கள் அதில் ஒத்திசைவாக இருக்க வேண்டும்.                                                                                           |
| ஒற்றை பாயிண்ட் அவுடேஜ்: தீய நபர்கள் மைய அதிகாரத்தை இலக்கு வைத்து வலையமைப்பை முடக்கக்கூடும்.                                                                                                                                       | ஒற்றை பாயிண்ட் அவுடேஜ் இல்லை: வலையமைப்பு பெரும்பான்மையான பங்கேற்பாளர்கள் தாக்கப்பட்டாலும் அல்லது நீக்கப்பட்டாலும் செயல்படும்.                                                                                                                       |
| வலையமைப்பில் பங்கேற்பாளர்களை ஒருங்கிணைப்பது மிகவும் எளிது மற்றும் மைய அதிகாரத்தால் கையாளப்படுகிறது. மைய அதிகாரம் வலையமைப்பு மேம்பாடுகள், நெறிமுறை புதுப்பிப்புகளை எந்த தடையும் இல்லாமல் பங்கேற்பாளர்களால் ஏற்றுக்கொள்ள வைக்கலாம். | ஒருங்கிணைப்பு பொதுவாக கடினமாக உள்ளது, ஏனெனில் ஒரே முகவர் வலையமைப்பு நிலை முடிவுகளை, நெறிமுறை புதுப்பிப்புகளை இறுதி சொல்ல முடியாது. மிக மோசமான நிலையில், நெறிமுறை மாற்றங்கள் தொடர்பாக முரண்பாடுகள் இருந்தால் வலையமைப்பு பிரிக்கப்பட வாய்ப்பு உள்ளது. |
| மைய அதிகாரம் தரவை цен்சர் செய்யலாம், இதனால் வலையமைப்பின் பகுதிகள் மற்ற பாகங்களுடன் தொடர்பு கொள்ள முடியாது. цен்சர் செய்வது மிகவும் கடினம், ஏனெனில் தகவல் வலையமைப்பில் பல வழிகளில் பரவக்கூடும்.                                    | வலையமைப்பில் பங்கேற்பு மைய அதிகாரத்தால் கட்டுப்படுத்தப்படுகிறது. யாரும் வலையமைப்பில் பங்கேற்கலாம்; எந்த “கேட்பீப்பர்கள்” இல்லை. வாய்ப்பின் செலவு மிகவும் குறைவாக இருக்கும்.                                                         |
| வலையமைப்பில் பங்கேற்பு மைய அதிகாரத்தால் கட்டுப்படுத்தப்படுகிறது. யாரும் வலையமைப்பில் பங்கேற்கலாம்.                                                                                                                                | எந்த “கேட்பீப்பர்கள்” இல்லை. வாய்ப்பின் செலவு மிகவும் குறைவாக இருக்கும். வாய்ப்பின் செலவு மிகவும் குறைவாக இருக்கும்.                                                                                                                |

குறிப்பு: இவை பொதுவான மாதிரிகள், ஒவ்வொரு வலையமைப்பிலும் சரிபோகாமல் இருக்கலாம். மேலும், உண்மையில் ஒரு வலையமைப்பு மையமையாக்கப்பட்ட/விரிவாக்கமில்லாத அளவுக்கான அளவுகோலில் உள்ளது; எந்த வலையமைப்பும் முழுமையாக மையமையாக்கப்படவில்லை அல்லது முழுமையாக விரிவாக்கமில்லாதது அல்ல.
