Ethereum-இல் உருவாக்க டெவலப்பர்கள் [ஸ்மார்ட் ஒப்பந்தங்களை](/developers/docs/smart-contracts/) எழுத வேண்டும் என்பது ஒரு பொதுவான தவறான கருத்தாகும். இது உண்மையல்ல.
Ethereum நெட்வொர்க் மற்றும் சமூகத்தின் சிறப்பம்சங்களில் ஒன்று, நீங்கள் எந்தவொரு நிரலாக்க மொழியிலும் [பங்கேற்க](/community/) முடியும் என்பதே.

Ethereum மற்றும் அதன் சமூகம் ஓப்பன் சோர்ஸை (open source) வரவேற்கிறது. கிளையண்ட் செயலாக்கங்கள், API-கள், மேம்பாட்டு கட்டமைப்புகள், சோதனை கருவிகள் போன்ற சமூகத் திட்டங்களை நீங்கள் பல்வேறு மொழிகளில் காணலாம்.

### எனது மொழி ஆதரிக்கப்படவில்லை என்றால் என்ன செய்வது {#other-lang}

கூடுதல் நிரலாக்க மொழிக்கான வளங்களை இணைக்க அல்லது மெய்நிகர் சமூகத்தைச் சுட்டிக்காட்ட நீங்கள் விரும்பினால், [ஒரு சிக்கலைத் திறப்பதன் (opening an issue)](https://github.com/ethereum/ethereum-org-website/issues/new/choose) மூலம் புதிய பக்கத்தைக் கோரலாம்.

தற்போது ஆதரிக்கப்படாத மொழியைப் பயன்படுத்தி பிளாக்செயினுடன் தொடர்புகொள்வதற்கான குறியீட்டை எழுத நீங்கள் விரும்பினால், Ethereum நெட்வொர்க்குடன் இணைக்க [JSON-RPC இடைமுகத்தைப்](/developers/docs/apis/json-rpc/) பயன்படுத்தலாம். TCP/IP-ஐப் பயன்படுத்தக்கூடிய எந்தவொரு நிரலாக்க மொழியும் இந்த இடைமுகத்தைப் பயன்படுத்தலாம்.
