## அறிமுகம் {#Introduction}

உங்கள் சொந்த [எத்தேரியம் முனையை](/developers/docs/nodes-and-clients/#what-are-nodes-and-clients) இயக்குவது சவாலானதாக இருக்கலாம், குறிப்பாகத் தொடங்கும்போதோ அல்லது வேகமாக அளவிடும்போதோ. உங்களுக்காக உகந்ததாக்கப்பட்ட முனை உள்கட்டமைப்புகளை இயக்கும் [பல சேவைகள்](#popular-node-services) உள்ளன, எனவே அதற்கு பதிலாக உங்கள் பயன்பாடு அல்லது தயாரிப்பை உருவாக்குவதில் நீங்கள் கவனம் செலுத்தலாம். இங்கு நொடித் சேவைகள் எப்படி செயல்படுகின்றன, அவற்றின் நன்மைகள் மற்றும் குறைபாடுகள் என்னென்ன என்பதையும், தொடங்க ஆர்வமானவர்களுக்கு சில வழங்குநர்களை பட்டியலிடுவோம்.

## முன்னேற்றக் கட்டுரை {#prerequisites}

முனைகள் மற்றும் கிளையண்ட்கள் என்றால் என்ன என்பது பற்றி உங்களுக்கு ஏற்கனவே புரியவில்லை என்றால், [முனைகள் மற்றும் கிளையண்ட்கள்](/developers/docs/nodes-and-clients/) என்பதைப் பார்க்கவும்.

## ஸ்டேக்கர்கள் {#stakoooooooooooooors}

தனியார் ஸோலோ ஸ்டேக்கர்கள் (Solo stakers) மூன்றாம் கட்சி வழங்குநர்களைப் பற்றிய ஒப்படைப்பு நம்பிக்கையில் இருக்காமல் அவர்களது சொந்த கட்டமைப்பை இயக்கவேண்டும். இதற்கு ஒரு உருவாக்கக் கிளையண்ட் (execution client) மற்றும் ஒரு உடன்பட்டு (consensus) கிளையண்ட் இரண்டையும் இயக்க வேண்டுமாகும். [The Merge](/roadmap/merge) க்கு முன், ஒருமித்த கிளையண்டை மட்டும் இயக்கி, செயல்படுத்தும் தரவுகளுக்கு ஒரு மையப்படுத்தப்பட்ட வழங்குநரைப் பயன்படுத்த முடிந்தது; இது இப்போது சாத்தியமில்லை - ஒரு தனி ஸ்டேக்கர் இரண்டு கிளையண்ட்களையும் இயக்க வேண்டும். இருப்பினும், இந்த செயல்திட்டத்தை எளிதாக்க பல சேவைகள் கிடைக்கின்றன.

[ஒரு முனையை இயக்குவது பற்றி மேலும் படிக்கவும்](/developers/docs/nodes-and-clients/run-a-node/).

இந்த பக்கத்தில் விவரிக்கப்பட்ட சேவைகள் ஸ்டேக்கிங் செய்யாத (non-staking) நொடிகளுக்கே ஆகும்.

## Node சேவைகள் எவ்வாறு செயல்படுகின்றன? {#how-do-node-services-work}

Node சேவை வழங்குநர்கள் உங்களுக்காக பின்னணியில் விநியோகிக்கப்பட்ட node client-களை இயக்குகிறார்கள், எனவே நீங்கள் அதைச் செய்ய வேண்டியதில்லை.

இந்த சேவைகள் வழக்கமாக blockchain-க்கு எழுதவும் படிக்கவும் பயன்படுத்தக்கூடிய API key-ஐ வழங்குகின்றன. அவை பெரும்பாலும் Mainnet உடன் கூடுதலாக [எத்தேரியம் டெஸ்ட்நெட்களுக்கான](/developers/docs/networks/#ethereum-testnets) அணுகலை உள்ளடக்கியிருக்கும்.

சில சேவைகள் உங்களுக்காக அவர்கள் நிர்வகிக்கும் உங்கள் சொந்த பிரத்யேக node-ஐ வழங்குகின்றன, மற்றவை node-களில் செயல்பாட்டை விநியோகிக்க load balancer-களைப் பயன்படுத்துகின்றன.

கிட்டத்தட்ட எல்லா node சேவைகளும் ஒருங்கிணைக்க மிகவும் எளிதானவை, உங்கள் சொந்தமாக ஹோஸ்ட் செய்யப்பட்ட node-ஐ மாற்ற உங்கள் குறியீட்டில் ஒரு வரியை மாற்றுவது அல்லது சேவைகளுக்கு இடையில் மாறுவது ஆகியவை அடங்கும்.

பெரும்பாலும் முனை சேவைகள் பலவிதமான [முனை கிளையண்ட்களையும்](/developers/docs/nodes-and-clients/#execution-clients) [வகைகளையும்](/developers/docs/nodes-and-clients/#node-types) இயக்கும், இது ஒரு API இல் கிளையண்ட் சார்ந்த முறைகளுக்கு கூடுதலாக முழு மற்றும் காப்பக முனைகளை அணுக உங்களை அனுமதிக்கிறது.

Node சேவைகள் உங்கள் தனிப்பட்ட keys அல்லது தகவல்களைச் சேமிக்காது மற்றும் சேமிக்கக்கூடாது என்பதை கவனத்தில் கொள்ள வேண்டும்.

## Node சேவையைப் பயன்படுத்துவதன் நன்மைகள் என்ன? {#benefits-of-using-a-node-service}

Node சேவையைப் பயன்படுத்துவதற்கான முக்கிய நன்மை என்னவென்றால், node-களை நீங்களே பராமரிக்கவும் நிர்வகிக்கவும் பொறியியல் நேரத்தை செலவிட வேண்டியதில்லை. இது உள்கட்டமைப்பு பராமரிப்பைப் பற்றி கவலைப்படாமல் உங்கள் தயாரிப்பை உருவாக்குவதில் கவனம் செலுத்த அனுமதிக்கிறது.

உங்களுக்கென்று நொடிகளை இயக்குவது சேமிப்பு, பரவலை, மற்றும் மதிப்புமிகு மின் இன்ஜினியராக இருந்தால்கூட மிக அதிகமாக சிக்கலை ஏற்படுத்தலாம். அளவீட்டைத் தூண்டும் போது மேலும் நொடிகளை உருவாக்குவது, நொடிகளை புதிய பதிப்புகளுக்கு மேம்படுத்துவது, மற்றும் நிலை ஜனீதியை உறுதி செய்வது போன்ற விஷயங்கள், உங்களின் விருப்பத்தின் அடிப்படையில் உள்ள வெப்3 தயாரிப்பை உருவாக்குவதில் மற்றும் வளங்களை செலவிடுவதில் அணுகக் கலக்கலாம்.

## ஒரு முனை சேவையைப் பயன்படுத்துவதன் தீமைகள் என்ன? {#cons-of-using-a-node-service}

ஒரு முனை சேவையைப் பயன்படுத்துவதன் மூலம் உங்கள் தயாரிப்பின் உள்கட்டமைப்பு அம்சத்தை நீங்கள் மையப்படுத்துகிறீர்கள். இந்த காரணத்திற்காக, பரவலாக்கத்திற்கு அதிக முக்கியத்துவம் கொடுக்கும் திட்டங்கள், 3ஆம் தரப்பினருக்கு அவுட்சோர்சிங் செய்வதை விட, தாங்களே முனைகளை ஹோஸ்ட் செய்ய விரும்பலாம்.

[உங்கள் சொந்த முனையை இயக்குவதன் நன்மைகள்](/developers/docs/nodes-and-clients/#benefits-to-you) பற்றி மேலும் படிக்கவும்.
