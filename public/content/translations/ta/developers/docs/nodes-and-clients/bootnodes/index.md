ஒரு புதிய முனை எத்தேரியம் நெட்வொர்க்கில் சேரும்போது, புதிய பியர்ஸ்களைக் கண்டறிவதற்காக அது ஏற்கனவே நெட்வொர்க்கில் உள்ள முனைகளுடன் இணைக்கப்பட வேண்டும். எத்தேரியம் நெட்வொர்க்கில் உள்ள இந்த நுழைவுப் புள்ளிகள் பூட்நோட்ஸ் என்று அழைக்கப்படுகின்றன. வாடிக்கையாளர்கள் வழக்கமாக பூட்நோட்ஸ்களின் பட்டியலை அவற்றில் ஹார்ட்கோட் செய்திருப்பார்கள். இந்த பூட்நோட்கள் பொதுவாக எத்தேரியம் ஃபவுண்டேஷனின் டெவலப்ஸ் குழு அல்லது வாடிக்கையாளர் குழுக்களால் இயக்கப்படுகின்றன. பூட்நோட்கள் நிலையான முனைகளைப் போன்றது அல்ல என்பதை கவனத்தில் கொள்ளவும். நிலையான முனைகள் மீண்டும் மீண்டும் அழைக்கப்படுகின்றன, ஆனால் இணைக்க போதுமான பியர்ஸ்கள் இல்லாதபோதும், ஒரு முனைக்கு சில புதிய இணைப்புகளை பூட்ஸ்டிராப் செய்ய வேண்டியிருக்கும் போதும் மட்டுமே பூட்நோட்கள் அழைக்கப்படுகின்றன.

## ஒரு பூட்நோட்டுடன் இணைக்கவும் {#connect-to-a-bootnode}

பெரும்பாலான வாடிக்கையாளர்கள் பூட்நோட்களின் உள்ளமைக்கப்பட்ட பட்டியலைக் கொண்டுள்ளனர், ஆனால் நீங்கள் உங்கள் சொந்த பூட்நோட்டை இயக்க விரும்பலாம் அல்லது வாடிக்கையாளரின் ஹார்ட்கோட் செய்யப்பட்ட பட்டியலில் இல்லாத ஒன்றைப் பயன்படுத்தலாம். இந்த நிலையில், உங்கள் வாடிக்கையாளரைத் தொடங்கும்போது நீங்கள் அவற்றைப் பின்வருமாறு குறிப்பிடலாம் (உதாரணம் Geth க்கானது, உங்கள் வாடிக்கையாளரின் ஆவணத்தைப் பார்க்கவும்):

```
geth --bootnodes "enode://<node ID>@<IP address>:<port>"
```

## ஒரு பூட்நோட்டை இயக்கவும் {#run-a-bootnode}

பூட்நோட்கள் ஒரு NAT ([நெட்வொர்க் முகவரி மொழிபெயர்ப்பு](https://www.geeksforgeeks.org/network-address-translation-nat/)) க்குப் பின்னால் இல்லாத முழு முனைகளாகும். ஒவ்வொரு முழு முனையும் பொதுவில் கிடைக்கும் வரை ஒரு பூட்நோட்டாக செயல்பட முடியும்.

நீங்கள் ஒரு முனையைத் தொடங்கும்போது, அது உங்கள் [enode](/developers/docs/networking-layer/network-addresses/#enode) ஐப் பதிவு செய்ய வேண்டும், இது மற்றவர்கள் உங்கள் முனையுடன் இணைக்கப் பயன்படுத்தக்கூடிய ஒரு பொது அடையாளங்காட்டியாகும்.

ஒவ்வொரு மறுதொடக்கத்திலும் enode பொதுவாக மீண்டும் உருவாக்கப்படுகிறது, எனவே உங்கள் பூட்நோட்டிற்காக ஒரு நிலையான enode-ஐ எவ்வாறு உருவாக்குவது என்பதை உங்கள் வாடிக்கையாளரின் ஆவணத்தில் பார்க்கவும்.

ஒரு நல்ல பூட்நோட்டாக இருக்க, அதனுடன் இணைக்கக்கூடிய பியர்ஸ்களின் அதிகபட்ச எண்ணிக்கையை அதிகரிப்பது ஒரு நல்ல யோசனையாகும். பல பியர்ஸ்களுடன் ஒரு பூட்நோட்டை இயக்குவது அலைவரிசை தேவையை கணிசமாக அதிகரிக்கும்.

## கிடைக்கக்கூடிய பூட்நோட்கள் {#available-bootnodes}

go-ethereum க்குள் உள்ளமைக்கப்பட்ட பூட்நோட்களின் பட்டியலை [இங்கே](https://github.com/ethereum/go-ethereum/blob/master/params/bootnodes.go#L23) காணலாம். இந்த பூட்நோட்கள் எத்தேரியம் ஃபவுண்டேஷன் மற்றும் கோ-எத்தேரியம் குழுவால் பராமரிக்கப்படுகின்றன.

தன்னார்வலர்களால் பராமரிக்கப்படும் பூட்நோட்களின் பிற பட்டியல்களும் கிடைக்கின்றன. குறைந்தது ஒரு அதிகாரப்பூர்வ பூட்நோட்டையாவது எப்போதும் சேர்ப்பதை உறுதிசெய்து கொள்ளவும், இல்லையெனில் நீங்கள் எக்லிப்ஸ் தாக்குதலுக்கு ஆளாக நேரிடலாம்.
