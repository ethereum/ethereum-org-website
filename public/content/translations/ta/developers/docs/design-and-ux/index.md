எத்திரியத்துடன் வடிவமைப்பதற்கு நீங்கள் புதியவரா? இது உங்களுக்கான சரியான இடம். Web3 வடிவமைப்பு மற்றும் ஆராய்ச்சியின் அடிப்படைகளை உங்களுக்கு அறிமுகப்படுத்த எத்திரியம் சமூகம் வளங்களை எழுதியுள்ளது. உங்களுக்குப் பரிச்சயமான பிற செயலி வடிவமைப்புகளிலிருந்து வேறுபடக்கூடிய முக்கியக் கருத்துகளைப் பற்றி நீங்கள் அறிந்துகொள்வீர்கள்.

முதலில் Web3 பற்றிய இன்னும் அடிப்படையான புரிதல் தேவையா? [**கற்றல் மையத்தைப் (Learn hub)**](/learn/) பார்க்கவும்.

## பயனர் ஆராய்ச்சியிலிருந்து தொடங்குங்கள் {#start-with-user-research}

பயனுள்ள வடிவமைப்பு என்பது பார்வைக்கு ஈர்க்கக்கூடிய பயனர் இடைமுகங்களை உருவாக்குவதைத் தாண்டிச் செல்கிறது. இது பயனரின் தேவைகள், நோக்கங்கள் மற்றும் உந்துதல் காரணிகளைப் பற்றிய ஆழமான புரிதலைப் பெறுவதை உள்ளடக்கியது. எனவே, அனைத்து வடிவமைப்பாளர்களும் தங்கள் பணி திட்டமிட்டபடியும் நோக்கத்துடனும் இருப்பதை உறுதிசெய்ய, [**இரட்டை வைர செயல்முறை (double diamond process)**](<https://en.wikipedia.org/wiki/Double_Diamond_(design_process_model)>) போன்ற ஒரு வடிவமைப்புச் செயல்முறையைப் பின்பற்றுமாறு நாங்கள் மிகவும் பரிந்துரைக்கிறோம்.

தற்போது மிகவும் அழுத்தமான UX சிக்கல்கள் என்ன என்பதை நீங்கள் பார்க்க விரும்பினால், இந்த **[தற்போதைய UX சிக்கல்களின் வரைபடத்தைப்](https://ethux.design/)** பார்க்கவும்.

- [Web3-க்கு அதிக UX ஆராய்ச்சியாளர்கள் மற்றும் வடிவமைப்பாளர்கள் தேவை](https://blog.akasha.org/akasha-conversations-9-web3-needs-more-ux-researchers-and-designers) - தற்போதைய வடிவமைப்பு முதிர்ச்சியின் மேலோட்டப் பார்வை
- [Web3-இல் UX ஆராய்ச்சிக்கான எளிய வழிகாட்டி](https://uxplanet.org/a-complete-guide-to-ux-research-for-web-3-0-products-d6bead20ebb1) - ஆராய்ச்சி செய்வது எப்படி என்பதற்கான எளிய வழிகாட்டி
- [Web3-இல் UX முடிவுகளை எவ்வாறு அணுகுவது](https://archive.devcon.org/archive/watch/6/data-empathy-how-to-approach-ux-decisions-in-web3/) - அளவு மற்றும் பண்புசார் ஆராய்ச்சியின் சுருக்கமான மேலோட்டப் பார்வை மற்றும் இரண்டிற்கும் இடையிலான வேறுபாடுகள் (காணொளி, 6 நிமிடம்)
- [Web3-இல் ஒரு UX ஆராய்ச்சியாளராக இருப்பது](https://medium.com/@georgia.rakusen/what-its-like-being-a-user-researcher-in-web3-6a4bcc096849) - Web3-இல் ஒரு UX ஆராய்ச்சியாளராக இருப்பது எப்படி இருக்கும் என்பது பற்றிய தனிப்பட்ட பார்வை

## Web3-இல் ஆராய்ச்சி ஆய்வுகள் {#research-in-web3}

## Web3-க்கான வடிவமைப்பு {#design-for-web3}

- [Web3 வடிவமைப்பு பிளேபுக்](https://learnweb3.design/) - வடிவமைப்பாளர்கள் மற்றும் நிறுவனர்களுக்கான Web3 UX கொள்கைகள், பரவலாக்கப்பட்ட நிதி (DeFi) வடிவங்கள், ஆளுகை வடிவமைப்பு, பணப்பை UX மற்றும் நெறிமுறை-நிலை சிந்தனை ஆகியவற்றின் கட்டமைப்புகள் மற்றும் குறிப்புகளின் விரிவான தொகுப்பு
- [Web3 UX வடிவமைப்பு கையேடு](https://web3ux.design/) - Web3 செயலிகளை வடிவமைப்பதற்கான நடைமுறை வழிகாட்டி
- [Web3 வடிவமைப்புக் கொள்கைகள்](https://medium.com/@lyricalpolymath/web3-design-principles-f21db2f240c1) - தொகுதிச்சங்கிலி அடிப்படையிலான பரவலாக்கப்பட்ட செயலிகளுக்கான (dapps) UX விதிகளின் கட்டமைப்பு
- [தொகுதிச்சங்கிலி வடிவமைப்புக் கொள்கைகள்](https://medium.com/design-ibm/blockchain-design-principles-599c5c067b6e) - IBM-இல் உள்ள தொகுதிச்சங்கிலி வடிவமைப்புக் குழு கற்றுக்கொண்ட பாடங்கள்
- [Neueux.com](https://neueux.com/apps) - பல்வேறு வடிகட்டுதல் விருப்பங்களைக் கொண்ட பயனர் ஓட்டங்களின் UI நிரலகம்
- [Web3-இன் பயன்பாட்டு நெருக்கடி: நீங்கள் தெரிந்து கொள்ள வேண்டியவை!](https://www.youtube.com/watch?v=oBSXT_6YDzg) - டெவலப்பர்களை மையமாகக் கொண்ட திட்ட உருவாக்கத்தின் ஆபத்துகள் குறித்த குழு விவாதம் (காணொளி, 34 நிமிடம்)
