Ethereum பிளாக்செயினில் உங்கள் முதல் [ஸ்மார்ட் ஒப்பந்தத்துடன்](/developers/docs/smart-contracts/) தொடர்பு கொள்ளவும், அதை [பயன்படுத்தவும்](/developers/docs/smart-contracts/deploying/) எங்களைப் போலவே நீங்களும் உற்சாகமாக இருக்கிறீர்கள் என்று நினைக்கிறேன்.

கவலைப்பட வேண்டாம், இது எங்களின் முதல் ஸ்மார்ட் ஒப்பந்தம் என்பதால், நாங்கள் அதை ஒரு [உள்ளூர் சோதனை நெட்வொர்க்கில்](/developers/docs/networks/) பயன்படுத்துவோம், எனவே நீங்கள் அதைப் பயன்படுத்தவும், நீங்கள் விரும்பும் அளவுக்கு அதனுடன் விளையாடவும் உங்களுக்கு எதுவும் செலவாகாது.

## எங்கள் ஒப்பந்தத்தை எழுதுதல் {#writing-our-contract}

முதல் படி [ரீமிக்ஸ்-ஐப் பார்வையிட்டு](https://remix.ethereum.org/) ஒரு புதிய கோப்பை உருவாக்குவதாகும். ரீமிக்ஸ் இடைமுகத்தின் மேல் இடது பகுதியில் ஒரு புதிய கோப்பைச் சேர்த்து, நீங்கள் விரும்பும் கோப்பு பெயரை உள்ளிடவும்.

![ரீமிக்ஸ் இடைமுகத்தில் ஒரு புதிய கோப்பைச் சேர்த்தல்](./remix.png)

புதிய கோப்பில், பின்வரும் குறியீட்டை ஒட்டுவோம்.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity >=0.5.17;

contract Counter {

    // எண்ணிக்கைகளின் எண்ணிக்கையை வைத்திருக்க, கையொப்பமிடப்படாத முழு எண் வகையின் பொது மாறி
    uint256 public count = 0;

    // நமது கவுண்டரை அதிகரிக்கும் செயல்பாடு
    function increment() public {
        count += 1;
    }

    // எண்ணிக்கை மதிப்பைப் பெற தேவையற்ற கெட்டர்
    function getCount() public view returns (uint256) {
        return count;
    }

}
```

நீங்கள் புரோகிராமிங்கில் பழகியிருந்தால், இந்த புரோகிராம் என்ன செய்கிறது என்பதை எளிதாக யூகிக்க முடியும். வரிக்கு வரி விளக்கம் இங்கே:

- வரி 4: `Counter` என்ற பெயருடன் ஒரு ஒப்பந்தத்தை வரையறுக்கிறோம்.
- வரி 7: எங்கள் ஒப்பந்தம் 0-இல் தொடங்கி `count` எனப் பெயரிடப்பட்ட ஒரு கையொப்பமிடப்படாத முழு எண்ணைச் சேமிக்கிறது.
- வரி 10: முதல் செயல்பாடு ஒப்பந்தத்தின் நிலையை மாற்றி, நமது `count` மாறியை `increment()` செய்யும்.
- இரண்டாவது செயல்பாடு, ஸ்மார்ட் ஒப்பந்தத்திற்கு வெளியே `count` மாறியின் மதிப்பைப் படிக்க உதவும் ஒரு கெட்டர் மட்டுமே. எங்களின் `count` மாறியை நாங்கள் பொதுவானதாக வரையறுத்துள்ளதால், இது அவசியமில்லை ஆனால் ஒரு உதாரணமாகக் காட்டப்பட்டுள்ளது என்பதை நினைவில் கொள்ளவும்.

எங்களின் முதல் எளிய ஸ்மார்ட் ஒப்பந்தத்திற்கு இதுவே போதுமானது. உங்களுக்குத் தெரிந்திருக்கலாம், இது ஜாவா அல்லது C++ போன்ற OOP (பொருள்-சார்ந்த நிரலாக்க) மொழிகளிலிருந்து ஒரு கிளாஸ் போல தோன்றுகிறது. இப்போது எங்கள் ஒப்பந்தத்துடன் விளையாட வேண்டிய நேரம் இது.

## எங்கள் ஒப்பந்தத்தை பயன்படுத்துதல் {#deploying-our-contract}

எங்களின் முதல் ஸ்மார்ட் ஒப்பந்தத்தை நாங்கள் எழுதியதால், இப்போது அதனுடன் விளையாட பிளாக்செயினில் பயன்படுத்துவோம்.

[பிளாக்செயினில் ஸ்மார்ட் ஒப்பந்தத்தை பயன்படுத்துவது](/developers/docs/smart-contracts/deploying/) என்பது உண்மையில், எந்த பெறுநர்களையும் குறிப்பிடாமல், தொகுக்கப்பட்ட ஸ்மார்ட் ஒப்பந்தத்தின் குறியீட்டைக் கொண்ட ஒரு பரிவர்த்தனையை அனுப்புவது மட்டுமே.

இடது பக்கத்தில் உள்ள கம்பைல் ஐகானைக் கிளிக் செய்வதன் மூலம் நாங்கள் முதலில் [ஒப்பந்தத்தைத் தொகுப்போம்](/developers/docs/smart-contracts/compiling/):

![ரீமிக்ஸ் கருவிப்பட்டியில் உள்ள கம்பைல் ஐகான்](./remix-compile-button.png)

பின்னர் கம்பைல் பொத்தானைக் கிளிக் செய்யவும்:

![ரீமிக்ஸ் சொலிடிட்டி கம்பைலரில் உள்ள கம்பைல் பொத்தான்](./remix-compile.png)

நீங்கள் “தானாகத் தொகுத்தல்” விருப்பத்தைத் தேர்ந்தெடுக்கலாம், இதன் மூலம் நீங்கள் உரை திருத்தியில் உள்ளடக்கத்தைச் சேமிக்கும்போது ஒப்பந்தம் எப்போதும் தொகுக்கப்படும்.

பின்னர் "பரிவர்த்தனைகளை பயன்படுத்தவும் மற்றும் இயக்கவும்" திரைக்குச் செல்லவும்:

![ரீமிக்ஸ் கருவிப்பட்டியில் உள்ள பயன்படுத்தும் ஐகான்](./remix-deploy.png)

நீங்கள் "பரிவர்த்தனைகளை பயன்படுத்தவும் மற்றும் இயக்கவும்" திரையில் வந்ததும், உங்கள் ஒப்பந்தத்தின் பெயர் தோன்றுவதை இருமுறை சரிபார்த்து, பயன்படுத்து என்பதைக் கிளிக் செய்யவும். பக்கத்தின் மேலே நீங்கள் காணக்கூடியது போல, தற்போதைய சூழல் “JavaScript VM” ஆகும், அதாவது வேகமாகவும் எந்தக் கட்டணமும் இல்லாமல் சோதனை செய்யவும், எங்கள் ஸ்மார்ட் ஒப்பந்தத்தை ஒரு உள்ளூர் சோதனை பிளாக்செயினில் நாங்கள் பயன்படுத்துவோம் மற்றும் ஊடாடுவோம்.

![ரீமிக்ஸ் சொலிடிட்டி கம்பைலரில் உள்ள பயன்படுத்தும் பொத்தான்](./remix-deploy-button.png)

நீங்கள் “Deploy” பொத்தானைக் கிளிக் செய்தவுடன், உங்கள் ஒப்பந்தம் கீழே தோன்றுவதைக் காண்பீர்கள். அதை விரிவாக்க இடதுபுறத்தில் உள்ள அம்புக்குறியைக் கிளிக் செய்யவும், இதன்மூலம் எங்கள் ஒப்பந்தத்தின் உள்ளடக்கத்தை நாங்கள் காண்போம். இது நமது `counter` மாறி, நமது `increment()` செயல்பாடு மற்றும் `getCounter()` கெட்டர்.

நீங்கள் `count` அல்லது `getCount` பொத்தானைக் கிளிக் செய்தால், அது உண்மையில் ஒப்பந்தத்தின் `count` மாறியின் உள்ளடக்கத்தை மீட்டெடுத்து அதைக் காண்பிக்கும். நாங்கள் இன்னும் `increment` செயல்பாட்டை அழைக்காததால், அது 0-ஐக் காட்ட வேண்டும்.

![ரீமிக்ஸ் சொலிடிட்டி கம்பைலரில் உள்ள செயல்பாட்டு பொத்தான்](./remix-function-button.png)

இப்போது பொத்தானைக் கிளிக் செய்வதன் மூலம் `increment` செயல்பாட்டை அழைப்போம். சாளரத்தின் கீழே செய்யப்படும் பரிவர்த்தனைகளின் பதிவுகள் தோன்றுவதை நீங்கள் காண்பீர்கள். `increment` பொத்தானுக்குப் பதிலாக தரவை மீட்டெடுக்க பொத்தானை அழுத்தும்போது பதிவுகள் வேறுபட்டிருப்பதைக் காண்பீர்கள். ஏனெனில் பிளாக்செயினில் தரவைப் படிப்பதற்கு எந்தப் பரிவர்த்தனைகளும் (எழுதுதல்) அல்லது கட்டணங்களும் தேவையில்லை. ஏனெனில் பிளாக்செயினின் நிலையை மாற்றுவதற்கு மட்டுமே ஒரு பரிவர்த்தனை செய்ய வேண்டும்:

![பரிவர்த்தனைகளின் ஒரு பதிவு](./transaction-log.png)

எங்கள் `increment()` செயல்பாட்டை அழைக்க ஒரு பரிவர்த்தனையை உருவாக்கும் இன்க்ரிமென்ட் பொத்தானை அழுத்திய பிறகு, கவுண்ட் அல்லது கெட்கவுண்ட் பொத்தான்களை மீண்டும் கிளிக் செய்தால், எங்கள் ஸ்மார்ட் ஒப்பந்தத்தின் புதிதாகப் புதுப்பிக்கப்பட்ட நிலையை, கவுண்ட் மாறி 0-ஐ விட பெரியதாக இருப்பதைக் காண்போம்.

![ஸ்மார்ட் ஒப்பந்தத்தின் புதிதாகப் புதுப்பிக்கப்பட்ட நிலை](./updated-state.png)

அடுத்த பயிற்சியில், [உங்கள் ஸ்மார்ட் ஒப்பந்தங்களில் நிகழ்வுகளை எவ்வாறு சேர்ப்பது](/developers/tutorials/logging-events-smart-contracts/) என்பதைப் பற்றி பார்ப்போம். நிகழ்வுகளைப் பதிவுசெய்வது உங்கள் ஸ்மார்ட் ஒப்பந்தத்தை பிழைத்திருத்தம் செய்வதற்கும் ஒரு செயல்பாட்டை அழைக்கும்போது என்ன நடக்கிறது என்பதைப் புரிந்துகொள்வதற்கும் ஒரு வசதியான வழியாகும்.
