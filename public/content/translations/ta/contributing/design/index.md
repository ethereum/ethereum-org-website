# ethereum.org-க்கான வடிவமைப்புப் பங்களிப்பு {#design-contributions}

எந்தவொரு திட்டத்திலும், வடிவமைப்பு ஒரு முக்கியமான அங்கமாகும். உங்கள் நேரத்தையும் வடிவமைப்புத் திறன்களையும் ethereum.orgக்கு அர்ப்பணிப்பதன் மூலம், எங்கள் பார்வையாளர்களுக்குச் சிறந்த பயனர் அனுபவத்தை மேம்படுத்த உதவலாம். திறந்த மூல திட்டத்திற்கு பங்களிப்பது, சம்பந்தப்பட்ட அனுபவத்தைப் பெறவும், உங்கள் திறன்களை ஒரு கூட்டு சூழலில் வளர்த்துக்கொள்ளவும் வாய்ப்பளிக்கிறது. மற்ற வடிவமைப்பாளர்கள், உருவாக்குநர்கள் மற்றும் சமூக உறுப்பினர்களுடன் இணைந்து பணியாற்றும் வாய்ப்பு உங்களுக்குக் கிடைக்கும். அவர்கள் அனைவருக்கும் தங்களுக்கே உரிய தனித்தன்மையான கண்ணோட்டங்களும் நுண்ணறிவுகளும் இருக்கும்.

இறுதியில், இது உங்கள் வடிவமைப்புத் திறன்களை வெளிப்படுத்தும் பல்வேறு மற்றும் ஈர்க்கக்கூடிய போர்ட்ஃபோலியோவை உருவாக்க ஒரு சிறந்த வழியாகும்.

## எப்படி பங்களிப்பது?

### <Emoji text=":one:" size={1} /> &nbsp;ஆரம்பகட்ட வடிவமைப்பு முன்மாதிரிகள் குறித்த கருத்துக்களை வழங்குங்கள் {#design-critique}

சில நேரங்களில் புதிய யோசனைகளைச் சோதிக்க உதவி தேவைப்படுகிறது. எந்தத் தொழில்நுட்ப அறிவும் இல்லாமல் பங்களிக்க இது ஒரு சிறந்த வழி.

1. வடிவமைப்புக் குழு [Discord](https://discord.com/invite/ethereum-org) மற்றும் [GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) இல் ஒரு மாதிரி வடிவமைப்பைப் பகிரும்.
2. கருத்துகள் செயல்பாடு வழியாகக் கருத்துகளை வழங்க வடிவமைப்புகள் வழியாக நீங்கள் வழிநடத்தப்படுவீர்கள்.
3. முடிவுகள் கிட்ஹப் பிரச்சினையில் பகிர்ந்து கொள்ளப்பட்டு, பின்னர் குழுவினரால் மூடப்படும்.

### <Emoji text=":two:" size={1} /> &nbsp;கணக்கெடுப்பு ஆராய்ச்சியில் பங்கேற்கவும் {#answer-surveys}

எங்கள் இணையதளத்திற்கு கருத்துகளை வழங்குங்கள்:

1. Ethereum.orgஐ பார்வையிடுங்கள் மற்றும் பக்கங்களை வாசியுங்கள்.
2. கீழ் வலது மூலையில் உள்ள கருத்து விட்ஜெட்டை கிளிக் செய்து, வடிவமைப்பு மற்றும் உள்ளடக்கம் தொடர்பான கேள்விகளுக்குப் பதிலளியுங்கள்.
3. இலவச வடிவ கேள்விகளில் கவனம் செலுத்துங்கள்.

### <Emoji text=":three:" size={1} /> &nbsp;இணையதளத்தில் வடிவமைப்பு தொடர்பான சிக்கல்களைக் கண்டறிந்து அவற்றைப் புகாரளிக்கவும் {#report-design-issues}

ethereum.org பல அம்சங்கள் மற்றும் உள்ளடக்கங்களுடன் வேகமாக வளரும் இணையதளமாகும். சில UIகள் எளிதில் பழமையானவை அல்லது மேம்படுத்தப்படலாம். நீங்கள் அத்தகைய எந்த உதாரணத்தையும் சந்தித்தால், அதை எங்கள் கவனத்திற்கு கொண்டு வர அறிவியுங்கள்.

1. இணையதளத்தைப் பார்வையிடுங்கள் மற்றும் அதன் வடிவமைப்பில் கவனம் செலுத்துங்கள்.
2. நீங்கள் ஏதேனும் காட்சி அல்லது UX சிக்கல்களைக் கண்டால், ஸ்க்ரீன்ஷாட்களையும் குறிப்புகளையும் எடுங்கள்.
3. ஒரு [பிழை அறிக்கையைப்](https://github.com/ethereum/ethereum-org-website/issues/new/choose) பயன்படுத்தி கண்டறியப்பட்ட சிக்கல்களைப் புகாரளிக்கவும்.

### <Emoji text=":four:" size={1} /> &nbsp;வடிவமைப்பு மாற்றங்களை முன்மொழியுங்கள் {#propose-design-changes}

வடிவமைப்பு சவால்களை எதிர்கொள்வதில் நீங்கள் வசதியாக உணர்ந்தால், எங்களின் GitHub சிக்கல்கள் பலகையைப் பார்வையிட்டு [வடிவமைப்பு தொடர்பான சிக்கல்களுக்கு](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) வடிகட்டலாம்.

1. எங்கள் இணையதளத்தைப் பார்வையிட்டு அதன் வடிவமைப்பில் கவனம் செலுத்துங்கள் அல்லது எங்கள் GitHub ரெபாசிட்டரிக்குச் சென்று [“வடிவமைப்பு தேவை” குறிச்சொல்லுடன்](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) கொடியிடப்பட்ட சிக்கல்களை மதிப்பாய்வு செய்யவும்.
2. தீர்வை யோசித்து அதை வடிவமைக்குங்கள். (முன்னுரிமை அளிக்கும் வகையில் எங்கள் [வடிவமைப்பு அமைப்பைப்](https://www.figma.com/community/file/1134414495420383395) பயன்படுத்துதல்).
3. தொடர்புடைய GitHub சிக்கலில் தீர்வை முன்மொழியுங்கள் அல்லது [புதிய ஒன்றை உருவாக்குங்கள்.](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. வடிவமைப்புக் குழு மதிப்பாய்வு செய்யக் காத்திருங்கள்.

### <Emoji text=":five:" size={1} /> &nbsp;வடிவமைப்பு அமைப்பை ஒன்றிணைந்து உருவாக்குங்கள் {#Contribute-to-design-system}

எங்கள் வடிவமைப்பு அமைப்பு ethereum.orgஐ வடிவமைப்பதை வேடிக்கையாகவும் எளிதாகவும் ஆக்குகிறது. நீங்கள் அனுபவமிக்க வடிவமைப்பாளராக இருந்தால், எங்கள் இணையதளத்திற்கு பல கூறுகளைத் தயாரிக்க உதவலாம்.

1. GitHub இல் உள்ள [வடிவமைப்பு அமைப்புப் பலகையிலிருந்து](https://github.com/ethereum/ethereum-org-website/labels/design%20system) பணியாற்றுவதற்கு ஒரு சிக்கலைத் தேர்ந்தெடுக்கவும் அல்லது புதிய ஒன்றை உருவாக்கவும்.
2. தேர்ந்தெடுக்கப்பட்ட இஷ்யூ, உங்களுக்கு ஒதுக்கப்பட வேண்டும் என்று கோருங்கள்.
3. ஃபிக்மாவில் கோரப்பட்ட கூறை வடிவமைக்கத் தொடங்குங்கள்.
4. மதிப்பாய்வு அல்லது வழிகாட்டல் தேவைப்பட்டால, அதைக் கிட்ஹப்பில் வடிவமைப்புக் குழுவுடன் பகிருங்கள்.
5. வடிவமைப்புக் குழு மதிப்பாய்வு செய்யும்.
6. வடிவமைப்புக் குழு மாற்றங்களை முதன்மை கோப்பில் இணைத்து, கோப்பை சமூகத்திற்கு வெளியிடும்.

### <Emoji text=":six:" size={1} /> &nbsp;இணையதளத்தில் வடிவமைப்பு தொடர்பான உள்ளடக்கத்தை எழுதுங்கள் {#write-design-articles}

எத்தேரியம் உருவாக்குநர் சமூகம் வலுவானது, எனினும் வடிவமைப்பாளர் கூட்டம் ஓரளவு பின்னடைவில் இருக்கிறது. நீங்கள் Web3 அறிவு கொண்ட வடிவமைப்பாளராக இருந்தால், தயவுசெய்து உங்கள் கற்றல்களைப் பரந்த சமூகத்துடன் பகிர்ந்து கொள்ளுங்கள், இதன் மூலம் நாம் அனைவரும் ஒன்றாக வளர்ந்து மேம்பட முடியும்; எங்களிடம் [Ethereum வடிவமைப்பது பற்றிய ஒரு பக்கம்](/developers/docs/design-and-ux/) உள்ளது, அதற்கு நீங்கள் பங்களிக்கலாம். எங்கள் [பட்டியலிடல் கொள்கைகளையும்](/contributing/design/adding-design-resources) நீங்கள் பார்க்கலாம்.

1. Ethereum.orgஇல் உள்ளடக்கப்பட வேண்டிய வடிவமைப்புத் தலைப்புகளை யோசித்து, அவை இந்தத் துறையில் உள்ள வடிவமைப்பாளர்களுக்குப் பயனுள்ளதாக இருக்கும்.
2. எங்கள் GitHub ரெபாசிட்டரிக்குச் சென்று, ஒரு தலைப்பை முன்மொழிந்து [ஒரு சிக்கலை எழுப்புங்கள்](https://github.com/ethereum/ethereum-org-website/issues/new) (இன்னும் உள்ளடக்கத்தை எழுத வேண்டாம்).
3. வடிவமைப்புக் குழு அங்கீகரிக்கக் காத்திருங்கள்.
4. அங்கீகரிக்கப்பட்டவுடன், உள்ளடக்கத்தை எழுதுங்கள்.
5. அதை அதற்குரிய கிட்ஹப் இஷ்யூவில் சமர்ப்பியுங்கள்.

### <Emoji text=":seven:" size={1} /> &nbsp;புதிய விளக்கப்படங்களை வரையுங்கள் {#prepare-illustrations}

சுருக்கமான தலைப்புகளை விளக்குவதற்கு விளக்கப்படங்கள் மிகச் சக்திவாய்ந்த கருவிகளில் ஒன்று. வரைபடங்கள் மற்றும் இன்ஃபோகிராஃபிக்ஸை சேர்ப்பதன் மூலம் பெரும் சாத்தியக்கூறு உள்ளது. எல்லாவற்றிற்கும் மேலாக, ஒரு படம் ஆயிரம் சொற்களுக்குச் சமானம்.

1. எங்கள் இணையதளத்திற்கு சென்று, புதிய இன்ஃபோகிராஃபிக்ஸ் சேர்க்கக்கூடிய பக்கங்களைப் பாருங்கள்.
2. விளக்கப்படப் பாணி எங்கள் [சொத்துக்களுடன்](/assets/) பொருந்துவதை உறுதிசெய்யுங்கள்.
3. எங்கள் GitHub ரெபாசிட்டரிக்குச் சென்று, விளக்கப்படத்தை முன்மொழிந்து [ஒரு சிக்கலை எழுப்புங்கள்](https://github.com/ethereum/ethereum-org-website/issues/new).
4. வடிவமைப்புக் குழு அதை மதிப்பாய்வு செய்யும்.
5. புதிய படத்தைச் நடைமுறையில் செயல்படுத்த உருவாக்குநரிடம் வேண்டுகோள் வைக்கப் புதிய இஷ்யூவை உருவாக்குகிறோம்.
