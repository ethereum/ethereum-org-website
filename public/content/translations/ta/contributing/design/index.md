# ethereum.org-க்கு வடிவமைப்பு பங்களிப்பு {#design-contributions}

எந்தவொரு திட்டத்திற்கும் வடிவமைப்பு ஒரு முக்கியமான அங்கமாகும், மேலும் உங்கள் நேரத்தையும் வடிவமைப்பு திறன்களையும் ethereum.org-க்கு அர்ப்பணிப்பதன் மூலம், எங்கள் பார்வையாளர்களுக்கான பயனர் அனுபவத்தை மேம்படுத்த நீங்கள் உதவலாம். ஒரு ஓப்பன் சோர்ஸ் (open-source) திட்டத்திற்குப் பங்களிப்பது, கூட்டுச் சூழலில் தொடர்புடைய அனுபவத்தைப் பெறவும் உங்கள் திறன்களை வளர்த்துக் கொள்ளவும் ஒரு வாய்ப்பை வழங்குகிறது. மற்ற வடிவமைப்பாளர்கள், டெவலப்பர்கள் மற்றும் சமூக உறுப்பினர்களுடன் இணைந்து பணியாற்ற உங்களுக்கு வாய்ப்பு கிடைக்கும், அவர்கள் அனைவரும் தங்களுக்குரிய தனித்துவமான கண்ணோட்டங்களையும் நுண்ணறிவுகளையும் கொண்டிருப்பார்கள்.

இறுதியாக, உங்கள் வடிவமைப்புத் திறன்களை வெளிப்படுத்தும் பன்முகத்தன்மை கொண்ட மற்றும் ஈர்க்கக்கூடிய போர்ட்ஃபோலியோவை (portfolio) உருவாக்க இது ஒரு சிறந்த வழியாகும்.

## எப்படிப் பங்களிப்பது?

### <Emoji text=":one:" size={1} /> &nbsp;ஆரம்பகால வடிவமைப்பு முன்மாதிரிகள் குறித்த கருத்துகளை வழங்கவும் {#design-critique}

எங்கள் ஆரம்பகால யோசனைகளைச் சோதிக்க சில நேரங்களில் எங்களுக்கு உதவி தேவைப்படுகிறது. எந்தவொரு தொழில்நுட்ப அறிவும் இல்லாமல் பங்களிக்க இது ஒரு சிறந்த வழியாகும்.

1. வடிவமைப்புக் குழு ஒரு மாதிரி வடிவமைப்பை (mockup design) [Discord](https://discord.com/invite/ethereum-org) மற்றும் [GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8)-இல் பகிரும்.
2. கருத்துகள் (comments) செயல்பாட்டின் மூலம் பின்னூட்டங்களை வழங்க வடிவமைப்புகள் மூலம் நீங்கள் வழிநடத்தப்படுவீர்கள்.
3. இதன் முடிவு GitHub சிக்கலில் (issue) பகிரப்பட்டு, பின்னர் குழுவால் மூடப்படும்.

### <Emoji text=":two:" size={1} /> &nbsp;கணக்கெடுப்பு ஆராய்ச்சியில் பங்கேற்கவும் {#answer-surveys}

இதன் மூலம் எங்கள் இணையதளம் குறித்த கருத்துகளை வழங்கவும்:

1. ethereum.org-ஐப் பார்வையிட்டு பக்கங்களைப் படிப்பதன் மூலம்.
2. கீழ் வலது மூலையில் உள்ள பின்னூட்ட விட்ஜெட்டைக் (feedback widget) கிளிக் செய்து, வடிவமைப்பு மற்றும் உள்ளடக்கம் தொடர்பான கேள்விகளுக்குப் பதிலளிப்பதன் மூலம்.
3. கட்டற்ற வடிவக் (free format) கேள்விகளில் கவனம் செலுத்துங்கள்.

### <Emoji text=":three:" size={1} /> &nbsp;இணையதளத்தில் வடிவமைப்பு தொடர்பான சிக்கல்களைக் கண்டறிந்து அவற்றைப் புகாரளிக்கவும் {#report-design-issues}

ethereum.org என்பது பல அம்சங்கள் மற்றும் உள்ளடக்கங்களைக் கொண்ட வேகமாக வளர்ந்து வரும் இணையதளமாகும். சில UI எளிதில் காலாவதியாகலாம் அல்லது மேம்படுத்தப்படலாம். அத்தகைய நிகழ்வை நீங்கள் சந்தித்தால், தயவுசெய்து அதைப் புகாரளிக்கவும், இதனால் அது எங்கள் கவனத்திற்கு வரும்.

1. இணையதளத்தைப் பார்வையிட்டு அதன் வடிவமைப்பில் கவனம் செலுத்துங்கள்.
2. ஏதேனும் காட்சி அல்லது UX சிக்கல்களைக் கண்டால் ஸ்கிரீன்ஷாட்களையும் குறிப்புகளையும் எடுக்கவும்.
3. கண்டறியப்பட்ட சிக்கல்களை [பிழை அறிக்கை (bug report)](https://github.com/ethereum/ethereum-org-website/issues/new/choose) மூலம் புகாரளிக்கவும்.

### <Emoji text=":four:" size={1} /> &nbsp;வடிவமைப்பு மாற்றங்களை முன்மொழியவும் {#propose-design-changes}

வடிவமைப்பு சவால்களை ஏற்றுக்கொள்வதில் உங்களுக்கு வசதியாக இருந்தால், நீங்கள் எங்கள் GitHub சிக்கல்கள் பலகையைப் பார்வையிடலாம் மற்றும் [வடிவமைப்பு தொடர்பான சிக்கல்களை (design-related issues)](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) வடிகட்டலாம்.

1. எங்கள் இணையதளத்தைப் பார்வையிட்டு அதன் வடிவமைப்பில் கவனம் செலுத்துங்கள் அல்லது எங்கள் GitHub களஞ்சியத்திற்குச் சென்று [“Design required” குறிச்சொல்லுடன் (tag)](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) கொடியிடப்பட்ட சிக்கல்களை மதிப்பாய்வு செய்யவும்.
2. தீர்வு குறித்து சிந்தித்து அதை வடிவமைக்கவும். (வெகுவாக எங்கள் [வடிவமைப்பு அமைப்பைப் (design system)](https://www.figma.com/community/file/1134414495420383395) பயன்படுத்தி).
3. தொடர்புடைய GitHub சிக்கலில் தீர்வை முன்மொழியவும் அல்லது [புதிய ஒன்றை உருவாக்கவும்.](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. வடிவமைப்புக் குழு மதிப்பாய்வு செய்யும் வரை காத்திருக்கவும்.

### <Emoji text=":five:" size={1} /> &nbsp;வடிவமைப்பு அமைப்பை ஒன்றாக உருவாக்குவோம் {#Contribute-to-design-system}

எங்கள் வடிவமைப்பு அமைப்பு ethereum.org-ஐ வடிவமைப்பதை வேடிக்கையாகவும் எளிதாகவும் ஆக்குகிறது. நீங்கள் ஒரு அனுபவமிக்க வடிவமைப்பாளராக இருந்தால், இணையதளத்திற்கான பல கூறுகளைத் தயாரிக்க எங்களுக்கு உதவலாம்.

1. GitHub-இல் உள்ள [வடிவமைப்பு அமைப்பு பலகையிலிருந்து (design system board)](https://github.com/ethereum/ethereum-org-website/labels/design%20system) வேலை செய்ய ஒரு சிக்கலைத் தேர்ந்தெடுக்கவும் அல்லது புதிய ஒன்றை உருவாக்கவும்.
2. தேர்ந்தெடுக்கப்பட்ட சிக்கலை உங்களுக்கு ஒதுக்குமாறு கோரவும்.
3. கோரப்பட்ட கூறுகளை Figma-இல் வடிவமைக்கத் தொடங்குங்கள்.
4. உங்களுக்கு மதிப்பாய்வு அல்லது வழிகாட்டுதல் தேவைப்பட்டவுடன் அதை GitHub-இல் வடிவமைப்புக் குழுவுடன் பகிரவும்.
5. வடிவமைப்புக் குழு மதிப்பாய்வு செய்யும்.
6. வடிவமைப்புக் குழு முக்கிய கோப்பில் மாற்றங்களை இணைத்து, கோப்பை சமூகத்திற்கு வெளியிடும்.

### <Emoji text=":six:" size={1} /> &nbsp;இணையதளத்தில் வடிவமைப்பு தொடர்பான உள்ளடக்கத்தை எழுதவும் {#write-design-articles}

Ethereum டெவலப்பர் சமூகம் வலுவாக உள்ளது, ஆனால் வடிவமைப்பு சமூகம் சற்று பின்தங்கியுள்ளது. நீங்கள் web3 அறிவு கொண்ட வடிவமைப்பாளராக இருந்தால், உங்கள் கற்றல்களைப் பெரிய சமூகத்துடன் பகிர்ந்துகொள்ளப் பரிசீலிக்கவும், இதனால் நாம் அனைவரும் ஒன்றாக வளர்ந்து மேம்பட முடியும்; நீங்கள் பங்களிக்கக்கூடிய [Ethereum-க்கான வடிவமைப்பு குறித்த பக்கம்](/developers/docs/design-and-ux/) எங்களிடம் உள்ளது. எங்கள் [பட்டியலிடல் கொள்கைகளையும் (listing policies)](/contributing/design/adding-design-resources) நீங்கள் சரிபார்க்கலாம்.

1. ethereum.org-இல் உள்ளடக்கப்பட வேண்டிய மற்றும் இந்தத் துறையில் உள்ள வடிவமைப்பாளர்களுக்குப் பயனளிக்கும் வடிவமைப்புத் தலைப்புகளைப் பற்றி சிந்தியுங்கள்.
2. எங்கள் GitHub களஞ்சியத்திற்குச் சென்று ஒரு தலைப்பை முன்மொழிந்து [ஒரு சிக்கலை எழுப்புங்கள் (raise an issue)](https://github.com/ethereum/ethereum-org-website/issues/new) (உள்ளடக்கத்தை இன்னும் எழுத வேண்டாம்).
3. வடிவமைப்புக் குழு ஒப்புதல் அளிக்கும் வரை காத்திருக்கவும்.
4. ஒப்புதல் கிடைத்தவுடன், உள்ளடக்கத்தை எழுதவும்.
5. தொடர்புடைய GitHub சிக்கலில் அதைச் சமர்ப்பிக்கவும்.

### <Emoji text=":seven:" size={1} /> &nbsp;புதிய விளக்கப்படங்களை வரையவும் {#prepare-illustrations}

சுருக்கமான தலைப்புகளை விளக்குவதற்கு காட்சிப்படுத்தல்கள் மிகவும் சக்திவாய்ந்த கருவிகளில் ஒன்றாகும். வரைபடங்கள் மற்றும் இன்போகிராபிக்ஸ் (infographics) சேர்ப்பதன் மூலம் மிகப்பெரிய சாத்தியக்கூறுகள் உள்ளன. எல்லாவற்றிற்கும் மேலாக, ஒரு படம் ஆயிரம் வார்த்தைகளைச் சொல்லும்.

1. எங்கள் இணையதளத்திற்குச் சென்று சில புதிய இன்போகிராபிக்ஸ் சேர்க்கக்கூடிய பக்கங்களைப் பார்க்கவும்.
2. விளக்கப்படத்தின் பாணி எங்கள் [சொத்துகளுக்கு (assets)](/assets/) ஒத்திருப்பதை உறுதிசெய்யவும்.
3. எங்கள் GitHub களஞ்சியத்திற்குச் சென்று விளக்கப்படத்தை முன்மொழிந்து [ஒரு சிக்கலை எழுப்புங்கள்](https://github.com/ethereum/ethereum-org-website/issues/new).
4. வடிவமைப்புக் குழு அதை மதிப்பாய்வு செய்யும்.
5. புதிய படத்தைச் செயல்படுத்த டெவலப்பரைக் கேட்க புதிய சிக்கலை உருவாக்குகிறோம்.
