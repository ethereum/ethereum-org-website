# ஸ்டேக்கிங் தயாரிப்புகள் அல்லது சேவைகளைச் சேர்த்தல் {#adding-staking-products-or-services}

பயனர்களை பாதுகாப்பாகவும் நம்பிக்கையுடன் வைத்துக் கொள்ளவும், சிறந்த வளங்களை பட்டியலிடுவதை உறுதிப்படுத்த விரும்புகிறோம்.

யாரும் ethereum.org-இல் ஒரு staking தயாரிப்பு அல்லது சேவையைச் சேர்க்க பரிந்துரை செய்யலாம். நாங்கள் தவறவிட்ட ஒன்று இருந்தால், **[தயவுசெய்து அதைப் பரிந்துரைக்கவும்](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A%2Ccontent+%3Afountain_pen%3A&template=suggest_staking_product.yaml)!**

தற்போது நாம் பின்வரும் பக்கங்களில் staking தயாரிப்புகள் மற்றும் சேவைகளை பட்டியலிடுகிறோம்:

- [தனிநபர் ஸ்டேக்கிங்](/staking/solo/)
- [ஒரு சேவையாக ஸ்டேக்கிங்](/staking/saas/)
- [ஸ்டேக்கிங் குளங்கள்](/staking/pools/)

Beacon Chain-இல் Proof-of-stake 2020 டிசம்பர் 1 முதல் நேரலையில் உள்ளது. Staking இன்னும் சற்றே புதியதாக இருந்தாலும், ethereum.org-இல் பரிந்துரைக்க ஒரு நீதி மற்றும் வெளிப்படையான கட்டமைப்பை உருவாக்க முயற்சித்துள்ளோம். ஆனால், பட்டியல் அளவுகோல்கள் காலத்தோடு மாறி வளரும், இறுதியாக ethereum.org வலைத்தளக் குழுவின் தீர்மானத்தின் படி இருக்கும்.

## முடிவு எடுக்கும் கட்டமைப்பு {#the-decision-framework}

Ethereum.org-இல் ஒரு தயாரிப்பை பட்டியலிடும் முடிவு ஒரே காரணத்தின் அடிப்படையில் செய்யப்படாது. ஒரு தயாரிப்பு அல்லது சேவையை பட்டியலிடும் போது பல அளவுகோல்கள் ஒன்றாக பரிசீலிக்கப்படுகின்றன. இவ்வளவுகோல்களில் அதிகம் பூர்த்தி செய்யப்படும்போது, அது பட்டியலிடப்பட வாய்ப்பு அதிகமாகும்.

**முதலில், இது எந்த வகை தயாரிப்பு அல்லது சேவை?**

- Node அல்லது client tools
- முக்கிய மேலாண்மை (Key management)
- ஸ்டாக்கிங் ஒரு சேவையாக (SaaS)
- Staking pool

தற்போது, இந்த வகைகளில் மட்டுமே தயாரிப்புகள் அல்லது சேவைகள் பட்டியலிடப்பட்டுள்ளன.

### சேர்ப்பதற்கான அளவுகோல்கள் {#criteria-for-inclusion}

Staking தயாரிப்புகள் அல்லது சேவைகள் சமர்ப்பிப்புகள் பின்வரும் அளவுகோல்களின் அடிப்படையில் மதிப்பீடு செய்யப்படும்:

**திட்டம் அல்லது சேவை எப்போது தொடங்கப்பட்டது?**

- தயாரிப்பு அல்லது சேவை பொதுமக்களுக்கு எப்போது கிடைக்கத் தொடங்கியது என்பதற்கான சான்றுகள் உள்ளதா?
- இது தயாரிப்பின் “battle tested” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

**திட்டம் தீவிரமாகப் பராமரிக்கப்படுகிறதா?**

- யார் இதில் பங்கேற்கிறார்கள்? யார் இதில் பங்கேற்கிறார்கள்?
- செயற்பாட்டில் பராமரிக்கப்படும் தயாரிப்புகள் மட்டுமே பரிசீலிக்கப்படும்.

**தயாரிப்பு அல்லது சேவை நம்பகமான/மனித இடைத்தரகர்கள் இல்லாமல் உள்ளதா?**

- பயனர்களின் பயணத்தில் எந்த படிகள் மனிதர்களை நம்பி நிதி விசைகளை வைத்திருப்பது அல்லது பரிசுகளை சரியாக விநியோகம் செய்ய வேண்டும் என்பதைத் தேவைபடுத்துகின்றன?
- இது தயாரிப்பு அல்லது சேவையின் “trustless” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

**திட்டம் துல்லியமான மற்றும் நம்பகமான தகவல்களை வழங்குகிறதா?**

- தயாரிப்பின் வலைத்தளம் சமீபத்திய, துல்லியமான மற்றும் தவறான தகவல் இல்லாதவையாக இருக்க வேண்டும். குறிப்பாக அது Ethereum நெறிமுறை அல்லது தொடர்புடைய தொழில்நுட்பங்களுக்கு சம்பந்தப்பட்டதாக இருந்தால்.
- Ethereum அல்லது தொடர்புடைய பிற தலைப்புகளில் தவறான தகவல், பழைய விவரங்கள், அல்லது தவறான விளக்கங்கள் கொண்ட சமர்ப்பிப்புகள் பட்டியலிடப்பட மாட்டாது அல்லது ஏற்கனவே பட்டியலிடப்பட்டிருந்தால் நீக்கப்படும்.

**எந்த தளங்கள் (platforms) ஆதரிக்கப்படுகின்றன?**

- அதாவது, Linux, macOS, Windows, iOS, Android

#### மென்பொருள் மற்றும் ஸ்மார்ட் ஒப்பந்தங்கள் {#software-and-smart-contracts}

ஏதேனும் தனிப்பயன் மென்பொருள் அல்லது ஸ்மார்ட் கான்ட்ராக்ட்கள் இருந்தால்:

**எல்லாம் ஓப்பன் சோர்ஸா?**

- Open source திட்டங்களுக்கு பொதுமக்களுக்கு கிடைக்கக்கூடிய source code repository இருக்க வேண்டும்
- இது தயாரிப்பின் “open source” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

**தயாரிப்பு _பீட்டா_ மேம்பாட்டில் இருந்து வெளிவந்துவிட்டதா?**

- தயாரிப்பு எந்த developmental cycle-இல் உள்ளது?
- Beta நிலைமையிலுள்ள தயாரிப்புகள் ethereum.org-இல் சேர்க்க பரிசீலிக்கப்பட மாட்டாது

**மென்பொருள் வெளிப்புறப் பாதுகாப்புத் தணிக்கைக்கு உட்பட்டுள்ளதா?**

- இல்லையெனில், வெளிப்புற ஆய்வை நடத்த திட்டமிடப்பட்டுள்ளதா?
- இது தயாரிப்பின் “audited” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

**திட்டத்தில் பிழை வெகுமதித் திட்டம் உள்ளதா?**

- இல்லையெனில், ஒரு பாதுகாப்பு bug bounty உருவாக்க திட்டமிடப்பட்டுள்ளதா?
- இது தயாரிப்பின் “bug bounty” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

#### நோட் அல்லது கிளையன்ட் கருவி {#node-or-client-tooling}

Node அல்லது client அமைப்பு, மேலாண்மை அல்லது மாற்றம் தொடர்பான மென்பொருள் தயாரிப்புகளுக்கு:

**எந்த கன்சென்சஸ் லேயர் கிளையன்ட்கள் (அதாவது, Lighthouse, Teku, Nimbus, Prysm, Grandine) ஆதரிக்கப்படுகின்றன?**

- எந்த clients ஆதரிக்கப்படுகின்றன? பயனருக்கு தேர்வு செய்ய வாய்ப்பா?
- இது தயாரிப்பின் “multi-client” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

#### ஒரு சேவையாக ஸ்டேக்கிங் {#staking-as-a-service}

[ஒரு சேவையாக ஸ்டேக்கிங் பட்டியல்களுக்கு](/staking/saas/) (அதாவது, ஒப்படைக்கப்பட்ட நோட் செயல்பாடு):

**சேவையை பயன்படுத்துவதற்கான கட்டணங்கள் என்ன?**

- கட்டண அமைப்பு என்ன, எ.கா., சேவைக்கு மாதாந்திரக் கட்டணம் உள்ளதா?
- வேறு எந்த staking தேவைகள் உள்ளன?

**பயனர்கள் ஒரு கணக்கிற்குப் பதிவு செய்ய வேண்டுமா?**

- யாரும் அனுமதி அல்லது KYC இல்லாமல் சேவையைப் பயன்படுத்த முடியுமா?
- இது தயாரிப்பின் “permissionless” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

**கையொப்பமிடும் சாவிகளையும், திரும்பப் பெறும் சாவிகளையும் யார் வைத்திருக்கிறார்கள்?**

- பயனருக்கு எந்த விசைகளுக்கு அணுகல் உள்ளது? சேவை எந்த விசைகளுக்கு அணுகல் பெறுகிறது? இது தயாரிப்பின் “trustless” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது?
- இது தயாரிப்பின் “trustless” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

**இயங்கப்படும் node-களின் client வித்தியாசம் என்ன?**

- எவ்வளவு சதவீத validator விசைகள் பெரும்பாலான consensus layer (CL) client மூலம் இயங்குகின்றன?
- கடந்த திருத்தம் வரை, Prysm பெரும்பாலான node operators மூலம் இயங்கும் consensus layer client ஆக உள்ளது, இது நெட்வொர்க்குக்கு அபாயகரமாகும். தற்போது எந்த CL client 33% நெட்வொர்க்கை பயன்படுத்துகிறதோ, அதன் பயன்பாட்டு தரவுகளை கேட்கின்றோம்.
- இது தயாரிப்பின் “diverse clients” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

#### ஸ்டேக்கிங் குளம் {#staking-pool}

[குழும ஸ்டேக்கிங் சேவைகளுக்கு](/staking/pools/):

**ஸ்டேக் செய்யத் தேவையான குறைந்தபட்ச ETH எவ்வளவு?**

- எ.கா., 0.01 ETH

**பங்கு கட்டணங்கள் அல்லது staking தேவைகள் என்ன?**

- பரிசுகளின் எத்தனை சதவீதம் கட்டணமாகக் கழிக்கப்படுகிறது?
- வேறு எந்த staking தேவைகள் உள்ளன?

**பணப்புழக்க டோக்கன் உள்ளதா?**

- பங்கு செய்யப்படும் token-கள் எவை? அவை எவ்வாறு செயல்படுகின்றன? ஒப்பந்த முகவரிகள் என்ன?
- இது தயாரிப்பின் “liquidity token” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

**பயனர்கள் ஒரு நோட் ஆபரேட்டராகப் பங்கேற்க முடியுமா?**

- Pooled நிதிகளைப் பயன்படுத்தி validator clients இயக்க என்ன தேவை?
- இது எந்த ஒருவரின், நிறுவனத்தின் அல்லது DAO அனுமதியை தேவையாக்குமா?
- இது தயாரிப்பின் “permissionless nodes” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

**Pool node operators-இன் client வித்தியாசம் என்ன?**

- எவ்வளவு சதவீத node operators பெரும்பாலான consensus layer (CL) client இயக்குகிறார்கள்?
- கடந்த திருத்தம் வரை, Prysm பெரும்பாலான node operators மூலம் இயங்கும் consensus layer client ஆக உள்ளது, இது நெட்வொர்க்குக்கு அபாயகரமாகும். தற்போது எந்த CL client 33% நெட்வொர்க்கை பயன்படுத்துகிறதோ, அதன் பயன்பாட்டு தரவுகளை கேட்கின்றோம்.
- இது தயாரிப்பின் “diverse clients” மதிப்பெண்களை நிர்ணயிக்க பயன்படுத்தப்படுகிறது.

### பிற அளவுகோல்கள்: இருந்தால் நல்லது {#other-criteria}

**பயனர் இடைமுகங்கள் (Ui) எந்தவெல்லாம் ஆதரிக்கப்படுகின்றன?**

- அதாவது, பிரவுசர் செயலி, டெஸ்க்டாப் செயலி, மொபைல் செயலி, CLI

**நோட் கருவிக்கு, மென்பொருளானது கிளையன்ட்களுக்கு இடையில் மாறுவதற்கு எளிதான வழியை வழங்குகிறதா?**

- பயனர் எளிதாகவும் பாதுகாப்பாகவும் clients மாற்ற முடியுமா?

**SaaS-க்கு, தற்போது இந்த சேவையால் எத்தனை சரிபார்ப்பாளர்கள் இயக்கப்படுகிறார்கள்?**

- இது உங்கள் சேவையின் வலிமையை எத்தனை வரை பரவியிருக்கின்றது என்பதை உணர உதவுகிறது.

## முடிவுகளை நாங்கள் எவ்வாறு காண்பிக்கிறோம் {#product-ordering}

மேலே உள்ள [சேர்ப்பதற்கான அளவுகோல்கள்](#criteria-for-inclusion) ஒவ்வொரு தயாரிப்பு அல்லது சேவைக்கும் ஒட்டுமொத்த மதிப்பெண்ணைக் கணக்கிடப் பயன்படுத்தப்படுகின்றன. இது குறிப்பிட்ட பொருளாதார அளவுகோல்களை பூர்த்தி செய்யும் தயாரிப்புகளை வரிசைப்படுத்தவும் காட்சிப்படுத்தவும் பயன்படுகிறது. பல அளவுகோல்களுக்கு ஆதாரங்கள் வழங்கப்பட்டால், தயாரிப்பு மேலே வரிசைப்படுத்தப்படும், சமமான மதிப்பெண்கள் இருந்தால் பதிவேற்றத்தின் போது சீரற்ற முறையில் வரிசைப்படுத்தப்படும்.

இந்த அளவுகோல்களுக்கான குறியீட்டு தர்க்கம் மற்றும் எடைகள் தற்போது எங்கள் ரெப்போவில் உள்ள [இந்த ஜாவாஸ்கிரிப்ட் கூறில்](https://github.com/ethereum/ethereum-org-website/blob/dev/src/components/Staking/StakingProductsCardGrid.js#L350) அடங்கியுள்ளன.
