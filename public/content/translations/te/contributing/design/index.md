ఏదైనా ప్రాజెక్ట్‌లో డిజైన్ అనేది ఒక కీలకమైన భాగం, మరియు మీ సమయాన్ని మరియు డిజైన్ నైపుణ్యాలను ethereum.org కు కేటాయించడం ద్వారా, మీరు మా సందర్శకుల వినియోగదారు అనుభవాన్ని మెరుగుపరచడంలో సహాయపడగలరు. ఓపెన్-సోర్స్ ప్రాజెక్ట్‌కు సహకరించడం ద్వారా సంబంధిత అనుభవాన్ని పొందడానికి మరియు సహకార వాతావరణంలో మీ నైపుణ్యాలను అభివృద్ధి చేసుకోవడానికి అవకాశం లభిస్తుంది. ఇతర డిజైనర్లు, డెవలపర్లు మరియు కమ్యూనిటీ సభ్యులతో కలిసి పనిచేసే అవకాశం మీకు లభిస్తుంది, వారందరికీ వారి స్వంత ప్రత్యేక దృక్కోణాలు మరియు అంతర్దృష్టులు ఉంటాయి.

అంతిమంగా, మీ డిజైన్ నైపుణ్యాలను ప్రదర్శించే విభిన్నమైన మరియు ఆకట్టుకునే పోర్ట్‌ఫోలియోను నిర్మించుకోవడానికి ఇది ఒక గొప్ప మార్గం.

## ఎలా సహకరించాలి? {#how-to-contribute}

### <Emoji text=":one:" size={1} /> &nbsp;ప్రారంభ డిజైన్ నమూనాలపై అభిప్రాయాన్ని అందించండి {#design-critique}

మా ప్రాథమిక ఆలోచనలను పరీక్షించడంలో మాకు కొన్నిసార్లు సహాయం అవసరం. ఎలాంటి సాంకేతిక పరిజ్ఞానం లేకుండా సహకరించడానికి ఇది ఒక గొప్ప మార్గం.

1. డిజైన్ బృందం [డిస్కార్డ్](https://discord.com/invite/ethereum-org) మరియు [GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) లో మాకప్ డిజైన్‌ను పంచుకుంటుంది.
2. కామెంట్స్ ఫంక్షన్ ద్వారా అభిప్రాయాన్ని అందించడానికి డిజైన్‌ల ద్వారా మీకు మార్గనిర్దేశం చేయబడుతుంది.
3. ఫలితం GitHub ఇష్యూలో భాగస్వామ్యం చేయబడుతుంది మరియు ఆ తర్వాత బృందం ద్వారా మూసివేయబడుతుంది.

### <Emoji text=":two:" size={1} /> &nbsp;సర్వే పరిశోధనలో పాల్గొనండి {#answer-surveys}

దీని ద్వారా మా వెబ్‌సైట్‌పై అభిప్రాయాన్ని అందించండి:

1. ethereum.org ని సందర్శించడం మరియు పేజీలను చదవడం ద్వారా.
2. కుడివైపు దిగువ మూలలో ఉన్న ఫీడ్‌బ్యాక్ విడ్జెట్‌పై క్లిక్ చేసి, డిజైన్ మరియు కంటెంట్‌కు సంబంధించిన ప్రశ్నలకు సమాధానం ఇవ్వడం ద్వారా.
3. స్వేచ్ఛా ఆకృతి (free format) ప్రశ్నలపై దృష్టి పెట్టండి.

### <Emoji text=":three:" size={1} /> &nbsp;వెబ్‌సైట్‌లో డిజైన్‌కు సంబంధించిన సమస్యలను కనుగొని వాటిని నివేదించండి {#report-design-issues}

ethereum.org అనేది అనేక ఫీచర్లు మరియు కంటెంట్‌తో వేగంగా అభివృద్ధి చెందుతున్న వెబ్‌సైట్. కొన్ని UI సులభంగా పాతబడిపోవచ్చు లేదా మెరుగుపరచబడవచ్చు. మీరు అలాంటి సందర్భాన్ని ఎదుర్కొంటే, దయచేసి దానిని నివేదించండి, తద్వారా అది మా దృష్టికి వస్తుంది.

1. వెబ్‌సైట్‌ను పరిశీలించండి మరియు దాని డిజైన్‌పై శ్రద్ధ వహించండి.
2. మీరు ఏవైనా దృశ్యపరమైన లేదా UX సమస్యలను చూస్తే స్క్రీన్‌షాట్‌లు మరియు నోట్స్ తీసుకోండి.
3. కనుగొన్న సమస్యలను [బగ్ రిపోర్ట్](https://github.com/ethereum/ethereum-org-website/issues/new/choose) ఉపయోగించి నివేదించండి.

### <Emoji text=":four:" size={1} /> &nbsp;డిజైన్ మార్పులను ప్రతిపాదించండి {#propose-design-changes}

మీరు డిజైన్ సవాళ్లను స్వీకరించడానికి సౌకర్యంగా భావిస్తే, మీరు మా GitHub ఇష్యూల బోర్డును సందర్శించవచ్చు మరియు [డిజైన్‌కు సంబంధించిన ఇష్యూల](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) కోసం ఫిల్టర్ చేయవచ్చు.

1. మా వెబ్‌సైట్‌ను పరిశీలించి, దాని డిజైన్‌పై శ్రద్ధ వహించండి లేదా మా GitHub రిపోజిటరీకి వెళ్లి [“Design required” ట్యాగ్‌తో](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) ఫ్లాగ్ చేయబడిన ఇష్యూలను సమీక్షించండి.
2. పరిష్కారంపై ఆలోచించండి మరియు దానిని డిజైన్ చేయండి. (ఆదర్శంగా మా [డిజైన్ సిస్టమ్‌ను](https://www.figma.com/community/file/1134414495420383395) ఉపయోగించి).
3. సంబంధిత GitHub ఇష్యూలో పరిష్కారాన్ని ప్రతిపాదించండి లేదా [కొత్తదాన్ని సృష్టించండి.](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. డిజైన్ బృందం సమీక్షించే వరకు వేచి ఉండండి.

### <Emoji text=":five:" size={1} /> &nbsp;కలిసి డిజైన్ సిస్టమ్‌ను నిర్మిద్దాం {#contribute-to-design-system}

మా డిజైన్ సిస్టమ్ ethereum.org ని డిజైన్ చేయడాన్ని సరదాగా మరియు సులభంగా చేస్తుంది. మీరు అనుభవజ్ఞులైన డిజైనర్ అయితే, వెబ్‌సైట్ కోసం అనేక భాగాలను సిద్ధం చేయడంలో మీరు మాకు సహాయపడగలరు.

1. GitHub లోని [డిజైన్ సిస్టమ్ బోర్డ్](https://github.com/ethereum/ethereum-org-website/labels/design%20system) నుండి పని చేయడానికి ఒక ఇష్యూను ఎంచుకోండి లేదా కొత్తదాన్ని సృష్టించండి.
2. ఎంచుకున్న ఇష్యూను మీకు కేటాయించమని అభ్యర్థించండి.
3. అభ్యర్థించిన భాగాన్ని Figma లో డిజైన్ చేయడం ప్రారంభించండి.
4. మీకు సమీక్ష లేదా మార్గదర్శకత్వం అవసరమైనప్పుడు దానిని GitHub లో డిజైన్ బృందంతో పంచుకోండి.
5. డిజైన్ బృందం సమీక్షిస్తుంది.
6. డిజైన్ బృందం ప్రధాన ఫైల్‌లో మార్పులను పొందుపరుస్తుంది మరియు ఫైల్‌ను కమ్యూనిటీకి ప్రచురిస్తుంది.

### <Emoji text=":six:" size={1} /> &nbsp;వెబ్‌సైట్‌లో డిజైన్‌కు సంబంధించిన కంటెంట్‌ను రాయండి {#write-design-articles}

ఎథీరియం డెవలపర్ కమ్యూనిటీ బలంగా ఉంది, కానీ డిజైన్ కమ్యూనిటీ కొద్దిగా వెనుకబడి ఉంది. మీరు Web3 పరిజ్ఞానం ఉన్న డిజైనర్ అయితే, దయచేసి మీ అభ్యాసాలను విస్తృత కమ్యూనిటీతో పంచుకోవడాన్ని పరిగణించండి, తద్వారా మనమందరం కలిసి ఎదగవచ్చు మరియు మెరుగుపడవచ్చు; మీరు సహకరించగల [ఎథీరియం కోసం డిజైన్ చేయడంపై ఒక పేజీ](/developers/docs/design-and-ux/) మా వద్ద ఉంది. మీరు మా [లిస్టింగ్ విధానాలను](/contributing/design/adding-design-resources) కూడా తనిఖీ చేయవచ్చు.

1. ethereum.org లో కవర్ చేయవలసిన మరియు ఈ రంగంలోని డిజైనర్లకు ప్రయోజనకరంగా ఉండే డిజైన్ అంశాలపై ఆలోచించండి.
2. మా GitHub రిపోజిటరీకి వెళ్లి, ఒక అంశాన్ని ప్రతిపాదిస్తూ [ఒక ఇష్యూను లేవనెత్తండి](https://github.com/ethereum/ethereum-org-website/issues/new) (ఇప్పుడే కంటెంట్‌ను రాయకండి).
3. డిజైన్ బృందం ఆమోదించే వరకు వేచి ఉండండి.
4. ఆమోదించబడిన తర్వాత, కంటెంట్‌ను రాయండి.
5. సంబంధిత GitHub ఇష్యూలో దానిని సమర్పించండి.

### <Emoji text=":seven:" size={1} /> &nbsp;కొత్త ఇలస్ట్రేషన్లను గీయండి {#prepare-illustrations}

నైరూప్య అంశాలను వివరించడానికి విజువలైజేషన్స్ అత్యంత శక్తివంతమైన సాధనాల్లో ఒకటి. రేఖాచిత్రాలు మరియు ఇన్ఫోగ్రాఫిక్స్‌ను జోడించడం ద్వారా అపారమైన సామర్థ్యం ఉంది. అన్నింటికంటే, ఒక చిత్రం వెయ్యి పదాలను చెప్పగలదు.

1. మా వెబ్‌సైట్‌కి వెళ్లి, కొన్ని కొత్త ఇన్ఫోగ్రాఫిక్స్‌ను జోడించగల పేజీలను చూడండి.
2. ఇలస్ట్రేషన్ శైలి మా [ఆస్తులకు](/assets/) అనుగుణంగా ఉండేలా చూసుకోండి.
3. మా GitHub రిపోజిటరీకి వెళ్లి, ఇలస్ట్రేషన్‌ను ప్రతిపాదిస్తూ [ఒక ఇష్యూను లేవనెత్తండి](https://github.com/ethereum/ethereum-org-website/issues/new).
4. డిజైన్ బృందం దానిని సమీక్షిస్తుంది.
5. కొత్త చిత్రాన్ని అమలు చేయమని డెవలపర్‌ను అడగడానికి మేము కొత్త ఇష్యూను సృష్టిస్తాము.
