## تعارف {#Introduction}

اپنا [Ethereum نوڈ](/developers/docs/nodes-and-clients/#what-are-nodes-and-clients) چلانا مشکل ہو سکتا ہے، خاص طور پر جب آپ شروعات کر رہے ہوں یا تیزی سے اسکیل کر رہے ہوں۔ ایسی [کئی سروسز](#popular-node-services) ہیں جو آپ کے لیے آپٹمائزڈ نوڈ انفراسٹرکچر چلاتی ہیں، تاکہ آپ اس کے بجائے اپنی ایپلیکیشن یا پروڈکٹ تیار کرنے پر توجہ مرکوز کر سکیں۔ ہم وضاحت کریں گے کہ نوڈ سروسز کیسے کام کرتی ہیں، انہیں استعمال کرنے کے فوائد اور نقصانات کیا ہیں اور اگر آپ شروعات کرنے میں دلچسپی رکھتے ہیں تو فراہم کنندگان کی فہرست بھی دیں گے۔

## پیشگی شرائط {#prerequisites}

اگر آپ کو پہلے سے یہ سمجھ نہیں ہے کہ نوڈز اور کلائنٹس کیا ہیں، تو [نوڈز اور کلائنٹس](/developers/docs/nodes-and-clients/) کو دیکھیں۔

## اسٹیکرز {#stakoooooooooooooors}

سولو اسٹیکرز کو فریق ثالث فراہم کنندگان پر انحصار کرنے کے بجائے اپنا انفراسٹرکچر خود چلانا چاہیے۔ اس کا مطلب ہے کہ ایک ایگزیکیوشن کلائنٹ کو کنسینسس کلائنٹ کے ساتھ ملا کر چلانا۔ [The Merge](/roadmap/merge) سے پہلے، صرف ایک کنسینسس کلائنٹ چلانا اور ایگزیکیوشن ڈیٹا کے لیے ایک سینٹرلائزڈ فراہم کنندہ کا استعمال کرنا ممکن تھا؛ اب یہ ممکن نہیں رہا - ایک سولو اسٹیکر کو دونوں کلائنٹس چلانے ہوں گے۔ تاہم، اس عمل کو آسان بنانے کے لیے سروسز دستیاب ہیں۔

[نوڈ چلانے کے بارے میں مزید پڑھیں](/developers/docs/nodes-and-clients/run-a-node/)۔

اس صفحہ پر بیان کردہ سروسز نان-اسٹیکنگ نوڈز کے لیے ہیں۔

## نوڈ سروسز کیسے کام کرتی ہیں؟ {#how-do-node-services-work}

نوڈ سروس فراہم کنندگان آپ کے لیے پردے کے پیچھے ڈسٹریبیوٹڈ نوڈ کلائنٹس چلاتے ہیں، تاکہ آپ کو ایسا نہ کرنا پڑے۔

یہ سروسز عام طور پر ایک API کلید فراہم کرتی ہیں جسے آپ بلاک چین پر لکھنے اور پڑھنے کے لیے استعمال کر سکتے ہیں۔ ان میں اکثر مین نیٹ کے علاوہ [Ethereum ٹیسٹ نیٹس](/developers/docs/networks/#ethereum-testnets) تک رسائی بھی شامل ہوتی ہے۔

کچھ سروسز آپ کو آپ کا اپنا ڈیڈیکیٹڈ نوڈ پیش کرتی ہیں جس کا وہ آپ کے لیے انتظام کرتی ہیں، جبکہ دیگر نوڈز میں سرگرمی کو تقسیم کرنے کے لیے لوڈ بیلنسرز کا استعمال کرتی ہیں۔

تقریباً تمام نوڈ سروسز کے ساتھ انٹیگریٹ کرنا انتہائی آسان ہے، جس میں آپ کے سیلف ہوسٹڈ نوڈ کو تبدیل کرنے کے لیے آپ کے کوڈ میں ایک لائن کی تبدیلیاں شامل ہیں، یا یہاں تک کہ خود سروسز کے درمیان سوئچ کرنا بھی شامل ہے۔

اکثر اوقات نوڈ سروسز مختلف قسم کے [نوڈ کلائنٹس](/developers/docs/nodes-and-clients/#execution-clients) اور [اقسام](/developers/docs/nodes-and-clients/#node-types) چلاتی ہیں، جس سے آپ کو ایک API میں کلائنٹ کے مخصوص طریقوں کے علاوہ فل اور آرکائیو نوڈز تک رسائی حاصل ہوتی ہے۔

یہ نوٹ کرنا ضروری ہے کہ نوڈ سروسز آپ کی نجی کلیدیں یا معلومات محفوظ نہیں کرتیں اور نہ ہی انہیں کرنی چاہئیں۔

## نوڈ سروس استعمال کرنے کے کیا فوائد ہیں؟ {#benefits-of-using-a-node-service}

نوڈ سروس استعمال کرنے کا بنیادی فائدہ یہ ہے کہ آپ کو خود نوڈز کو برقرار رکھنے اور ان کا انتظام کرنے میں انجینئرنگ کا وقت صرف نہیں کرنا پڑتا۔ یہ آپ کو انفراسٹرکچر کی دیکھ بھال کے بارے میں فکر کرنے کے بجائے اپنی پروڈکٹ بنانے پر توجہ مرکوز کرنے کی اجازت دیتا ہے۔

اپنے نوڈز چلانا اسٹوریج سے لے کر بینڈوتھ اور قیمتی انجینئرنگ کے وقت تک بہت مہنگا ہو سکتا ہے۔ اسکیلنگ کے وقت مزید نوڈز کو اسپن اپ کرنا، نوڈز کو تازہ ترین ورژنز میں اپ گریڈ کرنا، اور اسٹیٹ کی مستقل مزاجی کو یقینی بنانا جیسی چیزیں آپ کی مطلوبہ web3 پروڈکٹ کی تعمیر اور اس پر وسائل خرچ کرنے سے توجہ ہٹا سکتی ہیں۔

## نوڈ سروس استعمال کرنے کے کیا نقصانات ہیں؟ {#cons-of-using-a-node-service}

نوڈ سروس کا استعمال کر کے آپ اپنی پروڈکٹ کے انفراسٹرکچر کے پہلو کو سینٹرلائز کر رہے ہیں۔ اس وجہ سے، وہ پروجیکٹس جو ڈی سینٹرلائزیشن کو انتہائی اہمیت دیتے ہیں، وہ کسی فریق ثالث کو آؤٹ سورس کرنے کے بجائے سیلف ہوسٹنگ نوڈز کو ترجیح دے سکتے ہیں۔

[اپنا نوڈ چلانے کے فوائد](/developers/docs/nodes-and-clients/#benefits-to-you) کے بارے میں مزید پڑھیں۔
