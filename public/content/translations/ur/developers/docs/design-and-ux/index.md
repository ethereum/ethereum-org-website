کیا آپ ایتھیریم کے ساتھ ڈیزائننگ میں نئے ہیں؟ یہ آپ کے لیے بالکل صحیح جگہ ہے۔ ایتھیریم کمیونٹی نے آپ کو ویب 3 ڈیزائن اور ریسرچ کی بنیادی باتوں سے متعارف کرانے کے لیے وسائل لکھے ہیں۔ آپ ان بنیادی تصورات کے بارے میں جانیں گے جو آپ کے مانوس دیگر ایپ ڈیزائنز سے مختلف ہو سکتے ہیں۔

کیا پہلے ویب 3 کی مزید بنیادی سمجھ کی ضرورت ہے؟ [**لرن ہب**](/learn/) دیکھیں۔

## صارف کی ریسرچ سے شروعات کریں {#start-with-user-research}

مؤثر ڈیزائن صرف بصری طور پر پرکشش یوزر انٹرفیس بنانے سے کہیں آگے ہے۔ اس میں صارف کی ضروریات، مقاصد اور محرک عوامل کی گہری سمجھ حاصل کرنا شامل ہے۔ لہذا، ہم پرزور مشورہ دیتے ہیں کہ تمام ڈیزائنرز ایک ڈیزائن کا عمل اپنائیں، جیسے کہ [**ڈبل ڈائمنڈ پروسیس**](<https://en.wikipedia.org/wiki/Double_Diamond_(design_process_model)>)، تاکہ یہ یقینی بنایا جا سکے کہ ان کا کام سوچا سمجھا اور بامقصد ہے۔

- [ویب 3 کو مزید UX ریسرچرز اور ڈیزائنرز کی ضرورت ہے](https://blog.akasha.org/akasha-conversations-9-web3-needs-more-ux-researchers-and-designers) - موجودہ ڈیزائن کی پختگی کا جائزہ
- [ویب 3 میں UX ریسرچ کے لیے ایک سادہ گائیڈ](https://uxplanet.org/a-complete-guide-to-ux-research-for-web-3-0-products-d6bead20ebb1) - ریسرچ کرنے کے طریقے پر سادہ گائیڈ
- [ویب 3 میں UX فیصلوں تک کیسے پہنچیں](https://archive.devcon.org/archive/watch/6/data-empathy-how-to-approach-ux-decisions-in-web3/) - مقداری اور کوالٹیٹو ریسرچ اور دونوں کے درمیان فرق کا مختصر جائزہ (ویڈیو، 6 منٹ)
- [ویب 3 میں ایک ux ریسرچر ہونا](https://medium.com/@georgia.rakusen/what-its-like-being-a-user-researcher-in-web3-6a4bcc096849) - ویب 3 میں UX ریسرچر ہونے کے بارے میں ایک ذاتی نقطہ نظر
