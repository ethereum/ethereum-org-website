# ethereum.org میں ڈیزائن کی شراکت {#design-contributions}

ڈیزائن کسی بھی پروجیکٹ کا ایک اہم جزو ہے، اور اپنا وقت اور ڈیزائن کی مہارتیں ethereum.org کے لیے وقف کر کے، آپ ہمارے زائرین کے لیے صارف کے تجربے کو بہتر بنانے میں مدد کر سکتے ہیں۔ کسی اوپن سورس پروجیکٹ میں حصہ ڈالنا متعلقہ تجربہ حاصل کرنے اور باہمی تعاون کے ماحول میں اپنی صلاحیتوں کو فروغ دینے کا موقع فراہم کرتا ہے۔ آپ کو دوسرے ڈیزائنرز، ڈیولپرز، اور کمیونٹی ممبرز کے ساتھ کام کرنے کا موقع ملے گا، جن میں سے سبھی کے اپنے منفرد نقطہ نظر اور بصیرت ہوں گے۔

بالآخر، یہ ایک متنوع اور متاثر کن پورٹ فولیو بنانے کا ایک بہترین طریقہ ہے جو آپ کی ڈیزائن کی مہارتوں کو ظاہر کرتا ہے۔

## حصہ کیسے ڈالیں؟

### <Emoji text=":one:" size={1} /> &nbsp;ابتدائی ڈیزائن پروٹوٹائپس پر رائے دیں {#design-critique}

ہمیں بعض اوقات اپنے خام خیالات کو جانچنے میں مدد کی ضرورت ہوتی ہے۔ بغیر کسی تکنیکی علم کے حصہ ڈالنے کا یہ ایک بہترین طریقہ ہے۔

1. ڈیزائن ٹیم [Discord](https://discord.com/invite/ethereum-org) اور [GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) پر ایک موک اپ ڈیزائن شیئر کرے گی۔
2. تبصرے کے فنکشن کے ذریعے رائے دینے کے لیے آپ کو ڈیزائنز کے ذریعے رہنمائی فراہم کی جائے گی۔
3. نتیجہ GitHub ایشو میں شیئر کیا جائے گا اور پھر ٹیم کی طرف سے اسے بند کر دیا جائے گا۔

### <Emoji text=":two:" size={1} /> &nbsp;سروے ریسرچ میں حصہ لیں {#answer-surveys}

ہماری ویب سائٹ پر اس طرح رائے دیں:

1. ethereum.org پر جا کر اور صفحات کو پڑھ کر۔
2. نیچے دائیں کونے میں موجود فیڈ بیک وجیٹ پر کلک کر کے اور ڈیزائن اور مواد سے متعلق سوالات کے جوابات دے کر۔
3. فری فارمیٹ سوالات پر توجہ مرکوز کریں۔

### <Emoji text=":three:" size={1} /> &nbsp;ویب سائٹ پر ڈیزائن سے متعلق مسائل تلاش کریں اور انہیں رپورٹ کریں {#report-design-issues}

ethereum.org ایک تیزی سے بڑھتی ہوئی ویب سائٹ ہے جس میں بہت سی خصوصیات اور مواد موجود ہے۔ کچھ UI آسانی سے پرانے ہو سکتے ہیں یا انہیں بہتر بنایا جا سکتا ہے۔ اگر آپ کو ایسی کوئی مثال نظر آئے، تو براہ کرم اسے رپورٹ کریں تاکہ یہ ہماری توجہ میں آ سکے۔

1. ویب سائٹ کو دیکھیں اور اس کے ڈیزائن پر توجہ دیں۔
2. اگر آپ کو کوئی بصری یا UX مسائل نظر آئیں تو اسکرین شاٹس اور نوٹس لیں۔
3. پائے جانے والے مسائل کو [بگ رپورٹ](https://github.com/ethereum/ethereum-org-website/issues/new/choose) کا استعمال کرتے ہوئے رپورٹ کریں۔

### <Emoji text=":four:" size={1} /> &nbsp;ڈیزائن میں تبدیلیوں کی تجویز دیں {#propose-design-changes}

اگر آپ ڈیزائن چیلنجز کا مقابلہ کرنے میں آرام دہ محسوس کرتے ہیں، تو آپ ہمارے GitHub ایشوز بورڈ پر جا سکتے ہیں اور [ڈیزائن سے متعلق مسائل](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) کے لیے فلٹر کر سکتے ہیں۔

1. ہماری ویب سائٹ کو دیکھیں اور اس کے ڈیزائن پر توجہ دیں یا ہماری GitHub ریپوزٹری پر جائیں اور [“Design required” ٹیگ](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) کے ساتھ نشان زدہ مسائل کا جائزہ لیں۔
2. حل پر غور کریں اور اسے ڈیزائن کریں۔ (مثالی طور پر ہمارے [ڈیزائن سسٹم](https://www.figma.com/community/file/1134414495420383395) کا استعمال کرتے ہوئے)۔
3. متعلقہ GitHub ایشو میں حل تجویز کریں یا [ایک نیا بنائیں۔](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. ڈیزائن ٹیم کے جائزے کا انتظار کریں۔

### <Emoji text=":five:" size={1} /> &nbsp;مل کر ڈیزائن سسٹم بنائیں {#Contribute-to-design-system}

ہمارا ڈیزائن سسٹم ethereum.org کی ڈیزائننگ کو پرلطف اور آسان بناتا ہے۔ اگر آپ ایک تجربہ کار ڈیزائنر ہیں، تو آپ ویب سائٹ کے لیے بہت سے اجزاء تیار کرنے میں ہماری مدد کر سکتے ہیں۔

1. GitHub پر [ڈیزائن سسٹم بورڈ](https://github.com/ethereum/ethereum-org-website/labels/design%20system) سے کام کرنے کے لیے ایک ایشو منتخب کریں یا ایک نیا بنائیں۔
2. درخواست کریں کہ منتخب کردہ ایشو آپ کو تفویض کیا جائے۔
3. Figma میں مطلوبہ جزو کو ڈیزائن کرنا شروع کریں۔
4. جب آپ کو جائزے یا رہنمائی کی ضرورت ہو تو اسے GitHub پر ڈیزائن ٹیم کے ساتھ شیئر کریں۔
5. ڈیزائن ٹیم اس کا جائزہ لے گی۔
6. ڈیزائن ٹیم مین فائل میں تبدیلیاں شامل کرے گی اور فائل کو کمیونٹی کے لیے شائع کرے گی۔

### <Emoji text=":six:" size={1} /> &nbsp;ویب سائٹ پر ڈیزائن سے متعلق مواد لکھیں {#write-design-articles}

Ethereum ڈیولپر کمیونٹی مضبوط ہے، لیکن ڈیزائن کمیونٹی قدرے پیچھے رہ رہی ہے۔ اگر آپ web3 کے علم کے حامل ڈیزائنر ہیں، تو براہ کرم اپنی سیکھی ہوئی باتوں کو وسیع تر کمیونٹی کے ساتھ شیئر کرنے پر غور کریں تاکہ ہم سب مل کر ترقی کر سکیں اور بہتری لا سکیں؛ ہمارے پاس [Ethereum کے لیے ڈیزائننگ پر ایک صفحہ](/developers/docs/design-and-ux/) ہے جس میں آپ حصہ ڈال سکتے ہیں۔ آپ ہماری [لسٹنگ پالیسیاں](/contributing/design/adding-design-resources) بھی چیک کر سکتے ہیں۔

1. ڈیزائن کے ان موضوعات پر غور کریں جن کا احاطہ ethereum.org پر کیا جانا چاہیے اور جو اس شعبے میں ڈیزائنرز کے لیے فائدہ مند ہوں گے۔
2. ہماری GitHub ریپوزٹری پر جائیں اور ایک موضوع تجویز کرتے ہوئے [ایک ایشو اٹھائیں](https://github.com/ethereum/ethereum-org-website/issues/new) (ابھی مواد نہ لکھیں)۔
3. ڈیزائن ٹیم کی منظوری کا انتظار کریں۔
4. منظور ہونے کے بعد، مواد لکھیں۔
5. اسے متعلقہ GitHub ایشو میں جمع کرائیں۔

### <Emoji text=":seven:" size={1} /> &nbsp;نئی عکاسی (illustrations) بنائیں {#prepare-illustrations}

تجریدی موضوعات کی وضاحت کے لیے بصریات (Visualisations) سب سے طاقتور ٹولز میں سے ایک ہیں۔ خاکے اور انفوگرافکس شامل کرنے سے بے پناہ امکانات پیدا ہوتے ہیں۔ بہر حال، ایک تصویر ہزار الفاظ کے برابر ہوتی ہے۔

1. ہماری ویب سائٹ پر جائیں اور وہ صفحات دیکھیں جہاں کچھ نئے انفوگرافکس شامل کیے جا سکتے ہیں۔
2. اس بات کو یقینی بنائیں کہ عکاسی کا انداز ہمارے [اثاثوں (assets)](/assets/) سے مطابقت رکھتا ہو۔
3. ہماری GitHub ریپوزٹری پر جائیں اور عکاسی کی تجویز دیتے ہوئے [ایک ایشو اٹھائیں](https://github.com/ethereum/ethereum-org-website/issues/new)۔
4. ڈیزائن ٹیم اس کا جائزہ لے گی۔
5. ہم ایک ڈیولپر سے نئی تصویر کو نافذ کرنے کا کہنے کے لیے ایک نیا ایشو بناتے ہیں۔
