کسی بھی پروجیکٹ میں ڈیزائن ایک اہم جزو ہوتا ہے، اور اپنا وقت اور ڈیزائن کی مہارتیں <span dir="ltr">ethereum.org</span> کے لیے وقف کر کے، آپ ہمارے زائرین کے لیے صارف کے تجربے (user experience) کو بہتر بنانے میں مدد کر سکتے ہیں۔ کسی اوپن سورس پروجیکٹ میں حصہ لینا متعلقہ تجربہ حاصل کرنے اور باہمی تعاون کے ماحول میں اپنی مہارتوں کو نکھارنے کا موقع فراہم کرتا ہے۔ آپ کو دیگر ڈیزائنرز، ڈیولپرز، اور کمیونٹی ممبران کے ساتھ کام کرنے کا موقع ملے گا، جن میں سے ہر ایک کے اپنے منفرد نقطہ نظر اور بصیرتیں ہوں گی۔

مجموعی طور پر، یہ ایک متنوع اور متاثر کن پورٹ فولیو بنانے کا ایک بہترین طریقہ ہے جو آپ کی ڈیزائن کی مہارتوں کو نمایاں کرتا ہے۔

## آپ کیسے حصہ لے سکتے ہیں؟ {#how-to-contribute}

### <Emoji text=":one:" size={1} /> &nbsp;ابتدائی ڈیزائن پروٹوٹائپس پر رائے دیں {#design-critique}

ہمیں بعض اوقات اپنے ابتدائی خیالات کو جانچنے کے لیے مدد کی ضرورت ہوتی ہے۔ بغیر کسی تکنیکی علم کے حصہ لینے کا یہ ایک بہترین طریقہ ہے۔

1. ڈیزائن ٹیم [ڈسکارڈ](https://discord.com/invite/ethereum-org) اور [GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) پر ایک موک اپ (mockup) ڈیزائن شیئر کرے گی۔
2. کمنٹس کے فنکشن کے ذریعے رائے دینے کے لیے آپ کو ڈیزائنز کے حوالے سے رہنمائی فراہم کی جائے گی۔
3. نتائج کو GitHub ایشو میں شیئر کیا جائے گا اور پھر ٹیم کی جانب سے اسے بند کر دیا جائے گا۔

### <Emoji text=":two:" size={1} /> &nbsp;سروے ریسرچ میں حصہ لیں {#answer-surveys}

ہماری ویب سائٹ پر درج ذیل طریقوں سے اپنی رائے دیں:

1. <span dir="ltr">ethereum.org</span> کا دورہ کر کے اور صفحات کا مطالعہ کر کے۔
2. نیچے دائیں کونے میں موجود فیڈ بیک وجٹ پر کلک کر کے اور ڈیزائن اور مواد سے متعلق سوالات کے جوابات دے کر۔
3. فری فارمیٹ (free format) سوالات پر توجہ مرکوز کریں۔

### <Emoji text=":three:" size={1} /> &nbsp;ویب سائٹ پر ڈیزائن سے متعلق مسائل تلاش کریں اور انہیں رپورٹ کریں {#report-design-issues}

<span dir="ltr">ethereum.org</span> ایک تیزی سے ترقی کرنے والی ویب سائٹ ہے جس میں بہت سی خصوصیات اور مواد موجود ہے۔ کچھ UI (یوزر انٹرفیس) آسانی سے پرانے ہو سکتے ہیں یا انہیں بہتر بنایا جا سکتا ہے۔ اگر آپ کو کوئی ایسا مسئلہ نظر آئے، تو براہ کرم اسے رپورٹ کریں تاکہ ہماری توجہ اس جانب مبذول ہو سکے۔

1. ویب سائٹ کا جائزہ لیں اور اس کے ڈیزائن پر توجہ دیں۔
2. اگر آپ کو کوئی بصری (visual) یا UX (یوزر ایکسپیرینس) کا مسئلہ نظر آئے تو اسکرین شاٹس لیں اور نوٹس بنائیں۔
3. [بگ رپورٹ](https://github.com/ethereum/ethereum-org-website/issues/new/choose) کا استعمال کرتے ہوئے دریافت ہونے والے مسائل کو رپورٹ کریں۔

### <Emoji text=":four:" size={1} /> &nbsp;ڈیزائن میں تبدیلیوں کی تجویز دیں {#propose-design-changes}

اگر آپ ڈیزائن کے چیلنجز کا سامنا کرنے میں دلچسپی رکھتے ہیں، تو آپ ہمارے GitHub ایشوز بورڈ پر جا سکتے ہیں اور [ڈیزائن سے متعلق ایشوز](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) کو فلٹر کر سکتے ہیں۔

1. ہماری ویب سائٹ کا جائزہ لیں اور اس کے ڈیزائن پر توجہ دیں یا ہماری GitHub ریپوزٹری پر جائیں اور [“Design required” ٹیگ](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8) والے ایشوز کا جائزہ لیں۔
2. حل کے بارے میں سوچیں اور اسے ڈیزائن کریں۔ (مثالی طور پر ہمارے [ڈیزائن سسٹم](https://www.figma.com/community/file/1134414495420383395) کا استعمال کرتے ہوئے)۔
3. متعلقہ GitHub ایشو میں حل تجویز کریں یا [ایک نیا ایشو بنائیں۔](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. ڈیزائن ٹیم کے جائزے کا انتظار کریں۔

### <Emoji text=":five:" size={1} /> &nbsp;مل کر ڈیزائن سسٹم بنائیں {#contribute-to-design-system}

ہمارا ڈیزائن سسٹم <span dir="ltr">ethereum.org</span> کو ڈیزائن کرنا دلچسپ اور آسان بناتا ہے۔ اگر آپ ایک تجربہ کار ڈیزائنر ہیں، تو آپ ویب سائٹ کے لیے بہت سے اجزاء (components) تیار کرنے میں ہماری مدد کر سکتے ہیں۔

1. GitHub پر [ڈیزائن سسٹم بورڈ](https://github.com/ethereum/ethereum-org-website/labels/design%20system) سے کام کرنے کے لیے ایک ایشو منتخب کریں یا ایک نیا ایشو بنائیں۔
2. درخواست کریں کہ منتخب کردہ ایشو آپ کو تفویض (assign) کیا جائے۔
3. مطلوبہ جزو کو Figma میں ڈیزائن کرنا شروع کریں۔
4. جب آپ کو جائزے یا رہنمائی کی ضرورت ہو تو اسے GitHub پر ڈیزائن ٹیم کے ساتھ شیئر کریں۔
5. ڈیزائن ٹیم اس کا جائزہ لے گی۔
6. ڈیزائن ٹیم مین فائل میں تبدیلیاں شامل کرے گی اور فائل کو کمیونٹی کے لیے شائع کرے گی۔

### <Emoji text=":six:" size={1} /> &nbsp;ویب سائٹ پر ڈیزائن سے متعلق مواد لکھیں {#write-design-articles}

ایتھیریم ڈیولپر کمیونٹی مضبوط ہے، لیکن ڈیزائن کمیونٹی قدرے پیچھے رہ رہی ہے۔ اگر آپ Web3 کا علم رکھنے والے ڈیزائنر ہیں، تو براہ کرم اپنی سیکھی ہوئی باتوں کو وسیع تر کمیونٹی کے ساتھ شیئر کرنے پر غور کریں تاکہ ہم سب مل کر ترقی کر سکیں اور بہتری لا سکیں؛ ہمارے پاس [ایتھیریم کے لیے ڈیزائننگ پر ایک صفحہ](/developers/docs/design-and-ux/) موجود ہے جس میں آپ حصہ لے سکتے ہیں۔ آپ ہماری [لسٹنگ پالیسیز](/contributing/design/adding-design-resources) بھی دیکھ سکتے ہیں۔

1. ڈیزائن کے ان موضوعات پر غور کریں جن کا احاطہ <span dir="ltr">ethereum.org</span> پر کیا جانا چاہیے اور جو اس شعبے کے ڈیزائنرز کے لیے فائدہ مند ہوں۔
2. ہماری GitHub ریپوزٹری پر جائیں اور ایک موضوع تجویز کرتے ہوئے [ایک ایشو بنائیں](https://github.com/ethereum/ethereum-org-website/issues/new) (ابھی مواد نہ لکھیں)۔
3. ڈیزائن ٹیم کی منظوری کا انتظار کریں۔
4. منظوری ملنے کے بعد، مواد لکھیں۔
5. اسے متعلقہ GitHub ایشو میں جمع کرائیں۔

### <Emoji text=":seven:" size={1} /> &nbsp;نئی عکاسی (illustrations) بنائیں {#prepare-illustrations}

تجریدی (abstract) موضوعات کی وضاحت کے لیے بصری نمائش (visualisations) سب سے طاقتور ٹولز میں سے ایک ہے۔ خاکے (diagrams) اور انفوگرافکس شامل کرنے سے بے پناہ امکانات پیدا ہوتے ہیں۔ آخر کار، ایک تصویر ہزار الفاظ کے برابر ہوتی ہے۔

1. ہماری ویب سائٹ پر جائیں اور وہ صفحات دیکھیں جہاں کچھ نئے انفوگرافکس شامل کیے جا سکتے ہیں۔
2. اس بات کو یقینی بنائیں کہ عکاسی کا انداز ہمارے [اثاثوں (assets)](/assets/) سے مطابقت رکھتا ہو۔
3. ہماری GitHub ریپوزٹری پر جائیں اور عکاسی تجویز کرتے ہوئے [ایک ایشو بنائیں](https://github.com/ethereum/ethereum-org-website/issues/new)۔
4. ڈیزائن ٹیم اس کا جائزہ لے گی۔
5. ہم ایک نیا ایشو بناتے ہیں تاکہ کسی ڈیولپر سے نئی تصویر کو لاگو کرنے کا کہا جا سکے۔
