*مختصر تحریر:* پیکٹرا ہارڈ فورک ایتھیریم توثیق کاروں کو **<span dir="ltr">Type 1</span>** سے **<span dir="ltr">Type 2</span>** انخلا کی اسناد میں تبدیل کر کے زیادہ سے زیادہ مؤثر بیلنس اور کمپاؤنڈنگ کا انتخاب کرنے کی اجازت دیتا ہے۔ ایسا کرنے کا سرکاری ٹول <span dir="ltr">Launchpad</span> ہے۔ اس عمل کو واپس نہیں لیا جا سکتا۔

## جائزہ {#overview}

### کون متاثر ہوتا ہے؟ {#who-is-affected}

کوئی بھی جو توثیق کار چلاتا ہے - یہ ممکنہ طور پر کوئی ایسا شخص ہے جو اپنے زیر کنٹرول توثیق کار کا اشاریہ (جیسے، [توثیق کار <span dir="ltr">#12345</span>](https://beaconcha.in/validator/12345)) جانتا ہے۔ اگر آپ توثیق کار چلانے کے لیے کوئی پروٹوکول استعمال کرتے ہیں (جیسے، لیڈو <span dir="ltr">CSM</span> یا <span dir="ltr">Rocket Pool</span>)، تو آپ کو ان سے یہ معلوم کرنا ہوگا کہ آیا اور کب وہ <span dir="ltr">maxEB</span> کو سپورٹ کرتے ہیں۔

اگر آپ سیال اسٹیکنگ ٹوکن (lst) (جیسے، <span dir="ltr">rETH</span> یا <span dir="ltr">stETH</span>) کا استعمال کرتے ہوئے اسٹیک کرتے ہیں، تو کسی کارروائی کی ضرورت یا سفارش نہیں کی جاتی ہے۔

### "<span dir="ltr">maxEB</span>" کیا ہے؟ {#what-is-maxeb}

<span dir="ltr">maxEB</span> = توثیق کار کا زیادہ سے زیادہ مؤثر بیلنس۔ پیکٹرا ہارڈ فورک تک، ہر توثیق کار زیادہ سے زیادہ <span dir="ltr">32 ETH</span> پر کماتا ہے۔ پیکٹرا کے بعد، توثیق کاروں کے پاس اس تبدیلی کا انتخاب کر کے <span dir="ltr">1 ETH</span> کے اضافے کے ساتھ <span dir="ltr">32</span> اور <span dir="ltr">2048 ETH</span> کے درمیان کسی بھی بیلنس پر کمانے کا اختیار ہے۔

### توثیق کار کیسے انتخاب کرتا ہے؟ {#how-does-a-validator-opt-in}

ایک توثیق کار **<span dir="ltr">Type 1</span>** سے **<span dir="ltr">Type 2</span>** انخلا کی اسناد میں تبدیل کر کے <span dir="ltr">maxEB</span> تبدیلی کا انتخاب کرتا ہے۔ یہ پیکٹرا ہارڈ فورک کے لائیو ہونے کے بعد [<span dir="ltr">Launchpad</span> (توثیق کار کے اعمال)](https://launchpad.ethereum.org/validator-actions) پر کیا جا سکتا ہے۔ جیسا کہ **<span dir="ltr">Type 0</span>** → **<span dir="ltr">Type 1</span>** کے ساتھ ہے، **<span dir="ltr">Type 1</span>** → **<span dir="ltr">Type 2</span>** میں تبدیل کرنا ایک ناقابل واپسی عمل ہے۔

### انخلا کی اسناد کیا ہیں؟ {#whats-a-withdrawal-credential}

جب آپ توثیق کار چلاتے ہیں، تو آپ کے پاس انخلا کی اسناد کا ایک سیٹ ہوتا ہے۔ یہ آپ کے ڈپازٹ ڈیٹا json میں مل سکتے ہیں یا آپ انہیں اپنے توثیق کار کے <span dir="ltr">beaconcha.in</span> [ڈپازٹ ٹیب](https://beaconcha.in/validator/12345#deposits) پر دیکھ سکتے ہیں۔

1. **<span dir="ltr">Type 0</span>** انخلا کی اسناد: اگر آپ کے توثیق کار کی انخلا کی اسناد `0x00...` سے شروع ہوتی ہیں، تو آپ نے شپیلا ہارڈ فورک سے پہلے ڈپازٹ کیا تھا اور ابھی تک انخلا کا پتہ سیٹ نہیں کیا ہے۔

![Type 0 withdrawal credential](./0x00-wd.png)

2. **<span dir="ltr">Type 1</span>** انخلا کی اسناد: اگر آپ کے توثیق کار کی انخلا کی اسناد `0x01...` سے شروع ہوتی ہیں، تو آپ نے شپیلا ہارڈ فورک کے بعد ڈپازٹ کیا تھا یا پہلے ہی اپنی **<span dir="ltr">Type 0</span>** اسناد کو **<span dir="ltr">Type 1</span>** اسناد میں تبدیل کر لیا ہے۔

 ![Type 1 withdrawal credential](./0x01-wd.png)

3. **<span dir="ltr">Type 2</span>** انخلا کی اسناد: انخلا کی اسناد کی یہ نئی قسم `0x02...` سے شروع ہوگی اور پیکٹرا کے بعد فعال ہو جائے گی۔ **<span dir="ltr">Type 2</span>** انخلا کی اسناد والے توثیق کاروں کو بعض اوقات "**کمپاؤنڈنگ توثیق کار**" کہا جاتا ہے۔

| **اجازت ہے** | **اجازت نہیں ہے** |
| --- | --- |
| ✅ <span dir="ltr">Type 0</span> → <span dir="ltr">Type 1</span> | ❌ <span dir="ltr">Type 0</span> → <span dir="ltr">Type 2</span> |
| ✅ <span dir="ltr">Type 1</span> → <span dir="ltr">Type 2</span> | ❌ <span dir="ltr">Type 1</span> → <span dir="ltr">Type 0</span> |
|  | ❌ <span dir="ltr">Type 2</span> → <span dir="ltr">Type 1</span> |
|  | ❌ <span dir="ltr">Type 2</span> → <span dir="ltr">Type 0</span> |

### خطرات {#risks}

<span dir="ltr">MaxEB</span> ایک توثیق کار کو اپنا پورا بیلنس دوسرے توثیق کار کو بھیجنے کے قابل بناتا ہے۔ استحکام (consolidation) کی درخواست جمع کرانے والے صارفین کو اس ٹرانزیکشن کے ماخذ اور مندرجات کی تصدیق کرنی چاہیے جس پر وہ دستخط کر رہے ہیں۔ <span dir="ltr">maxEB</span> کی خصوصیات سے فائدہ اٹھانے کا سرکاری ٹول <span dir="ltr">Launchpad</span> ہے۔ اگر آپ کسی فریق ثالث کے ٹول کو استعمال کرنے کا فیصلہ کرتے ہیں، تو آپ کو تصدیق کرنی چاہیے کہ:

- ماخذ توثیق کار کی عوامی کلید (pubkey) اور انخلا کا پتہ اس توثیق کار سے میل کھاتا ہے جسے وہ کنٹرول کرتے ہیں
- ہدف توثیق کار کی عوامی کلید درست ہے اور ان کی ہے
- درخواست ایک تبدیلی ہے، استحکام نہیں، اگر وہ کسی دوسرے توثیق کار کو فنڈز بھیجنے کا ارادہ نہیں رکھتے ہیں
- ٹرانزیکشن پر درست انخلا کے پتے کے ذریعے دستخط کیے جا رہے ہیں

ہم **سختی سے تجویز کرتے ہیں** کہ آپ جس بھی فریق ثالث ٹول کو استعمال کرنے کا ارادہ رکھتے ہیں اس پر [EthStaker کمیونٹی](https://ethstaker.org/about) کے ساتھ تبادلہ خیال کریں۔ یہ آپ کے طریقہ کار کی جانچ پڑتال کرنے اور غلطیوں سے بچنے کے لیے ایک مددگار جگہ ہے۔ اگر آپ کوئی بدنیتی پر مبنی یا غلط کنفیگر شدہ ٹول استعمال کرتے ہیں، تو **آپ کا پورا توثیق کار بیلنس کسی ایسے توثیق کار کو بھیجا جا سکتا ہے جسے آپ کنٹرول نہیں کرتے** — اور اسے واپس حاصل کرنے کا کوئی طریقہ نہیں ہوگا۔

## تکنیکی تفصیلات {#technical-details}

### بہاؤ (Flow) {#the-flow}

`ConsolidationRequest` آپریشن کے دو استعمال ہوں گے:

1. موجودہ توثیق کار کو **<span dir="ltr">Type 1</span>** سے **<span dir="ltr">Type 2</span>** توثیق کار میں تبدیل کرنا
2. دیگر توثیق کاروں کو موجودہ **<span dir="ltr">Type 2</span>** توثیق کار میں مستحکم کرنا

**<span dir="ltr">Type 1</span>** سے **<span dir="ltr">Type 2</span>** توثیق کار کی تبدیلی میں، *ماخذ* اور *ہدف* دونوں وہی توثیق کار ہوں گے جسے آپ تبدیل کر رہے ہیں۔ اس آپریشن پر گیس خرچ ہوگی اور اسے دیگر استحکام کی درخواستوں کے پیچھے قطار میں لگایا جائے گا۔ یہ قطار ڈپازٹ کی قطار سے **الگ** ہے اور نئے توثیق کار کے ڈپازٹس سے متاثر نہیں ہوتی ہے اور اسے [<span dir="ltr">pectrified.com</span>](https://pectrified.com/) پر دیکھا جا سکتا ہے۔

توثیق کاروں کو مستحکم کرنے کے لیے، آپ کے پاس ایک *ہدف توثیق کار* ہونا چاہیے جس کے پاس **<span dir="ltr">Type 2</span>** انخلا کی اسناد ہوں۔ یہ مستحکم کیے جانے والے کسی بھی توثیق کار کے بیلنس کی منزل ہے، اور اشاریہ محفوظ رکھا جاتا ہے۔

### <span dir="ltr">Type 2</span> میں تبدیل کرنے کے تقاضے {#requirements-for-converting-to-type-2}

یہ اس پہلے توثیق کار کے لیے درکار ہوگا جسے آپ **<span dir="ltr">Type 2</span>** میں تبدیل کرتے ہیں۔ اس توثیق کار کا اشاریہ محفوظ اور فعال رہتا ہے۔ تبدیلی کے لیے، *ماخذ توثیق کار* == *ہدف توثیق کار*۔

توثیق کار کو لازمی طور پر...

- فعال ہونا چاہیے
- **<span dir="ltr">Type 1</span>** انخلا کی اسناد ہونی چاہئیں
- خروج کی حالت میں (یا کٹوتی کا شکار) نہیں ہونا چاہیے
- زیر التوا دستی طور پر متحرک کردہ انخلا نہیں ہونے چاہئیں (یہ خودکار سویپس پر لاگو نہیں ہوتا)

![conversion illustration](./conversion.png)

### استحکام کے تقاضے {#requirements-for-consolidating}

یہ تبدیلی جیسا *ہی آپریشن* ہے لیکن یہ تب ہوتا ہے جب *ماخذ توثیق کار* *ہدف توثیق کار* سے مختلف ہو۔ ہدف توثیق کار کا اشاریہ محفوظ رہتا ہے اور ماخذ توثیق کار سے بیلنس قبول کرتا ہے۔ ماخذ توثیق کار کا اشاریہ `EXITED` حالت میں ڈال دیا جاتا ہے۔

اس صورت میں، ماخذ توثیق کار کے لیے مندرجہ بالا تمام تقاضے یکساں ہیں، نیز:

- کم از کم <span dir="ltr">~27.3</span> گھنٹے (ایک `SHARD_COMMITTEE_PERIOD`) تک فعال رہا ہو

ہدف توثیق کار کو لازمی طور پر

- **<span dir="ltr">Type 2</span>** انخلا کی اسناد ہونی چاہئیں
- خروج کی حالت میں نہیں ہونا چاہیے۔

![consolidation illustration](./consolidation.png)

### استحکام کی درخواست {#the-consolidation-request}

استحکام کی درخواست پر ماخذ توثیق کار سے وابستہ انخلا کے پتے کے ذریعے دستخط کیے جائیں گے اور اس میں درج ذیل شامل ہوں گے:

1. ماخذ توثیق کار کا پتہ (جیسے، `0x15F4B914A0cCd14333D850ff311d6DafbFbAa32b`)
2. ماخذ توثیق کار کی عوامی کلید (جیسے، `0xa1d1ad0714035353258038e964ae9675dc0252ee22cea896825c01458e1807bfad2f9969338798548d9858a571f7425c`)
3. اس ہدف توثیق کار کی عوامی کلید

تبدیلی میں، 2 اور 3 یکساں ہوں گے۔ یہ آپریشن [<span dir="ltr">Launchpad</span>](https://launchpad.ethereum.org/) پر کیا جا سکتا ہے۔

### دستخط کرنے کے تقاضے {#signing-requirements}

`ConsolidationRequest` جمع کرانے کے لیے، **ماخذ توثیق کار کے انخلا کے پتے** کو درخواست پر دستخط کرنا ہوں گے۔ یہ توثیق کار کے فنڈز پر کنٹرول کو ثابت کرتا ہے۔

### کس چیز پر دستخط کیے جاتے ہیں؟ {#what-is-signed}

`ConsolidationRequest` آبجیکٹ کا ڈومین سے الگ کردہ [سائننگ روٹ](https://github.com/ethereum/consensus-specs/blob/master/specs/phase0/beacon-chain.md#compute_signing_root) استعمال کیا جاتا ہے۔

- **ڈومین:** `DOMAIN_CONSOLIDATION_REQUEST`
- **سائننگ روٹ فیلڈز:**
  - `source_pubkey`: `BLSPubkey`
  - `target_pubkey`: `BLSPubkey`
  - `source_address`: `ExecutionAddress`

نتیجے میں آنے والا **BLS دستخط** درخواست کے ساتھ جمع کرایا جاتا ہے۔

نوٹ: دستخط انخلا کے پتے کے ذریعے کیے جاتے ہیں، توثیق کار کی کلید کے ذریعے نہیں۔

### جزوی انخلا {#partial-withdrawals}

**<span dir="ltr">Type 1</span>** اسناد والے توثیق کاروں کو ان کے اضافی بیلنس (<span dir="ltr">32 ETH</span> سے زیادہ کچھ بھی) کے خودکار، گیس کے بغیر سویپس ان کے انخلا کے پتے پر ملتے ہیں۔ چونکہ **<span dir="ltr">Type 2</span>** ایک توثیق کار کو <span dir="ltr">1 ETH</span> کے اضافے میں بیلنس کو کمپاؤنڈ کرنے کی اجازت دیتا ہے، اس لیے یہ <span dir="ltr">2048 ETH</span> تک پہنچنے تک خود بخود بیلنس کو سویپ نہیں کرے گا۔ **<span dir="ltr">Type 2</span>** توثیق کاروں پر جزوی انخلا کو دستی طور پر متحرک کیا جانا چاہیے اور اس پر گیس خرچ ہوگی۔

## استحکام کے ٹولز {#consolidation-tooling}

استحکام کو منظم کرنے کے لیے کئی ٹولز دستیاب ہیں۔ ایتھیریم فاؤنڈیشن کا بنایا ہوا سرکاری ٹول [<span dir="ltr">Launchpad</span>](https://launchpad.ethereum.org/en/validator-actions) ہے۔ اسٹیکنگ کمیونٹی کے اداروں کے بنائے ہوئے فریق ثالث ٹولز بھی موجود ہیں جو ایسی خصوصیات پیش کر سکتے ہیں جو <span dir="ltr">Launchpad</span> فراہم نہیں کرتا۔ اگرچہ یہاں موجود ٹولز کا ایتھیریم فاؤنڈیشن کے ذریعے آڈٹ یا توثیق نہیں کی گئی ہے، لیکن درج ذیل کمیونٹی کے معروف اراکین کے اوپن سورس ٹولز ہیں۔

## اکثر پوچھے گئے سوالات (FAQ) {#faq}

### کیا انتخاب کرنے سے میری تجویز کی قسمت یا انعامات بدل جاتے ہیں؟ {#change-luck-or-rewards}

نہیں۔ انتخاب کرنے سے آپ کے تجویز کے امکانات کم نہیں ہوتے - آپ کے فرائض اور تجویز کا انتخاب یکساں رہتا ہے۔ مثال کے طور پر، اگر آپ کے پاس دو <span dir="ltr">32 ETH</span> توثیق کار بمقابلہ ایک <span dir="ltr">64 ETH</span> توثیق کار ہے، تو آپ کے پاس بلاک تجویز کرنے اور انعامات حاصل کرنے کے لیے منتخب ہونے کے یکساں کل امکانات ہوں گے۔

### کیا انتخاب کرنے سے میرے کٹوتی کے خطرے میں تبدیلی آتی ہے؟ {#change-slashing-risk}

چھوٹے یا غیر پیشہ ور آپریٹرز کے لیے، مختصر جواب نہیں ہے۔ طویل جواب یہ ہے کہ، تیز الرٹنگ کے ساتھ فی نوڈ کئی توثیق کار چلانے والے پیشہ ور آپریٹرز کے لیے، کم توثیق کاروں میں مستحکم ہونا کٹوتی پر ردعمل ظاہر کرنے اور سلسلہ وار ایونٹس کو روکنے کی ان کی صلاحیت کو کم کر سکتا ہے۔ اس خطرے کو دور کرنے کے لیے تمام توثیق کاروں کے لیے ابتدائی کٹوتی کا *جرمانہ* ڈرامائی طور پر <span dir="ltr">1 ETH</span> (فی <span dir="ltr">32 ETH</span>) سے کم کر کے <span dir="ltr">0.0078125 ETH</span> (فی <span dir="ltr">32 ETH</span>) کر دیا گیا ہے۔

### کیا مجھے تبدیل کرنے کے لیے اپنے توثیق کار سے خروج کرنا ہوگا؟ {#exit-validator}

نہیں۔ آپ خروج کیے بغیر اسی جگہ تبدیل کر سکتے ہیں۔

### تبدیل / مستحکم ہونے میں کتنا وقت لگے گا؟ {#how-long}

کم از کم <span dir="ltr">27.3</span> گھنٹے لیکن استحکام بھی ایک قطار کے تابع ہیں۔ یہ قطار ڈپازٹ اور انخلا کی قطاروں سے آزاد ہے اور ان سے متاثر نہیں ہوتی ہے۔

### کیا میں اپنا توثیق کار اشاریہ رکھ سکتا ہوں؟ {#keep-validator-index}

ہاں۔ اسی جگہ تبدیلی یکساں توثیق کار اشاریہ رکھتی ہے۔ اگر آپ متعدد توثیق کاروں کو مستحکم کرتے ہیں، تو آپ صرف *ہدف توثیق کار* کا اشاریہ رکھ سکیں گے۔

### کیا میں تصدیقات (attestations) سے محروم رہوں گا؟ {#miss-attestations}

کسی دوسرے توثیق کار میں استحکام کے دوران، ماخذ توثیق کار سے خروج کر لیا جاتا ہے اور ہدف توثیق کار پر بیلنس فعال ہونے سے پہلے <span dir="ltr">~27</span> گھنٹے کا انتظار کا دورانیہ ہوتا ہے۔ یہ دورانیہ **کارکردگی کے میٹرکس کو متاثر نہیں کرتا ہے**۔

### کیا مجھ پر جرمانے عائد ہوں گے؟ {#incur-penalties}

نہیں۔ جب تک آپ کا توثیق کار آن لائن ہے، آپ پر جرمانے عائد نہیں ہوں گے۔

### کیا مستحکم کیے جانے والے توثیق کاروں کے انخلا کے پتے یکساں ہونے چاہئیں؟ {#withdrawal-addresses-match}

نہیں۔ لیکن *ماخذ* کو اپنے پتے سے درخواست کی اجازت دینی چاہیے۔

### کیا تبدیل کرنے کے بعد میرے انعامات کمپاؤنڈ ہوں گے؟ {#rewards-compound}

ہاں۔ **<span dir="ltr">Type 2</span>** اسناد کے ساتھ، <span dir="ltr">32 ETH</span> سے زیادہ کے انعامات خود بخود دوبارہ اسٹیک ہو جاتے ہیں — لیکن فوری طور پر نہیں۔ ایک چھوٹے بفر (جسے [*ہسٹریسس (hysteresis)*](https://eth2book.info/capella/part2/incentives/balances/#hysteresis) کہا جاتا ہے) کی وجہ سے، اضافی رقم کے دوبارہ اسٹیک ہونے سے پہلے آپ کے بیلنس کو **مزید تقریباً <span dir="ltr">1.25 ETH</span>** تک پہنچنے کی ضرورت ہے۔ لہذا <span dir="ltr">33.0 ETH</span> پر کمپاؤنڈ ہونے کے بجائے، یہ <span dir="ltr">33.25</span> (مؤثر بیلنس = <span dir="ltr">33 ETH</span>)، پھر <span dir="ltr">34.25</span> (مؤثر بیلنس = <span dir="ltr">34 ETH</span>)، اور اسی طرح آگے ہوتا ہے۔

### کیا میں تبدیل کرنے کے بعد بھی خودکار سویپس حاصل کر سکتا ہوں؟ {#automatic-sweep}

خودکار سویپس صرف <span dir="ltr">2048</span> سے زیادہ کے اضافی بیلنس کے ساتھ ہوں گے۔ دیگر تمام جزوی انخلا کے لیے، آپ کو انہیں دستی طور پر متحرک کرنے کی ضرورت ہوگی۔

### کیا میں اپنا ارادہ بدل کر <span dir="ltr">Type 2</span> سے واپس <span dir="ltr">Type 1</span> پر جا سکتا ہوں؟ {#go-back-to-type1}

نہیں۔ **<span dir="ltr">Type 2</span>** میں تبدیل کرنا ناقابل واپسی ہے۔

### اگر میں متعدد توثیق کاروں کو مستحکم کرنا چاہتا ہوں، تو کیا مجھے پہلے ہر ایک کو <span dir="ltr">Type 2</span> میں تبدیل کرنا ہوگا؟ {#consolidate-multiple-validators}

جی نہیں! ایک توثیق کار کو <span dir="ltr">Type 2</span> میں تبدیل کریں پھر اسے ہدف کے طور پر استعمال کریں۔ اس <span dir="ltr">Type 2</span> ہدف میں مستحکم کیے گئے دیگر تمام توثیق کار <span dir="ltr">Type 1</span> یا <span dir="ltr">Type 2</span> ہو سکتے ہیں۔

### میرا توثیق کار آف لائن ہے یا <span dir="ltr">32 ETH</span> سے کم ہے - کیا میں اب بھی اسے تبدیل کر سکتا ہوں؟ {#offline-or-below-32eth}

ہاں۔ جب تک یہ فعال ہے (خروج نہیں کیا گیا) اور آپ اس کے انخلا کے پتے کے ساتھ دستخط کر سکتے ہیں، آپ اسے تبدیل کر سکتے ہیں۔
