사용자가 이더리움을 자신 있게 탐색할 수 있도록, 다양한 기능을 갖춘 지갑 환경을 포괄하는 여러 지갑을 보여드리고자 합니다.

누구나 자유롭게 ethereum.org에 지갑 추가를 제안할 수 있습니다. 저희가 놓친 지갑이 있다면 제안해 주세요!

현재 지갑이 나열된 곳:

- [ethereum.org/wallets/find-wallet/](/wallets/find-wallet/)

이더리움에서 지갑은 빠르게 변화하고 있습니다. ethereum.org에서는 공정한 검토 기준을 마련하기 위해 노력해 왔지만, 등록 기준은 시간이 지남에 따라 변경되고 발전할 것입니다.

## 결정 프레임워크 {#the-decision-framework}

### 포함 기준: 필수 요건 {#the-must-haves}

- **보안 테스트를 거친 제품** - 감사, 내부 보안 팀, 오픈 소스 코드 또는 기타 방법을 통해 지갑의 보안이 신뢰할 수 있어야 합니다. 이는 사용자의 위험을 줄이고 보안을 중요하게 생각한다는 것을 보여줍니다.
- **6개월 이상 "라이브" 상태였거나 평판이 좋은 그룹에서 출시한 지갑** - 이는 보안의 또 다른 지표입니다. 6개월은 치명적인 버그와 악용 사례가 발견되기에 충분한 시간입니다. 프로젝트로서 빠르게 버려지는 포크를 걸러내기 위해 6개월을 요구합니다.
- **활발한 팀의 참여** - 이는 품질을 보장하고 사용자가 문의에 대한 지원을 받을 수 있도록 돕습니다.
- **정직하고 정확한 등록 정보** - 프로젝트에서 제안하는 모든 등록 정보는 정직하고 정확해야 합니다. 오픈 소스가 아님에도 "오픈 소스"라고 선언하는 등 등록 정보를 위조하는 제품은 삭제됩니다.
- **연락처** - 지갑의 연락처는 변경 사항이 있을 때 정확한 정보를 얻는 데 큰 도움이 됩니다. 이를 통해 향후 정보를 수집할 때 ethereum.org 업데이트를 관리하기 쉽게 유지할 수 있습니다.
- **EIP-1559 (유형 2) 트랜잭션** - 지갑은 메인넷 이더리움의 트랜잭션을 위해 EIP-1559 (유형 2) 트랜잭션을 지원해야 합니다.
- **우수한 사용자 경험** - UX는 주관적이지만, 여러 핵심 팀원이 제품을 테스트하고 사용하기 어렵다고 판단하는 경우, 지갑 등록을 거부할 권리가 있으며 대신 개선을 위한 유용한 제안을 제공할 것입니다. 이는 주로 초보자로 구성된 사용자 기반을 보호하기 위한 조치입니다.
- **이더리움 중심** - 지갑은 주로 이더리움에 초점을 맞춘 경험을 제공해야 합니다. 즉, 이더리움(또는 모든 레이어 2 (l2))이 기본 네트워크로 설정되어 있고, ERC 자산이 제대로 지원되며, 기능이 이더리움 생태계와 일치해야 합니다. UI에서 대체 레이어 1을 우선시하는 지갑은 등록되지 않습니다. 

### 제품 삭제 {#product-removals}

- **업데이트된 정보** - 지갑 제공자는 제공된 정보의 유효성과 관련성을 보장하기 위해 6개월마다 지갑 정보를 다시 제출할 책임이 있습니다(제품에 변경 사항이 없더라도). 제품 팀이 이를 이행하지 않을 경우, ethereum.org는 페이지에서 해당 프로젝트를 삭제할 수 있습니다. 

### 기타 기준: 권장 요건 {#the-nice-to-haves}

- **전 세계적인 접근성** - 지갑에 지리적 제한이나 특정 사람들의 서비스 접근을 배제하는 KYC 요구 사항이 없습니다.
- **다국어 지원** - 지갑이 여러 언어로 번역되어 전 세계 사용자가 접근할 수 있습니다.
- **오픈 소스** - 전체 프로젝트의 코드베이스(모듈뿐만 아니라)에 접근할 수 있어야 하며, 더 넓은 커뮤니티의 PR을 수락해야 합니다.
- **비수탁형** - 사용자가 자신의 자금을 통제합니다. 제품이 사라지더라도 사용자는 여전히 자금에 접근하고 이동할 수 있습니다.
- **하드웨어 월렛 지원** - 사용자가 하드웨어 월렛을 연결하여 트랜잭션에 서명할 수 있습니다.
- **WalletConnect** - 사용자가 WalletConnect를 사용하여 탈중앙화 애플리케이션 (dapp)에 연결할 수 있습니다.
- **이더리움 RPC 엔드포인트 가져오기** - 사용자가 노드 RPC 데이터를 가져와서 원하는 노드나 다른 EVM 호환 네트워크에 연결할 수 있습니다.
- **NFT** - 사용자가 지갑에서 자신의 NFT를 보고 상호작용할 수 있습니다.
- **이더리움 애플리케이션 연결** - 사용자가 이더리움 애플리케이션에 연결하고 사용할 수 있습니다.
- **스테이킹** - 사용자가 지갑을 통해 직접 스테이킹할 수 있습니다.
- **스왑** - 사용자가 지갑을 통해 토큰을 스왑할 수 있습니다.
- **멀티체인 네트워크** - 지갑이 기본적으로 사용자가 여러 블록체인 네트워크에 접근할 수 있도록 지원합니다.
- **레이어 2 네트워크** - 지갑이 기본적으로 사용자가 레이어 2 (l2) 네트워크에 접근할 수 있도록 지원합니다.
- **가스 수수료 맞춤 설정** - 지갑이 사용자가 트랜잭션 가스 수수료(기본 수수료, 우선순위 수수료, 최대 수수료)를 맞춤 설정할 수 있도록 허용합니다.
- **ENS 지원** - 지갑이 사용자가 ENS 이름으로 트랜잭션을 보낼 수 있도록 허용합니다.
- **ERC-20 지원** - 지갑이 사용자가 ERC-20 토큰 컨트랙트를 가져오거나, 자동으로 ERC-20 토큰을 쿼리하고 표시할 수 있도록 허용합니다.
- **암호화폐 구매** - 지갑이 사용자가 직접 암호화폐를 구매하고 온보딩할 수 있도록 지원합니다.
- **법정화폐로 판매** - 지갑이 사용자가 암호화폐를 판매하고 카드나 은행 계좌로 직접 법정화폐를 인출할 수 있도록 지원합니다.
- **다중서명** - 지갑이 트랜잭션 서명을 위해 다중서명을 지원합니다.
- **소셜 복구** - 지갑이 보호자를 지원하며, 사용자가 시드 구문을 분실한 경우 이 보호자들을 사용하여 지갑을 복구할 수 있습니다.
- **전담 지원 팀** - 지갑에 사용자가 문제를 겪을 때 문의할 수 있는 전담 지원 팀이 있습니다.
- **교육 자료/문서** - 제품에는 사용자를 돕고 교육하기 위해 잘 설계된 온보딩 경험이 있어야 합니다. 또는 기사나 동영상과 같은 사용 방법 콘텐츠의 증거가 있어야 합니다.

## 유지 관리 {#maintenance}

이더리움의 유동적인 특성상 팀과 제품이 생겨나고 사라지며 매일 혁신이 일어나고 있으므로, 저희는 다음과 같은 목적으로 콘텐츠에 대한 정기적인 점검을 수행할 것입니다.

- 등록된 모든 지갑과 탈중앙화 애플리케이션 (dapp)이 여전히 당사의 기준을 충족하는지 확인
- 현재 등록된 제품보다 당사의 기준을 더 많이 충족하는 제안된 제품이 없는지 확인

ethereum.org는 오픈 소스 커뮤니티에 의해 유지 관리되며, 최신 상태를 유지하기 위해 커뮤니티의 도움에 의존하고 있습니다. 등록된 지갑에 대해 업데이트가 필요한 정보를 발견하면 [이슈를 열거나](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=wallet+%3Apurse%3A&template=suggest_wallet.yaml) [풀 리퀘스트](https://github.com/ethereum/ethereum-org-website/pulls)를 보내주세요!
