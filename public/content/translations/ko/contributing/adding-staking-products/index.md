사용자의 안전과 신뢰를 유지하면서 가능한 최고의 리소스를 나열하고자 합니다.

누구나 자유롭게 ethereum.org에 스테이킹 제품 또는 서비스 추가를 제안할 수 있습니다. 저희가 놓친 것이 있다면 **[제안해 주세요](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A%2Ccontent+%3Afountain_pen%3A&template=suggest_staking_product.yaml)!**

현재 다음 페이지에 스테이킹 제품 및 서비스를 나열하고 있습니다.

- [솔로 스테이킹](/staking/solo/)
- [서비스형 스테이킹](/staking/saas/)
- [스테이킹 풀](/staking/pools/)

비콘 체인(Beacon Chain)의 지분 증명(PoS)은 2020년 12월 1일부터 가동되었습니다. 스테이킹은 아직 비교적 새로운 분야이므로 ethereum.org에서는 공정하고 투명한 검토 기준을 마련하고자 노력해 왔습니다. 하지만 등재 기준은 시간이 지남에 따라 변경되고 발전할 것이며, 최종 결정은 ethereum.org 웹사이트 팀의 재량에 달려 있습니다.

## 결정 프레임워크 {#the-decision-framework}

ethereum.org에 제품을 등재하는 결정은 어느 한 가지 요소에만 의존하지 않습니다. 제품이나 서비스를 등재할 때는 여러 기준을 종합적으로 고려합니다. 이러한 기준을 더 많이 충족할수록 등재될 가능성이 높아집니다.

**첫째, 어떤 범주의 제품 또는 서비스인가요?**

- 노드 또는 클라이언트 도구
- 키 관리
- 서비스형 스테이킹(SaaS)
- 스테이킹 풀

현재는 이러한 범주의 제품 또는 서비스만 등재하고 있습니다.

### 포함 기준 {#criteria-for-inclusion}

제출된 스테이킹 제품 또는 서비스는 다음 기준에 따라 평가됩니다.

**프로젝트 또는 서비스가 언제 출시되었나요?**

- 제품 또는 서비스가 대중에게 공개된 시점을 증명할 수 있는 자료가 있나요?
- 이는 제품의 "실전 테스트(battle tested)" 점수를 결정하는 데 사용됩니다.

**프로젝트가 활발하게 유지 관리되고 있나요?**

- 프로젝트를 개발하는 활발한 팀이 있나요? 누가 참여하고 있나요?
- 활발하게 유지 관리되는 제품만 고려 대상이 됩니다.

**제품 또는 서비스에 신뢰해야 하는/인적 중개자가 없나요?**

- 사용자의 여정 중 자금의 키를 보관하거나 보상을 적절히 분배하기 위해 사람을 신뢰해야 하는 단계는 무엇인가요?
- 이는 제품 또는 서비스의 "무신뢰" 점수를 결정하는 데 사용됩니다.

**프로젝트가 정확하고 신뢰할 수 있는 정보를 제공하나요?**

- 제품의 웹사이트에 최신 상태의 정확하고 오해의 소지가 없는 정보가 포함되어 있는 것이 매우 중요하며, 특히 이더리움 프로토콜이나 기타 관련 기술과 관련된 경우 더욱 그렇습니다.
- 이더리움이나 기타 관련 주제에 대해 잘못된 정보, 오래된 세부 정보 또는 오해의 소지가 있는 진술이 포함된 제출물은 등재되지 않으며, 이미 등재된 경우 삭제됩니다.

**어떤 플랫폼을 지원하나요?**

- 예: Linux, macOS, Windows, iOS, Android

#### 소프트웨어 및 스마트 컨트랙트 {#software-and-smart-contracts}

관련된 맞춤형 소프트웨어 또는 스마트 컨트랙트의 경우:

**모든 것이 오픈 소스인가요?**

- 오픈 소스 프로젝트는 공개적으로 접근 가능한 소스 코드 저장소를 갖추어야 합니다.
- 이는 제품의 "오픈 소스" 점수를 결정하는 데 사용됩니다.

**제품이 _베타_ 개발 단계를 벗어났나요?**

- 제품이 개발 주기의 어느 단계에 있나요?
- 베타 단계의 제품은 ethereum.org 포함 대상으로 고려되지 않습니다.

**소프트웨어가 외부 보안 감사를 거쳤나요?**

- 그렇지 않다면, 외부 감사를 진행할 계획이 있나요?
- 이는 제품의 "감사 완료" 점수를 결정하는 데 사용됩니다.

**프로젝트에 버그 바운티 프로그램이 있나요?**

- 그렇지 않다면, 보안 버그 바운티를 만들 계획이 있나요?
- 이는 제품의 "버그 바운티" 점수를 결정하는 데 사용됩니다.

#### 노드 또는 클라이언트 도구 {#node-or-client-tooling}

노드 또는 클라이언트 설정, 관리, 마이그레이션과 관련된 소프트웨어 제품의 경우:

**어떤 합의 레이어 클라이언트(예: 라이트하우스, 테쿠, 님버스, 프리즘, Grandine)를 지원하나요?**

- 어떤 클라이언트를 지원하나요? 사용자가 선택할 수 있나요?
- 이는 제품의 "다중 클라이언트" 점수를 결정하는 데 사용됩니다.

#### 서비스형 스테이킹 {#staking-as-a-service}

[서비스형 스테이킹 등재](/staking/saas/)(예: 위임된 노드 운영)의 경우:

**서비스 이용과 관련된 수수료는 얼마인가요?**

- 수수료 구조는 어떻게 되나요? (예: 서비스에 대한 월별 수수료가 있나요?)
- 추가적인 스테이킹 요구 사항이 있나요?

**사용자가 계정에 가입해야 하나요?**

- 허가나 KYC 없이 누구나 서비스를 이용할 수 있나요?
- 이는 제품의 "무허가성" 점수를 결정하는 데 사용됩니다.

**서명 키와 인출 키는 누가 보관하나요?**

- 사용자는 어떤 키에 대한 접근 권한을 유지하나요? 서비스는 어떤 키에 대한 접근 권한을 얻나요?
- 이는 제품의 "무신뢰" 점수를 결정하는 데 사용됩니다.

**운영 중인 노드의 클라이언트 다양성은 어떠한가요?**

- 다수결 합의 레이어(CL) 클라이언트가 실행하는 검증자 키의 비율은 얼마나 되나요?
- 마지막 편집 기준으로, 프리즘(Prysm)은 대다수의 노드 운영자가 실행하는 합의 레이어 클라이언트이며, 이는 네트워크에 위험할 수 있습니다. 현재 네트워크의 33% 이상이 특정 CL 클라이언트를 사용하고 있다면, 해당 사용량과 관련된 데이터를 요청합니다.
- 이는 제품의 "다양한 클라이언트" 점수를 결정하는 데 사용됩니다.

#### 스테이킹 풀 {#staking-pool}

[풀 스테이킹 서비스](/staking/pools/)의 경우:

**스테이킹에 필요한 최소 ETH는 얼마인가요?**

- 예: 0.01 ETH

**관련된 수수료나 스테이킹 요구 사항은 무엇인가요?**

- 보상의 몇 퍼센트가 수수료로 차감되나요?
- 추가적인 스테이킹 요구 사항이 있나요?

**유동성 토큰이 있나요?**

- 관련된 토큰은 무엇인가요? 어떻게 작동하나요? 컨트랙트 주소는 무엇인가요?
- 이는 제품의 "유동성 토큰" 점수를 결정하는 데 사용됩니다.

**사용자가 노드 운영자로 참여할 수 있나요?**

- 풀링된 자금을 사용하여 검증자 클라이언트를 실행하려면 무엇이 필요한가요?
- 개인, 회사 또는 DAO의 허가가 필요한가요?
- 이는 제품의 "무허가성 노드" 점수를 결정하는 데 사용됩니다.

**풀 노드 운영자의 클라이언트 다양성은 어떠한가요?**

- 다수결 합의 레이어(CL) 클라이언트를 실행하는 노드 운영자의 비율은 얼마나 되나요?
- 마지막 편집 기준으로, 프리즘(Prysm)은 대다수의 노드 운영자가 실행하는 합의 레이어 클라이언트이며, 이는 네트워크에 위험할 수 있습니다. 현재 네트워크의 33% 이상이 특정 CL 클라이언트를 사용하고 있다면, 해당 사용량과 관련된 데이터를 요청합니다.
- 이는 제품의 "다양한 클라이언트" 점수를 결정하는 데 사용됩니다.

### 기타 기준: 있으면 좋은 것들 {#other-criteria}

**어떤 사용자 인터페이스를 지원하나요?**

- 예: 브라우저 앱, 데스크톱 앱, 모바일 앱, CLI

**노드 도구의 경우, 소프트웨어가 클라이언트 간에 쉽게 전환할 수 있는 방법을 제공하나요?**

- 사용자가 도구를 사용하여 쉽고 안전하게 클라이언트를 변경할 수 있나요?

**SaaS의 경우, 현재 서비스에서 운영 중인 검증자는 몇 명인가요?**

- 이를 통해 지금까지 서비스가 얼마나 도달했는지 파악할 수 있습니다.

## 결과 표시 방법 {#product-ordering}

위의 [포함 기준](#criteria-for-inclusion)은 각 제품 또는 서비스의 누적 점수를 계산하는 데 사용됩니다. 이는 특정 객관적 기준을 충족하는 제품을 정렬하고 소개하는 수단으로 사용됩니다. 증거가 제공된 기준이 많을수록 제품이 더 높은 순위로 정렬되며, 동점일 경우 로드 시 무작위로 배치됩니다.

이러한 기준에 대한 코드 로직과 가중치는 현재 저장소의 [이 JavaScript 컴포넌트](https://github.com/ethereum/ethereum-org-website/blob/dev/src/components/Staking/StakingProductsCardGrid/index.tsx#L350)에 포함되어 있습니다.
