デザインはあらゆるプロジェクトにおいて重要な要素です。あなたの時間とデザインスキルをethereum.orgに注ぐことで、訪問者のユーザーエクスペリエンス向上に貢献できます。オープンソースプロジェクトへの貢献は、関連する経験を積み、協力的な環境でスキルを伸ばす機会を提供します。他のデザイナー、開発者、コミュニティメンバーと協力する機会があり、彼らはそれぞれ独自の視点や洞察を持っています。

最終的に、これはあなたのデザインスキルを示す多様で印象的なポートフォリオを構築するための素晴らしい方法です。

## 貢献する方法 {#how-to-contribute}

### <Emoji text=":one:" size={1} /> &nbsp;初期デザインプロトタイプへのフィードバックの提供 {#design-critique}

私たちは時折、初期のアイデアをテストするための助けを必要とします。これは、技術的な知識がなくても貢献できる素晴らしい方法です。

1. デザインチームが[ディスコード](https://discord.com/invite/ethereum-org)および[GitHub](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8)でモックアップデザインを共有します。
2. コメント機能を通じてフィードバックを提供できるよう、デザインの案内が行われます。
3. 結果はGitHubのIssueで共有され、その後チームによってクローズされます。

### <Emoji text=":two:" size={1} /> &nbsp;アンケート調査への参加 {#answer-surveys}

以下の方法でウェブサイトに関するフィードバックを提供してください。

1. ethereum.orgにアクセスし、ページを読む。
2. 右下にあるフィードバックウィジェットをクリックし、デザインやコンテンツに関する質問に答える。
3. 自由記述形式の質問に焦点を当てる。

### <Emoji text=":three:" size={1} /> &nbsp;ウェブサイト上のデザイン関連のIssueを見つけて報告する {#report-design-issues}

ethereum.orgは、多くの機能とコンテンツを持つ急成長中のウェブサイトです。一部のUIはすぐに古くなったり、改善の余地があったりします。そのような例に遭遇した場合は、私たちが対応できるように報告してください。

1. ウェブサイトを見て回り、デザインに注目する。
2. 視覚的またはUX上の問題を見つけた場合は、スクリーンショットを撮り、メモを残す。
3. [バグレポート](https://github.com/ethereum/ethereum-org-website/issues/new/choose)を使用して、見つけた問題を報告する。

### <Emoji text=":four:" size={1} /> &nbsp;デザイン変更の提案 {#propose-design-changes}

デザインの課題に取り組むことに抵抗がなければ、GitHubのIssueボードにアクセスし、[デザイン関連のIssue](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8)でフィルタリングすることができます。

1. ウェブサイトを見て回りデザインに注目するか、GitHubリポジトリにアクセスして[「Design required」タグ](https://github.com/ethereum/ethereum-org-website/labels/design%20required%20%F0%9F%8E%A8)が付いたIssueを確認する。
2. 解決策を考案し、デザインする（理想的には私たちの[デザインシステム](https://www.figma.com/community/file/1134414495420383395)を使用する）。
3. 該当するGitHubのIssueで解決策を提案するか、[新しく作成する。](https://github.com/ethereum/ethereum-org-website/issues/new?assignees=&labels=feature+%3Asparkles%3A&template=feature_request.yaml&title=Feature+request)
4. デザインチームのレビューを待つ。

### <Emoji text=":five:" size={1} /> &nbsp;デザインシステムを一緒に構築する {#contribute-to-design-system}

私たちのデザインシステムにより、ethereum.orgのデザインが楽しく簡単になります。経験豊富なデザイナーであれば、ウェブサイト用の多くのコンポーネントを準備するのを手伝うことができます。

1. GitHubの[デザインシステムボード](https://github.com/ethereum/ethereum-org-website/labels/design%20system)から取り組むIssueを選択するか、新しく作成する。
2. 選択したIssueを自分に割り当てるようリクエストする。
3. Figmaでリクエストされたコンポーネントのデザインを開始する。
4. レビューやガイダンスが必要になったら、GitHub上でデザインチームと共有する。
5. デザインチームがレビューを行う。
6. デザインチームがメインファイルに変更を組み込み、コミュニティにファイルを公開する。

### <Emoji text=":six:" size={1} /> &nbsp;ウェブサイトにデザイン関連のコンテンツを書く {#write-design-articles}

イーサリアムの開発者コミュニティは強力ですが、デザインコミュニティは少し遅れをとっています。Web3の知識を持つデザイナーであれば、私たちが共に成長し改善できるように、あなたの学びをより広いコミュニティと共有することを検討してください。貢献できる[イーサリアムのデザインに関するページ](/developers/docs/design-and-ux/)があります。また、[掲載ポリシー](/contributing/design/adding-design-resources)も確認できます。

1. ethereum.orgで取り上げるべきであり、この分野のデザイナーにとって有益なデザイントピックを考案する。
2. GitHubリポジトリにアクセスし、トピックを提案する[Issueを立てる](https://github.com/ethereum/ethereum-org-website/issues/new)（まだコンテンツは書かないでください）。
3. デザインチームの承認を待つ。
4. 承認されたら、コンテンツを書く。
5. 該当するGitHubのIssueで提出する。

### <Emoji text=":seven:" size={1} /> &nbsp;新しいイラストを描く {#prepare-illustrations}

視覚化は、抽象的なトピックを説明するための最も強力なツールの1つです。図やインフォグラフィックを追加することには計り知れない可能性があります。結局のところ、1枚の画像は千の言葉を語ることができます。

1. ウェブサイトにアクセスし、新しいインフォグラフィックを追加できそうなページを確認する。
2. イラストのスタイルが私たちの[アセット](/assets/)と一致していることを確認する。
3. GitHubリポジトリにアクセスし、イラストを提案する[Issueを立てる](https://github.com/ethereum/ethereum-org-website/issues/new)。
4. デザインチームがレビューを行う。
5. 新しい画像を実装するよう開発者に依頼するための新しいIssueを作成する。
