サイドチェーンは、[イーサリアム](/)から独立して稼働し、双方向のブリッジによってイーサリアム・メインネットに接続される独立したブロックチェーンです。サイドチェーンは、独自のブロックパラメータや[コンセンサスアルゴリズム](/developers/docs/consensus-mechanisms/)を持つことができ、これらは多くの場合、トランザクションの効率的な処理のために設計されています。ただし、サイドチェーンはイーサリアムのセキュリティ特性を継承しないため、その使用にはトレードオフが伴います。[レイヤー2 (L2) スケーリングソリューション](/layer-2/)とは異なり、サイドチェーンは状態の変更やトランザクションデータをイーサリアム・メインネットにポストしません。

また、サイドチェーンは高いスループットを達成するために、ある程度の分散化やセキュリティを犠牲にします（[スケーラビリティのトリレンマ](https://vitalik.eth.limo/general/2021/05/23/scaling.html)）。しかし、イーサリアムは分散化とセキュリティを妥協することなくスケーリングすることに取り組んでいます。

## サイドチェーンの仕組み {#how-do-sidechains-work}

サイドチェーンは独立したブロックチェーンであり、異なる履歴、開発ロードマップ、および設計上の考慮事項を持っています。サイドチェーンは表面上はイーサリアムといくつかの類似点を共有しているかもしれませんが、いくつかの特徴的な機能を持っています。

### コンセンサスアルゴリズム {#consensus-algorithms}

サイドチェーンをユニークなもの（つまり、イーサリアムとは異なるもの）にしている特徴の1つは、使用されるコンセンサスアルゴリズムです。サイドチェーンはコンセンサスをイーサリアムに依存せず、ニーズに合った代替のコンセンサスプロトコルを選択できます。サイドチェーンで使用されるコンセンサスアルゴリズムの例には、以下のものがあります。

- [プルーフ・オブ・オーソリティ](/developers/docs/consensus-mechanisms/poa/)
- [デリゲーテッド・プルーフ・オブ・ステーク](https://en.bitcoin.it/wiki/Delegated_proof_of_stake)
- [ビザンチン・フォールト・トレランス](https://decrypt.co/resources/byzantine-fault-tolerance-what-is-it-explained)

イーサリアムと同様に、サイドチェーンにはトランザクションを検証および処理し、ブロックを生成し、ブロックチェーンの状態を保存する検証ノードがあります。バリデータはまた、ネットワーク全体でコンセンサスを維持し、悪意のある攻撃からネットワークを保護する責任があります。

#### ブロックパラメータ {#block-parameters}

イーサリアムは、[ブロック時間](/developers/docs/blocks/#block-time)（新しいブロックを生成するのにかかる時間）と[ブロックサイズ](/developers/docs/blocks/#block-size)（ガスで表されるブロックごとのデータ量）に制限を設けています。逆に、サイドチェーンは高いスループット、高速なトランザクション、および低手数料を達成するために、より速いブロック時間やより高いガスリミットなど、異なるパラメータを採用することがよくあります。

これにはいくつかの利点がありますが、ネットワークの分散化とセキュリティに重大な影響を及ぼします。速いブロック時間や大きなブロックサイズなどのブロックパラメータは、フル・ノードを実行する難易度を高め、チェーンの保護を少数の「スーパーノード」に委ねることになります。このようなシナリオでは、バリデータの共謀やチェーンの悪意のある乗っ取りの可能性が高まります。

ブロックチェーンが分散化を損なうことなくスケーリングするためには、ノードの実行が誰にでも開かれている必要があり、必ずしも専用のハードウェアを持つ当事者だけのものであってはなりません。これが、誰もがイーサリアムネットワーク上で[フル・ノードを実行](/developers/docs/nodes-and-clients/#why-should-i-run-an-ethereum-node)できるようにするための取り組みが進行中である理由です。

### EVM互換性 {#evm-compatibility}

一部のサイドチェーンはEVM互換であり、[イーサリアム仮想マシン (EVM)](/developers/docs/evm/) 向けに開発されたコントラクトを実行できます。EVM互換のサイドチェーンは、[Solidityで書かれた](/developers/docs/smart-contracts/languages/)スマート・コントラクトや、その他のEVMスマート・コントラクト言語をサポートしています。つまり、イーサリアム・メインネット用に書かれたスマート・コントラクトは、EVM互換のサイドチェーンでも機能します。

これは、サイドチェーンで[分散型アプリケーション (dapp)](/developers/docs/dapps/)を使用したい場合、このサイドチェーンに[スマート・コントラクト](/developers/docs/smart-contracts/)をデプロイするだけで済むことを意味します。見た目も、使い勝手も、動作もメインネットとまったく同じです。Solidityでコントラクトを書き、サイドチェーンのRPCを介してチェーンとやり取りします。

サイドチェーンはEVM互換であるため、イーサリアムネイティブのdappにとって有用な[スケーリングソリューション](/developers/docs/scaling/)と見なされています。dappをサイドチェーンに配置することで、特にメインネットが混雑している場合、ユーザーはより低いガス代とより高速なトランザクションを享受できます。

ただし、前述のように、サイドチェーンの使用には重大なトレードオフが伴います。各サイドチェーンは独自のセキュリティに責任を持ち、イーサリアムのセキュリティ特性を継承しません。これにより、ユーザーに影響を与えたり、資金を危険にさらしたりする可能性のある悪意のある行動のリスクが高まります。

### 資産の移動 {#asset-movement}

独立したブロックチェーンがイーサリアム・メインネットのサイドチェーンになるためには、イーサリアム・メインネットとの間での資産の送金を容易にする機能が必要です。イーサリアムとのこのインターオペラビリティは、ブロックチェーンブリッジを使用して実現されます。[ブリッジ](/bridges/)は、イーサリアム・メインネットとサイドチェーンにデプロイされたスマート・コントラクトを使用して、それらの間の資金のブリッジングを制御します。

ブリッジはユーザーがイーサリアムとサイドチェーンの間で資金を移動するのに役立ちますが、資産が2つのチェーン間で物理的に移動するわけではありません。代わりに、チェーン間で価値を送金するために、通常はミンティングとバーンを伴うメカニズムが使用されます。[ブリッジの仕組み](/developers/docs/bridges/#how-do-bridges-work)の詳細。

## サイドチェーンのメリットとデメリット {#pros-and-cons-of-sidechains}

| メリット | デメリット |
| --------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| サイドチェーンを支える技術は確立されており、広範な研究と設計の改善の恩恵を受けています。 | サイドチェーンは、スケーラビリティのためにある程度の分散化とトラストレス性をトレードオフにします。 |
| サイドチェーンは一般的な計算をサポートし、EVM互換性を提供します（イーサリアムネイティブのdappを実行できます）。 | サイドチェーンは独自のコンセンサス・メカニズムを使用しており、イーサリアムのセキュリティ保証の恩恵を受けません。 |
| サイドチェーンは異なるコンセンサスモデルを使用してトランザクションを効率的に処理し、ユーザーのトランザクション手数料を削減します。 | サイドチェーンはより高いトラスト前提を必要とします（例：悪意のあるサイドチェーンバリデータのクオラムが不正を働く可能性があります）。 |
| EVM互換のサイドチェーンにより、dappはそのエコシステムを拡大できます。 | |
